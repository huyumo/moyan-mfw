🏗️ 最终方案：方案 B — workspace 内部引用 + 构建时路径转换

源码用 @internal/xxx 开发，发布时 tsc-alias 转为 npm 路径

📐 核心原则

开发时（workspace 内）
所有跨包引用使用 @internal/base-backend / @internal/base-frontend / @internal/base-shared
pnpm workspace 的 node_modules 软链接自动解析 → 标准 JS 模块解析，零 tsconfig hack

构建时（npm publish 前）
tsc 编译后用 tsc-alias 将 @internal/xxx 转为消费者路径：
@internal/base-backend → moyan-mfw-base/backend
@internal/base-shared → moyan-mfw-base/shared

消费时（npm install）
用户 npm i moyan-mfw-extension-ad，dist 里的 require 是 moyan-mfw-base/backend
Node.js 通过 exports 字段解析到 dist/backend/index.js → 标准 npm 行为
