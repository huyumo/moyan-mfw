确认方案 A，但需补充 3 个设计约束

生成到扩展包内部 + 约束机制解决路由冲突 / npm 发布 / 内部页面调用

约束 1：命名空间路由前缀

每个扩展包在 extension.json 声明 routePrefix: "/ext/ad"

所有 definePageConfig.path 必须以此为前缀

框架 CI / 启动时校验：重复前缀 → 报错

约束 2：扩展包内置 API 生成

每个扩展包自带 api.build.cjs，从自己的 Swagger JSON 生成到 src/frontend/apis/

通过 package.json exports 暴露 ./frontend/apis

业务层：import { ApiAdFindAll } from 'moyan-extension-ad/frontend/apis'

约束 3：路由按组拆分导出

export const adTypeRoutes / adPlacementRoutes / adContentRoutes

业务层按需引入，自定义页面可替换某一组

全量导出 adRoutes 向后兼容

完整调用链

扩展包内部 Vue 组件
  └─ import { ApiAdFindAll } from '../../apis'  ← 就近引用

业务层自定义页面
  └─ import { ApiAdFindAll } from 'moyan-extension-ad/frontend/apis'

业务层 Service
  └─ import { AdService } from 'moyan-extension-ad/backend'  ← 直接注入
