🔴 #6：peerDependencies 的 "workspace:*" → npm 发布后写什么？

当前 package.json："moyan-base-backend": "workspace:*"。npm 发布时必须换成语义化版本号，否则业务层 npm install 会失败。

方案 A：pnpm publish 自动转换（推荐）

pnpm 原生支持："workspace:^" → publish 时自动变成 "^1.0.0"

// 开发期
"moyan-base-backend": "workspace:^"

// pnpm publish 后自动变成
"moyan-base-backend": "^1.2.0"

零脚本，pnpm 内置。版本号取自目标包实际的 package.json version。

方案 B：prepublishOnly 脚本手动替换

写 node 脚本读取 base-backend 的 package.json version，再替换 peerDependencies

缺点：多写一个脚本，增加维护负担

📌 推荐方案 A：workspace:^ 自动转换

当前写法改一下："moyan-base-backend": "workspace:^"

  • workspace:^ → publish 后 ^1.0.0（兼容同主版本）
  • workspace:~ → publish 后 ~1.2.0（兼容同次版本）
  • workspace:* → publish 后 1.2.0（精确版本）

扩展包推荐用 workspace:^，允许业务层使用兼容的 base-backend 新版本。
