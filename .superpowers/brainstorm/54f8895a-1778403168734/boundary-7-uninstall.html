🔴 #7：卸载扩展包后，数据库表 & 权限节点的清理

业务层从 extensions 配置中移除一个扩展包 → 重启后，数据库表和权限节点还在

方案 A：标注为"未启用"，不删除数据（推荐一期）

卸载时框架检测到扩展包不在配置中 → 标记状态为 disabled

表保留（避免误删数据），权限节点保留但不可分配

清理：提供 CLI 命令 ext:cleanup moyan-extension-ad，由开发者手动执行

类比：WordPress 卸载插件 → 表和选项保留在数据库中

方案 B：动删除（危险）

移除配置 → 启动时自动 DROP TABLE + 删除权限节点

优点：干净

风险：误删生产数据不可恢复

📌 推荐方案 A：soft-disabled + 手动清理 CLI

  • 启动时检测：配置中的扩展包 ≠ 已安装的扩展包 → 标记 disabled
  • extension.json 新增 onUninstall 生命周期钩子（未来实现）
  • 清理操作永远是 开发者手动触发，框架不自动删任何数据
