🔴 #8：两个扩展包声明了同名 permCode

扩展包 A：payment:order:view | 扩展包 B：payment:order:view → 冲突

这是最后一个 🔴 架构级问题

A

方案 A：启动时冲突检测 → 报错终止（推荐）

框架启动时扫描所有扩展包的 permCodeNodes，发现重复 permCode → 拒绝启动

错误信息：

⚠️  permCode 冲突：
  "payment:order:view" 被以下扩展包同时声明：
  - moyan-extension-payment
  - moyan-extension-payment-plus
请选择一个保留，或联系扩展包提供方解决
B

方案 B：业务层映射重命名

业务层在 extension.json 或配置中提供 remap

permCodeRemap: {
  "moyan-extension-payment-plus:payment:order:view": "payment-plus:order:view"
}

缺点：增加复杂度；重映射后扩展包内部代码仍用旧 permCode

C

方案 C：先注册者优先，后注册者跳过

按扩展包加载顺序，第一个声明的生效，后面的跳过并警告

缺点：隐性行为，开发者不知道哪个扩展包的权限"丢"了

📌 推荐方案 A：拒绝启动 + 明确错误

与清单校验器思路一致：发现问题 → 立即报错，不猜测意图。

这是启动时的一次性检查，零运行时开销。
