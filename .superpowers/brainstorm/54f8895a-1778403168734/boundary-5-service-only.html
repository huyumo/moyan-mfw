🔴 #5：业务层只用 Service，不想暴露扩展包 Controller

场景：业务层想用 AdService 内部创建广告，但不需要 AdController 暴露 HTTP API

方案 A：扩展包拆分 Module → 业务层按需导入

扩展包提供两个 Module：

// extension-ad 导出
export { AdCoreModule }     // 仅 Service + Entity（无 Controller）
export { AdModule }         // CoreModule + Controller（完整）

业务层只用 Service：modules: [AdCoreModule]

业务层要完整 API：modules: [AdModule]

方案 B：Controller 用 @Optional 守卫控制

框架提供 @ExtensionGuard，业务层配置中可禁用某个扩展包的 Controller

缺点：Controller 仍被注册，只是拦截请求；不如不注册干净

📌 推荐方案 A

每个扩展包都导出 CoreModule + 完整 Module。

Core 只含 Service + Entity + DTO，零 HTTP 暴露。业务层精确控制暴露面。
