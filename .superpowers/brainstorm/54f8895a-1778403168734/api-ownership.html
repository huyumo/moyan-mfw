扩展包 API 代码应放在哪里？

moyan-api 从 Swagger JSON 自动生成 API 调用类，生成的代码需要有一个"家"

A

方案 A：放在业务层 base-frontend

生成到 packages/base-frontend/src/apis/extension-ad/

优点：与现有 sys API 同一体系，业务层直接 import；api.build.cjs 只需加一个源

缺点：扩展包自身的 Vue 组件需要跨包引用 API 类

B

方案 B：放在扩展包内部

生成到 extension-ad/src/frontend/apis/

优点：扩展包自包含，Vue 组件就近引用；npm 发布时自带 API 代码

缺点：生成的代码需独立配置源；业务层引用路径不统一

C

方案 C：双份生成 + re-export 桥接

生成到 base-frontend/src/apis/extension-ad/，扩展包通过 frontend/apis/index.ts re-export

优点：兼顾两方，业务层和扩展包都自然引用

缺点：文件归属权模糊（生成代码在 base-frontend，re-export 在 extension）
