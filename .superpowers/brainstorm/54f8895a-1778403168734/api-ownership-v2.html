扩展包 API 代码应放在哪里？

修正：base-frontend 是框架基础设施，业务层和扩展层都依赖它。
API 代码生成产物不能写入框架层目录。

A

方案 A：生成到扩展包内部

生成到 extension-ad/src/frontend/apis/

做法：扩展包自身维护 api.build.cjs，或业务层 api.build.cjs 中 output 指向扩展包目录

优点：扩展包自包含，Vue 组件就近引用；npm 发布自带 API 代码；业务层 import 路径统一

缺点：需要确认扩展包 package.json exports 暴露 ./frontend/apis 子路径

B

方案 B：生成到业务层 frontend/

生成到 frontend/src/apis/extension-ad/

做法：业务层 api.build.cjs 增加扩展包 Swagger 源，output 指向业务层自身

优点：业务层完全掌控 API 代码；无需修改扩展包 exports

缺点：扩展包的 Vue 组件需从业务层 import API → 循环依赖风险；npm 发布时 API 代码不在扩展包内

C

方案 C：扩展包提供 Service 层接口，不暴露原始 API

扩展包封装一个 AdSdk 类，内部调用 API，对外暴露方法

做法：扩展包用 moyan-api 生成代码放在内部，再封装一层语义化 SDK

优点：最大封装性；业务层只需 AdSdk.getPlacements()；API 变更对业务层透明

缺点：多一层抽象；SDK 方法需与 API 保持同步
