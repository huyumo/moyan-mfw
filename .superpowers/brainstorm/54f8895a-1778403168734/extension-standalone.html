扩展包自启动 + 统一 Core

扩展包 = 特殊业务层。有自己的 main.ts，但 bootstrap 逻辑由 core 提供

当前：业务层

// backend/src/main.ts
import { createBaseBackendApp } from 'moyan-base-backend'
import { AppModule } from './app.modules'

createBaseBackendApp({
  modules: [AppModule],
  name: '墨焱业务后端',
}).then(app => app.listen(3000))

目标：扩展包

// extension-ad/src/backend/main.ts
import { createExtensionBackendApp } from 'moyan-base-backend'
import { AdModule, Ad } from './index'

createExtensionBackendApp({
  name: '广告管理',
  module: AdModule,
  entities: [Ad],
  manifest: extensionManifest,
}).then(({ app }) => app.listen(3002))

Core 统一做的事

  • 启动 NestJS + TypeORM（同 createBaseBackendApp 底层）
  • 注册 Swagger 分组
  • 注册扩展包实体到 TypeORM
  • 注入扩展包权限节点（从 extension.json 读取）
  • 全局异常过滤器 / 拦截器 / ValidationPipe

逻辑复用，零重复。每个扩展包只需声明"我有什么"即可。

开发调试：一行命令

# 终端 1：启动扩展包后端（独立进程）
pnpm --filter moyan-extension-ad dev:backend

# 终端 2：启动扩展包前端（独立 Vite）
pnpm --filter moyan-extension-ad dev:frontend

# 访问 http://localhost:5200/ext/ad/placement → 直接开发扩展包页面
# Swagger：http://localhost:3002/api-docs → 扩展包独有 API 文档

完全不依赖业务层。扩展包自己就是一个完整的"微型应用"。

业务层最终集成时

扩展包开发完 → 业务层引入，也只是声明式配置

// backend/src/main.ts
createBaseBackendApp({
  extensions: [                         // ← 框架层新增字段
    { name: 'moyan-extension-ad', routePrefix: '/ads' },
  ],
  // 框架内部：读取 extension.json → 合并 entities、permCodeNodes → 校验 → 注入
})

// frontend/src/main.ts
createBaseAdminApp({
  extensions: [
    { name: 'moyan-extension-ad', routePrefix: '/ads' },
  ],
})

关键洞察：createExtensionApp 和 createBaseApp 底层是同一套引擎，区别只是前者加载单个扩展包，后者加载多个扩展包 + 业务模块。
