🔴 #2：API 代码（moyan-api 生成）随包发布 or 安装后生成？

扩展包的 src/frontend/apis/ 由 moyan-api 生成。npm 发布时这些文件怎么处理？

方案 A：随包发布（推荐）

发布前跑 api:build，产物提交到 npm。

优点：业务层安装即用，零额外步骤

风险：生成的代码基于发布时的 Swagger，如果后端版本不匹配可能不一致

缓解：API 类只声明 path/method，不携带业务逻辑，跨版本通常兼容

方案 B：安装后生成

发布不含 API 代码。业务层 api.build.cjs 增加扩展包 Swagger 源。

优点：生成的代码与运行时后端完全一致

缺点：需要业务层手动配置 Swagger 源地址；扩展包不再"即装即用"

📌 推荐方案 A：随包发布

原因：

  • API 类本质是类型声明 + 路径常量，换版本几乎不会变
  • 即装即用是扩展包的核心体验 — npm install → import → use
  • 如果真有重大变更，本来就要发 MAJOR 版本

注意：扩展包的 .gitignore 不能忽略 src/frontend/apis/（与业务层相反）
