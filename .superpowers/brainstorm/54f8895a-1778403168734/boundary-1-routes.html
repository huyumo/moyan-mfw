🔴 #1：npm 发布后，import.meta.glob 失效 → 路由如何预构建？

当前 extension-ad 用 import.meta.glob('./views/**/index.{ts,tsx}') 扫描路由
这在 workspace 模式正常工作，但 npm 发布后代码已被 tsc 编译，glob 不会出现在产物中。

方案 A：构建时生成静态路由数组（推荐）

扩展包 package.json prebuild 钩子执行脚本，扫描 views 目录生成 routes.gen.ts

产物：

// extension-ad/src/frontend/routes.gen.ts（自动生成）
import PlacementList from './views/placement/Index.vue'
export const routes = [
  { page: PlacementList, path: 'ext/ad/placement', name: '广告位管理', ... },
]

优点：运行时零开销，npm 完全兼容

代价：加页面后必须跑构建，不跑就是旧路由

方案 B：延迟注册机制

npm 包只暴露 RouteRecordRaw[]，路由数据在 extension.json 中声明

做法：extension.json 新增 routes 字段，列出每个页面的路径和组件名

缺点：手动维护清单，容易漏；组件路径映射复杂

📌 推荐方案 A

这是行业标准做法（Nuxt、VitePress 同理）。构建脚本很小（~30 行），prepublishOnly 前自动跑。

开发期仍用 import.meta.glob（即时），发布时走构建产物。
