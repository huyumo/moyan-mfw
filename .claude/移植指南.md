# 跨设备移植指南

> 版本：1.0.0 | 创建：2026-03-30

---

## 概述

本项目的规则系统采用**相对路径**设计，支持在不同设备间无缝移植。

---

## 文件结构

```
项目根目录/
├── .claude/
│   ├── .claude.md              # 项目宪法（规则引用）
│   ├── config.json             # 详细配置（相对路径）
│   ├── TASK.md                 # 活跃任务追踪
│   └── rules/
│       ├── core/               # 核心规则（会话流程、任务管理、记忆保存）
│       ├── optional/           # 可选规则
│       └── project-specific/   # 项目特定规则
│
└── docs/
    └── 04-项目实施/
        └── 05-任务追踪/        # 任务详情文档
```

---

## 移植方式

### 方式一：完整复制项目目录（推荐）

**步骤**：
1. 复制整个项目目录到新设备
2. 确保 `.claude/` 目录完整复制
3. 在新设备上打开项目

**优点**：
- 所有规则和配置完整保留
- 任务状态、记忆文件全部保留
- Git 历史记录完整

---

### 方式二：仅复制 `.claude/` 目录

**步骤**：
1. 复制 `.claude/` 目录到新设备的相同项目
2. 确保目录结构一致

**适用场景**：
- 项目代码已存在，仅需同步规则配置
- 多台设备开发同一项目

---

### 方式三：Git 同步

**步骤**：
1. 将 `.claude/` 目录纳入 Git 管理
2. 推送到远程仓库
3. 在新设备上 `git pull`

**注意**：
- 需要在 `.gitignore` 中排除敏感文件（如包含 API 密钥）
- `TASK.md` 建议纳入版本控制

## 路径说明

### 相对路径（已实现）

所有配置文件使用相对路径引用：

```json
{
  "rules": {
    "core": {
      "session-flow": {
        "source": ".claude/rules/core/01-session-flow.md"
      }
    }
  }
}
```

**优点**：
- 与设备无关
- 支持任意目录位置
- 跨平台兼容

### 绝对路径（已移除）

~~`C:\Users\16898\.claude\rules\core\...`~~

**问题**：
- 依赖特定用户目录
- 无法在其他设备使用
- Windows/Linux/macOS 路径格式不同

## 用户级记忆处理

### 问题

用户级记忆存放在 `[用户目录]/.claude/memory/`，不同设备路径不同。

### 解决方案

#### 方案 A：使用项目级记忆

**推荐**：将记忆保存在项目目录内

**位置**：`[project]/.claude/memory/`

**修改配置**：
```json
{
  "memory": {
    "save-location": ".claude/memory/"
  }
}
```

**优点**：
- 随项目一起移植
- 设备无关
- 团队成员共享

#### 方案 B：同步用户级记忆

如需要跨项目共享记忆：

1. 在云存储（OneDrive/Dropbox）创建记忆目录
2. 在原位置创建符号链接

**Windows 示例**：
```powershell
mklink /D "C:\Users\用户名\.claude\memory" "D:\OneDrive\Claude\memory"
```

## 常见问题

### Q1: 换设备后规则不生效？

**检查**：
1. `.claude.md` 是否存在于项目根目录
2. `config.json` 中的路径是否正确
3. 规则文件是否存在

**解决**：确保 `.claude/` 目录完整复制

---

### Q2: 多台设备同时工作会冲突吗？

**可能**。多会话锁机制可以检测冲突，但建议：

- 同一时间只在一台设备上工作
- 或在 TASK.md 中记录当前使用的设备

---

### Q3: 用户级记忆如何同步？

**推荐**：使用项目级记忆代替用户级记忆

如必须同步用户级记忆，参考"方案 B：同步用户级记忆"
