# 技术主管角色检查清单

## 架构设计

- [ ] 技术选型是否符合项目约束
- [ ] 系统边界是否清晰
- [ ] 模块职责是否单一
- [ ] 接口定义是否稳定
- [ ] 数据一致性如何保证
- [ ] 是否有单点故障风险

## 代码质量

- [ ] 代码是否符合现有规范
- [ ] 是否有重复代码可抽取
- [ ] 函数/类职责是否单一
- [ ] 错误处理是否完整
- [ ] 边界条件是否处理
- [ ] 是否有安全漏洞（注入、XSS、CSRF 等）

## 性能考虑

- [ ] 是否有 N+1 查询问题
- [ ] 是否有缓存策略
- [ ] 大数据量场景是否考虑
- [ ] 是否有性能瓶颈风险

## 可维护性

- [ ] 代码是否可读
- [ ] 关键逻辑是否有注释
- [ ] 是否有单元测试覆盖
- [ ] 配置是否可外部化
- [ ] 日志是否便于排查问题

## 与现有系统集成

- [ ] 是否复用现有组件
- [ ] 是否影响现有功能
- [ ] 数据库变更是否兼容
- [ ] API 变更是否向后兼容
