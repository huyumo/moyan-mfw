## 团队角色快速参考

| 角色 | 标识 | 职责简述 |
|------|------|----------|
| 项目经理 | `@pm` | 项目规划、任务分派、进度跟踪 |

| UI 设计师 | `@ui` | 界面设计、交互设计、UI 规范 |
| 技术主管 | `@tech-lead` | 架构设计、技术方案、代码审查 |

| 后端工程师 | `@backend` | 后端开发、API 设计、数据库 |
| 测试工程师 | `@qa` | 测试计划、测试用例、质量保障 |

## 常用指令

### 任务分派
```
@product [任务描述]
@ui [任务描述]
@backend [任务描述]
@frontend [任务描述]
@qa [任务描述]
@doc [任务描述]
```

### 通用指令
```
/plan - 制定项目计划
/status - 查看项目状态
/tasks - 查看任务列表
/review - 发起审查
/meeting [主题] - 发起会议
```

## 工作流程

1. 用户提出需求
2. 项目经理（Claude）分析并分派
3. 对应角色执行
4. 文档维护专员记录归档
5. 项目审查验收
6. 项目经理交付
