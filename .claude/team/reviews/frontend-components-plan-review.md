## 2. 高优先级问题汇总（必须修复）

### P-01：组件用户需求描述缺失
**提出者**: @product | **严重等级**: 🔴 高

**问题描述**: 文档仅列出组件功能和技术规格，缺少每个组件解决的用户痛点和使用场景描述。例如 `MfwUserPicker` 未说明用户在什么业务场景下使用、解决什么问题。

**改进建议**: 为每个 P0/P1 级组件补充"用户需求"字段：
```markdown
### MfwUserPicker 用户需求
- **目标用户**: 后台管理员、客服专员
- **使用场景**: 表单中需要选择用户/部门的场景（如任务分配、权限授予）
- **解决问题**: 避免手动输入用户 ID，提供可视化选择体验
```

---

### P-02：验收标准不可量化
**提出者**: @product, @qa | **严重等级**: 🔴 高

**问题描述**: "支持搜索 + 表格一体化"、"支持用户/部门选择"等验收标准无法客观衡量。

**改进建议**: 将验收标准改为可测试条目：
```markdown
### MfwUserPicker 验收标准
- [ ] 支持关键词搜索并返回结果（响应时间≤300ms）
- [ ] 支持单选/多选模式切换
- [ ] 支持按部门树筛选
- [ ] 选择后返回标准用户数据结构 { id, name, departmentId }
```

---

### P-03：组件分层边界模糊
**提出者**: @tech-lead | **严重等级**: 🔴 高

**问题描述**: `MfwPageScene` 与 `MfwTableList` 职责交叉，文档提到"集成"但分层关系不清晰。`business/` 目录定位模糊。

**改进建议**: 明确分层原则，建立组件依赖规则：
```
基础层（不可依赖其他组件）: display/, layout/
中间层（仅可依赖基础层）: form/, table/, picker/, editor/
高层（可依赖所有低层）: page/, business/, feedback/
```

---

### P-04：类型系统缺少 Slot 和 Expose 定义
**提出者**: @tech-lead, @doc | **严重等级**: 🔴 高

**问题描述**: 类型定义模板仅包含 Props 和 Emits，缺少 Slots 和 Exposed 接口定义。

**改进建议**: 补充完整类型定义模板：
```typescript
/** Slots 接口 */
export interface ComponentNameSlots {
  header?: (props: { title: string }) => VNode[];
  default?: () => VNode[];
}

/** 暴露实例接口 */
export interface ComponentNameInstance {
  refresh: () => Promise<void>;
  reset: () => void;
}
```

---

### P-05：CSS 变量体系不完整
**提出者**: @ui | **严重等级**: 🔴 高

**问题描述**: 缺少关键 Design Tokens（阴影、圆角、间距、字体、动效、状态色）。

**改进建议**: 补充完整设计令牌体系。

---

### P-06：可访问性（a11y）要求完全缺失
**提出者**: @ui | **严重等级**: 🔴 高

**问题描述**: 文档未提及键盘导航、焦点管理、ARIA 属性、屏幕阅读器兼容性。

**改进建议**: 在验收标准中增加：
```markdown
- [ ] 所有交互元素支持键盘 Tab 导航
- [ ] 弹窗组件支持 ESC 关闭和焦点陷阱
- [ ] 图标按钮提供 aria-label 文本
- [ ] 颜色对比度符合 WCAG 2.1 AA 标准
```

---

### P-07：边界条件测试场景缺失
**提出者**: @qa | **严重等级**: 🔴 高

**问题描述**: 各组件未定义边界条件测试场景（空数据、超大文件、10 万 + 行数据等）。

**改进建议**: 为每个组件补充"边界条件测试矩阵"。

---

### P-08：MfwPageScene 场景定义模糊
**提出者**: @product | **严重等级**: 🔴 高

**问题描述**: "搜索面板 + 表格联动"过于抽象，未明确支持哪些搜索类型、表格功能清单。

**改进建议**: 细化功能清单：
```markdown
### MfwPageScene 功能清单
- 搜索面板：支持 Input/Select/DatePicker/TreeSelect
- 表格功能：排序、筛选、分页、导出、自定义列
- 联动机制：搜索条件变化自动触发表格刷新
```

---

### P-09：组件目录结构与导出路径不一致
**提出者**: @frontend | **严重等级**: 🔴 高

**问题描述**: 计划文档定义的目录结构与实际项目结构不一致。

**改进建议**: 调整实际目录结构以匹配计划文档，或修正文档反映实际结构。

---

### P-10：优先级排序缺乏依据
**提出者**: @product | **严重等级**: 🔴 高

**问题描述**: P0/P1/P2 分类未说明判定标准，`MfwAuthButton` 列为 P2 但权限按钮通常是高频刚需。

**改进建议**: 补充优先级评估维度（使用频率、业务影响范围、不可替代性），建议重新评估 `MfwAuthButton` 优先级至 P1。

---

### P-11：缺少统一 API 文档模板
**提出者**: @doc | **严重等级**: 🔴 高

**问题描述**: 文档缺少完整的 API 文档模板结构（Props 表格、Events 表格、Slots 表格、使用示例）。

**改进建议**: 在文档中增加标准 API 文档模板章节。
