## 🚀 快速开始

### 方式一：分派任务给特定角色

```
@product 设计一个用户注册流程，包含邮箱验证
@ui 设计登录页面的界面
@backend 开发用户管理 API
@qa 编写权限模块的测试用例
@doc 整理本周的项目会议纪要
```

### 方式二：让项目经理协调

```
请安排团队完成用户注册功能
我们需要进行一次需求评审
帮我协调前端和后端对接口径
```

### 方式三：使用通用指令

```
/plan 制定项目计划
/status 查看项目状态
/tasks 查看任务列表
/review 发起审查
```

## 💡 最佳实践

### 任务分派

✅ **推荐**：
- 明确任务目标和预期成果
- 指定优先级和截止日期
- 提供必要的背景信息

❌ **避免**：
- 模糊的任务描述
- 同时分派多个不相关任务
- 缺少验收标准

### 会议效率

✅ **推荐**：
- 会前准备议程
- 会后输出纪要和待办
- 跟踪待办事项完成情况
