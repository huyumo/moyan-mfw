**永远不做不该做的事情。**

**Why:** 用户明确要求项目必须 1:1 还原原型图片，不允许添加任何原型外的功能、交互或元素。任何超出原型的需求都属于"不该做的事"。同时，必须严格遵守团队权限规则，没有写入权限时不擅自编辑文件。

**How to apply:** 
1. 需求分析时，只记录原型图片中实际存在的内容，不添加、不扩展、不优化
2. 发现原型中没有的功能需求时，先询问用户确认，不擅自决定
3. 没有编辑/写入权限时，只记录问题，等待有权限的角色执行修改
4. 严格遵循 Harness hooks 流程，不绕过任何检查
5. 每次行动前先问自己：这是该我做的吗？这符合原型吗？
