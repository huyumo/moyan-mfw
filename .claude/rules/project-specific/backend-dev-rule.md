## API 开发检查清单

### 接口设计
- [ ] 路径命名是否规范（RESTful）
- [ ] 请求/响应类型是否完整
- [ ] 错误码是否定义
- [ ] 是否有 API 文档（Swagger/注释）

### 数据库操作
- [ ] TypeORM Entity 类型是否完整
- [ ] 是否有事务处理（多表操作）
- [ ] 是否有软删除（如需要）
- [ ] 索引是否合理

### 安全检查
- [ ] 输入验证是否完整
- [ ] SQL 注入防护（使用 ORM）
- [ ] 认证/鉴权是否正确
- [ ] 敏感数据是否加密

---

## 禁止行为

| 禁止项 | 原因 | 替代方案 |
|--------|------|----------|
| 原生 SQL | SQL 注入风险 | 使用 TypeORM |
| 明文密码 | 安全风险 | bcrypt 加密 |
| 硬编码配置 | 维护困难 | 环境变量 |
| 未处理异常 | 服务崩溃 | try-catch + 错误码 |

---

## 开发完成验证

```bash
# 类型检查
pnpm run typecheck

# 单元测试
pnpm run test:unit

# 服务启动
pnpm run dev
```
