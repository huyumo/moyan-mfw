# 测试修复规则

> 状态：**实验验证** | 创建：2026-04-02

---

## 目的

规范 E2E 测试修复流程，确保 AI 先验证实际页面 DOM 再修改测试代码。

---

## 核心规则

### 执行前强制检查

**修复 E2E 测试失败时，必须执行以下步骤，按顺序执行：**

1. **读取测试规范**：`docs/03-框架规范/01-前端规范/09-测试规范.md`
   - 如未读取，停止修复，先读取

2. **运行失败测试**：获取具体错误信息
   ```bash
   pnpm run test:e2e -- --grep "测试名称"
   ```

3. **验证实际 DOM 结构**（关键步骤）：
   - 启动前端服务
   - 使用 Playwright codegen 查看实际选择器：
     ```bash
     npx playwright codegen http://localhost:5174/页面路径
     ```
   - 记录实际选择器

4. **对比并修复**：
   - 对比测试代码选择器与实际选择器
   - 使用稳定选择器（role、text、test-id）
   - 避免不稳定选择器（CSS class）

---

## 选择器优先级

| 优先级 | 选择器类型 | 示例 |
|--------|------------|------|
| 1 | Role + name | `getByRole('button', { name: '搜索' })` |
| 2 | Text | `getByText('用户管理')` |
| 3 | Test ID | `getByTestId('search-btn')` |
| 4 | CSS selector | `.search-form`（最后选择） |

---

## 禁止行为

- ❌ 不得假设页面结构
- ❌ 不得修改前端代码匹配测试
- ❌ 不得跳过测试
- ❌ 不得使用 `any` 类型断言

---

## 验证检查

修复完成后，必须执行：

- [ ] 测试规范已读取
- [ ] 实际 DOM 已验证（有 codegen 输出或 Grep 搜索结果）
- [ ] 测试已运行通过
- [ ] 选择器使用稳定类型

---

## 协作接口（新增）

### 测试编写前必须确认

| 确认项 | 确认方式 | 模板 |
|--------|----------|------|
| 页面 DOM 结构 | Grep 搜索组件源码 | `docs/02-团队/05-协作模板/页面结构验证报告.md` |
| API 可用性 | 运行 API 测试 | `docs/02-团队/05-协作模板/API变更通知单.md` |
| data-testid 存在 | 检查组件代码 | - |

### 发现问题时的处理流程

```
发现选择器不稳定 →
  1. 记录到页面结构验证报告
  2. 通知前端开发添加 data-testid
  3. 等待前端更新后再编写测试
```

## 示例

**错误修复流程**：
```
AI: "测试失败了，我修改选择器..."
→ 验证点：是否读取了测试规范？
→ 验证点：是否使用 codegen 验证实际 DOM？
→ 如未执行，强制停止
```

**正确修复流程**：
```
AI: "我来修复这个测试失败"
1. 读取 docs/03-框架规范/.../09-测试规范.md
2. 运行测试获取错误：Selector '.search-form' not found
3. 启动 codegen，发现实际选择器是 'form.el-form'
4. 更新测试代码使用 getByRole('form')
5. 运行测试验证通过
```
