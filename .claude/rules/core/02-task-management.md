## 目的

规范任务文件的格式和更新流程，确保任务状态清晰可追踪。

---

## TASK.md 格式规范

### 文件头（YAML Front Matter）

```yaml
---
task: 任务名称（必填）
status: pending | in_progress | blocked | completed（必填）
priority: P0 | P1 | P2 | P3（必填）
started: YYYY-MM-DD（必填）
updated: YYYY-MM-DD HH:mm（必填）
session: 会话 ID（必填）
lock: 时间戳（可选，用于多会话锁）
assignee: @角色（可选）
---
```

### 字段说明

| 字段 | 说明 | 可选值 |
|------|------|--------|
| task | 任务名称，一句话描述 | 任意字符串 |
| status | 当前状态 | pending, in_progress, blocked, completed |
| priority | 优先级 | P0, P1, P2, P3 |
| started | 任务开始日期 | YYYY-MM-DD |
| updated | 最后更新时间 | YYYY-MM-DD HH:mm |
| session | 最后更新的会话 ID | 会话 ID 字符串 |
| lock | 会话锁时间戳 | Unix 时间戳（秒） |
| assignee | 负责人 | @角色 |

## 必选章节

### 1. 当前目标

**说明**：一句话描述当前任务的核心目标

**格式**：
```markdown
## 当前目标
[一句话描述]
```

**示例**：
```markdown
## 当前目标
开发墨焱框架前端组件库，参考网约车 admin 项目组件，建立规范化组件体系。
```

---

### 2. 已完成

**说明**：已完成的任务项列表

**格式**：
```markdown
## 已完成
- [x] 任务 1
- [x] 任务 2
```

---

### 3. 进行中

**说明**：正在进行中的任务，每个任务包含状态、相关文件、阻塞点

**格式**：
```markdown
## 进行中
- [ ] 任务名称（优先级）
  - 相关文件：`路径`
  - 当前状态：描述
  - 阻塞点：无/具体阻塞
```

---

### 4. 待开始

**说明**：待开始的任务列表，标注优先级

**格式**：
```markdown
## 待开始
- [ ] 任务名称（P0/P1/P2/P3）
```

---

### 5. 相关文件

**说明**：与当前任务相关的文件路径列表

**格式**：
```markdown
## 相关文件
- `路径 1`
- `路径 2`
```

---

**说明**：任务执行过程中的重要决策记录

**格式**：
```markdown
## 关键决策
- 决策内容 + 原因
```

---

### 备注

**说明**：其他需要说明的信息

**格式**：
```markdown
## 备注
[其他信息]
```

## 任务状态定义

| 状态 | 说明 | 流转条件 |
|------|------|----------|
| pending | 待开始 | 任务已创建，等待执行 |
| in_progress | 进行中 | 任务正在执行 |
| blocked | 阻塞 | 遇到阻塞，等待解决 |
| completed | 已完成 | 任务完成并验收 |

**状态流转图**：
```
pending → in_progress → blocked → in_progress → completed
                     ↘--------------↗
```

---

## 优先级定义

| 优先级 | 说明 | 响应时间 | 示例 |
|--------|------|----------|------|
| P0 | 紧急且重要 | 立即处理 | 线上 Bug、核心功能 |
| P1 | 重要不紧急 | 24 小时内 | 关键功能、重要文档 |
| P2 | 紧急不重要 | 48 小时内 | 一般优化、文档更新 |
| P3 | 不紧急不重要 | 待排期 | 技术债务、体验优化 |

## 更新规则

### 会话开始
- 读取 TASK.md
- 向用户确认任务状态
- 如状态与实际不符，立即更新

### 会话中
- 任务完成时立即标记 `[x]`
- 遇到阻塞时更新 `blocked` 状态
- 重大决策时记录到"关键决策"

### 会话结束
- 必须更新 `updated` 和 `session` 字段
- 必须更新进度（已完成/进行中/待开始）
- 必须记录下一步行动

---

## 违规处理

| 违规项 | 处理动作 |
|--------|----------|
| TASK.md 不存在 | 停止工作，先创建 |
| 文件格式错误 | 停止工作，先修复 |
| 未更新 TASK.md | 会话结束前强制提醒 |
| 状态与实际不符 | 立即更新 |

## 任务文件位置

### 活跃任务
- 位置：`[project]/.claude/TASK.md`
- 特点：轻量级，快速更新
- 用途：会话开始读取，了解当前任务

### 任务详情
- 位置：`[project]/docs/04-项目实施/05-任务追踪/[编号]-[名称].md`
- 特点：重量级，完整记录
- 用途：任务详情、决策记录、历史追溯
