## 使用说明

### 方式 1：复制模板到分析文件

```bash
# 复制模板到输出目录
cp .claude/harness/templates/analysis-template.md .claude/harness/output/analysis-$(date +%Y%m%d-%H%M%S).md
```

### 方式 2：在 TASK.md 中添加分析章节

在 TASK.md 中添加：

```markdown
## 需求分析
[复制上述模板内容]
```

### 方式 3：直接在对话中输出

AI 在对话中直接输出分析结果，用户确认后再开始实现。

## Hook 检查规则

PreCode Hook 会检查以下必要章节是否存在：

- [ ] 需求理解（关键词：`需求理解`、`核心需求`、`用户需要`）
- [ ] 优缺点分析（关键词：`优点 `、` 缺点 `、`权衡`、` 利弊`）
- [ ] 边界条件（关键词：`边界`、` 限制 `、` 适用范围`）
- [ ] 实现思路（关键词：`实现`、` 方案 `、` 步骤`）
- [ ] 文件影响（关键词：`文件 `、` 编辑 `、` 创建`、` 涉及`）

缺少任何一项都会阻塞编码流程。
