# .claude 目录说明

本目录包含 Claude 协作相关的配置和文档，与项目本身的 `docs/` 目录分离。

---

## 目录结构

```
.claude/
├── README.md           # 本说明文件
├── team/               # 团队协作配置
│   ├── README.md       # 团队欢迎文档和快速开始
│   ├── team-charter.md # 团队章程（角色定义、协作流程）
│   ├── task-template.md# 任务管理模板
│   ├── meeting-template.md # 会议纪要模板
│   └── meetings/       # 会议纪要存档目录
└── plans/              # 项目计划文件
└── memory/             # 记忆文件（由系统管理）
```

---

## 团队协作

本项目采用虚拟团队协作模式，由 Claude 担任项目经理统一协调。

### 角色列表

| 角色 | 标识 | 职责 |
|------|------|------|
| 项目经理 | `@pm` | 统一接收和分派任务 |
| 产品经理 | `@product` | 需求分析、产品设计 |
| UI 设计师 | `@ui` | 界面设计 |
| 技术主管 | `@tech-lead` | 架构设计、技术评审 |
| 前端工程师 | `@frontend` | 前端开发 |
| 后端工程师 | `@backend` | 后端开发、API 设计 |
| 测试工程师 | `@qa` | 测试用例、质量保障 |
| 项目审查 | `@audit` | 文档/代码审查 |
| 文档维护专员 | `@doc` | 会议纪要、文档管理 |
| 项目维护 | `@maintainer` | 版本管理 |

### 快速开始

查看 [团队协作入门](./team/README.md)

### 使用示例

```
@product 设计一个用户注册流程
@ui 设计登录页面
@backend 开发用户 API
@doc 整理本周会议纪要
```

| 目录 | 用途 | 读者 |
|------|------|------|
