## 会话流程规则（强制执行）

### 会话开始时

**必须按顺序执行以下步骤，未完成前不得开始工作：**

```
1. 读取 .claude/TASK.md
       ↓
2. 向用户确认任务状态
   - "我看到当前任务是 [X]，状态是 [Y]，是否正确？"
   - 如有变化，先更新 TASK.md
       ↓
3. 确认当日/本次会话目标
   - "今天/这次会话计划完成什么？"
       ↓
4. 开始工作
```

**违规处理**：如未执行会话开始流程，用户应提醒"请先读取 TASK.md 并确认任务状态"

---

### 会话结束时

**必须按顺序执行以下步骤，未完成前不得结束会话：**

```
1. 总结本次会话完成的工作
   - 列出所有完成项
       ↓
2. 更新 .claude/TASK.md
   - 标记已完成任务
   - 更新进行中任务状态
   - 记录下一步行动
       ↓
3. 记忆保存检查
   - 如有重要决策，询问"是否需要保存到记忆？"
       ↓
4. 确认会话结束
```

**违规处理**：用户应在会话结束前提醒"请先更新 TASK.md"

### 代码规范（核心要求）

#### 组件命名
| 项目 | 规范 | 示例 |
|------|------|------|
| 组件名前缀 | PascalCase + Mfw | `MfwFormCard` |
| 组件目录 | camel-case + 连字符 | `mfw-form-card/` |
| 主组件文件 | PascalCase | `Index.vue` 或 `MfwFormCard.tsx` |
| 子组件文件 | kebab-case | `column-control.vue` |
| composable | camel-case + use | `use-draft-dox.ts` |
| 类型定义 | camel-case + types | `types.ts` |

#### 类型定义
- 完整定义所有 Props/Emits/Expose 类型
- 禁止使用 `any`（除非绝对必要且有注释说明）
- 复杂配置项必须有独立接口

#### 技术栈选择
- 基础组件库：TSX + `defineComponent`
- 业务组件：`<script setup lang="ts">`
- 先类型后实现

## 违规处理清单

| 违规场景 | 处理动作 |
|----------|----------|
| 未读 TASK.md 就开始工作 | 停止工作，先读取并确认任务 |
| TASK.md 不存在 | 停止工作，先创建任务文件 |
| 未更新 TASK.md 就想结束会话 | 强制提醒更新 |
| 违反组件命名规范 | 指出并要求修正 |
| 类型定义使用 any | 要求补充完整类型 |
| 未遵循会话流程 | 暂停工作，先补全流程 |

## 多会话锁机制

当同时打开多个 VSCode 窗口时：

```yaml
# .claude/TASK.md 头部包含
session: be2d86c6-3b77-43f3-bfba-6fadbea86dfb  # 当前会话 ID
lock: 1711784520  # 时间戳锁（5 分钟有效期）
```

- 写入前检查 `lock` 时间戳
- 如 5 分钟内有其他会话写入 → 警告用户"有其他会话正在工作"
- 写入时更新 `lock` 时间戳
