> 版本：1.0.0 | 最后更新：2026-03-30 | 生效：立即生效

---

## 核心规则（强制执行）

### 会话流程
- **会话开始**：读取 `TASK.md` → 确认任务状态 → 确认会话目标
- **任务执行**：方案确认后 → 更新任务状态 TASK.md → 开始执行
- **会话结束**：自我反思 → 质量检查 → 任务归档 → 更新任务状态 TASK.md → 总结

### 虚拟团队模式
具体工作专人专办，禁止跨职责。任务分派格式：`@角色 [任务描述]`

| @pm | @product | @ui | @tech-lead | @frontend | @backend | @qa | @audit | @doc | @maintainer |
|-----|----------|-----|------------|-----------|----------|-----|--------|------|-------------|
| 项目经理 | 产品 | UI | 技术 | 前端 | 后端 | 测试 | 审查 | 文档 | 维护 |

### 代码规范
- **组件命名**：PascalCase + Mfw 前缀（如 `MfwFormCard`），目录 camel-case + 连字符
- **类型定义**：禁止 `any`，完整定义 Props/Emits/Expose
- **技术栈**：基础组件 TSX + `defineComponent`，业务组件 `<script setup lang="ts">`

### 文档规范
- 活跃任务：`TASK.md`（轻量级，根目录）
- 任务详情：`docs/04-项目实施/05-任务追踪/YYYY-MM-DD-HHMM-任务名称.md`（重量级）

### 多会话锁机制
写入前检查 `lock` 时间戳，如 5 分钟内有其他会话写入 → 警告用户。

## 违规处理
| 违规 | 处理 |
|------|------|
| 未读 TASK.md | 停止，先读取确认 |
| TASK.md 不存在 | 停止，先创建 |
| 未更新 TASK.md 就执行 | 立即停止，先更新 |
| 未更新 TASK.md 就结束 | 强制提醒更新 |
| 违反命名规范 | 指出并修正 |
| 类型定义用 any | 要求补充完整类型 |
