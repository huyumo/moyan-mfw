# API 集成测试执行说明

> 创建时间：2026-04-01

## 前置条件

1. **确保后端服务正在运行**
   ```bash
   # 在 packages/base-backend 目录下
   npm run start:dev
   ```

2. **确认后端地址**: http://localhost:3000

3. **确认默认账号**:
   - 管理员：`admin` / `Admin@123`
   - 测试员：`test` / `Test@123`

## 执行测试

在项目根目录执行：

```bash
node test-api-runner.js
```

## 测试输出

测试执行后会输出：

1. **控制台输出**: 实时显示每个测试用例的执行状态
   - `[x]` - 通过
   - `[!]` - 失败/阻塞
   - `[-]` - 跳过

2. **生成的文件**:
   - `test-api-report.md` - 完整的测试报告（Markdown 格式）
   - `test-checklist-update.md` - 测试清单更新内容

## 故障排查

### 问题 1: 所有测试都失败
**原因**: 后端服务未启动
**解决**: 确保 `npm run start:dev` 正在运行

### 问题 2: 认证相关测试失败
**原因**: 默认账号不存在或密码错误
**解决**: 运行种子数据脚本 `npm run seed`

### 问题 3: 权限相关测试失败
**原因**: Token 权限不足
**解决**: 使用 admin 账号登录获取 Token

## 失败用例记录

如果有失败的测试用例，在清单文件的"附录 B：失败用例详情"表中记录：

| 字段 | 说明 |
|------|------|
| 用例编号 | 如 AUTH-001-03 |
| 用例名称 | 失败的场景描述 |
| 请求信息 | HTTP 方法、URL、请求体 |
| 预期响应 | 预期的状态码和响应 |
| 实际响应 | 实际返回的状态码和响应 |
| 错误日志 | 错误堆栈信息 |
| 严重程度 | 致命/严重/一般/轻微 |
| 复现步骤 | 详细的复现步骤 |
| 可能原因 | 初步分析的原因 |
| 解决状态 | 待解决/已修复 |
