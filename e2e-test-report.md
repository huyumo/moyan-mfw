### 1. APP-04: 创建应用 - 填写并提交

**文件**: `e2e/tests/app-management.spec.ts:44:3`

**错误类型**: `TimeoutError`

**错误信息**:
```
locator.fill: Timeout 8000ms exceeded.
Call log:
  - waiting for locator('[data-testid="app-name-input"] input')
```

**问题分析**:
- 测试尝试在创建应用弹窗中填写应用名称输入框
- 输入框定位器 `[data-testid="app-name-input"] input` 在 8 秒内未找到
- 可能原因：
  1. 弹窗未正确打开
  2. 输入框的 DOM 结构与预期不符（可能没有嵌套的 `input` 元素）
  3. `data-testid` 属性值不正确

**预防建议**:
- 检查前端组件的 `data-testid` 属性设置是否正确
- 确认输入框的实际 DOM 结构（是否需要 `.el-input__inner` 而不是 `input`）
- 增加弹窗打开后的等待时间

## 🐛 发现的 Bug

### Bug #1: 应用创建弹窗输入框定位失败

**严重程度**: 🔴 高

**影响范围**: 应用管理功能

**问题描述**:
创建应用弹窗中的输入框无法通过 `[data-testid="app-name-input"] input` 定位器找到，导致自动化测试失败。

**复现步骤**:
1. 登录系统
2. 进入应用管理页面 `/sys/app`
3. 点击"创建应用"按钮
4. 弹窗打开后，尝试定位应用名称输入框

**预期行为**: 输入框应能被正确定位并填写

**实际行为**: 定位器超时，找不到输入框元素

**根本原因分析**:
前端组件可能使用了 Element Plus 的输入框组件，实际 DOM 结构可能是：
```html
<div data-testid="app-name-input" class="el-input">
  <input class="el-input__inner" />
</div>
```
测试代码使用了 `[data-testid="app-name-input"] input`，但可能需要使用 `.el-input__inner` 或直接使用 `[data-testid="app-name-input"] .el-input__inner`

**修复建议**:
1. 检查前端 `AppManagement.vue` 组件中输入框的 `data-testid` 设置
2. 确认 Element Plus 输入框的实际渲染结构
3. 更新测试定位器为正确的选择器

### Bug #2: 应用类型配置数据缺失

**严重程度**: 🟡 中

**影响范围**: 应用创建流程

**问题描述**:
在创建应用时，应用类型下拉列表为空，导致多个工作流测试被跳过。

**复现步骤**:
1. 登录系统
2. 进入应用管理页面
3. 点击"创建应用"按钮
4. 点击应用类型下拉选择器
5. 下拉列表为空

**预期行为**: 下拉列表应显示可用的应用类型（如"供应商管理"、"订单管理"等）

**实际行为**: 下拉列表为空

**根本原因分析**:
1. 数据库种子数据可能未正确初始化应用类型
2. 应用类型同步机制可能存在问题
3. 测试环境数据库可能缺少必要的初始数据

**修复建议**:
1. 检查 `packages/base-backend/src/database/seeds/` 中的应用类型种子数据
2. 确认 `app-type-sync.ts` 同步逻辑是否正确执行
3. 在测试初始化时确保应用类型数据已同步
