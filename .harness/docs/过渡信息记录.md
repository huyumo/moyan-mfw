## 二、需要清理的目录/文件

### 2.1 嵌套目录问题

**问题目录**：
- `.harness/.harness/output/` - 嵌套的输出目录

**形成原因**：
- 之前 `settings.json` 中使用 `cd .harness && pnpm run hook:xxx`
- `findProjectRoot()` 在 `.harness` 目录下返回错误的项目根目录
- 相对路径 `.harness/output` 被解析为 `.harness/.harness/output`

**修复状态**：
- ✅ `settings.json` 已改为 `pnpm --filter @claude-code/harness`
- ✅ `paths.ts` 和 `session-start-hook.ts` 已增加嵌套检测
- ⚠️ 遗留的 `.harness/.harness` 目录需要清理

### 2.2 错误位置的日志文件

**问题文件**：
- `.harness/scripts/output/logs/hook-calls.log` - 错误位置的日志

**形成原因**：
- `run-hook.ts` 和 `hook-call-logger.ts` 使用 `__dirname` 计算路径
- 在某些执行环境下 `__dirname` 指向脚本所在目录而非项目根目录

**修复状态**：
- ✅ 脚本已修改为使用 `HARNESS_ROOT = path.resolve(__dirname, '..')`
- ⚠️ 遗留的错误日志文件需要清理

## 五、已知问题和注意事项

### 5.1 SessionStart Hooks

**注意**: SessionStart hooks 在会话开始时自动触发，但如果 `.claude/settings.json` 在会话开始后才更新，则不会触发。

**解决方案**: 确保在启动新会话前 `settings.json` 配置已完成。

### 5.2 路径解析

`paths.ts` 中的 `findProjectRoot()` 已增加嵌套目录检测：

```typescript
if (path.basename(currentDir) === '.harness') {
  const parentDir = path.dirname(currentDir);
  if (fs.existsSync(path.join(parentDir, '.harness', 'config', 'paths.json'))) {
    return parentDir;
  }
}
```
