**发现的配置路径不一致问题**:

| 组件 | 期望路径 | 实际路径 | 状态 |
|------|----------|----------|------|
| `docs-template-hook.ts` | `.claude/harness/config/docs.json` | - | ⚠️ 硬编码路径 |
| `paths.json` 定义 | `.harness/config/docs.json` | - | ✅ 正确配置 |
| 实际文件位置 | - | `.harness/config/docs.example.json` | ✅ 存在 |

**问题详情**:

`docs-template-hook.ts` 第 47-51 行使用硬编码路径:
```typescript
const configPaths = [
  path.join(projectRoot, '.claude', 'harness', 'config', 'docs.json'),
  path.join(projectRoot, '.claude', 'harness', 'config', 'docs.example.json')
];
```

而 `paths.json` 定义配置路径为:
```json
"docs": ".harness/config/docs.json"
```

**影响**:
- Hook 无法找到项目配置文件
- 回退使用通用模板（功能正常，但无法使用项目定制模板）

## 七、改进建议

### 7.1 短期修复

修改 `docs-template-hook.ts`，使用统一的路径配置系统:

```typescript
// 修改 loadConfig 函数
function loadConfig(projectRoot: string): { config: any | null; path: string | null } {
  const paths = loadPathsConfig(projectRoot);
  const configPaths = [
    paths.input.config.docs,  // 主配置路径
    paths.input.config.docs.replace('.json', '.example.json')  // 示例配置
  ];

  for (const configPath of configPaths) {
    if (fs.existsSync(configPath)) {
      try {
        const content = fs.readFileSync(configPath, 'utf-8');
        return { config: JSON.parse(content), path: configPath };
      } catch (e) {
        continue;
      }
    }
  }

  return { config: null, path: null };
}
```
