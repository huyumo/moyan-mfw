## 配置特点

### 1. 技术栈准确性 ✅

所有配置文件反映项目实际使用的技术栈，避免误导 Agent。

### 2. 规范完整性 ✅

覆盖以下维度：
- **安全规范**: SQL 注入防护、密码哈希、JWT 认证
- **API 规范**: RESTful 命名、统一错误格式、分页
- **数据库规范**: 事务、索引、迁移、软删除
- **日志规范**: 审计日志、敏感信息过滤
- **代码风格**: ESLint、Prettier、Vue script setup

### 3. Vue 特定规则 ✅

针对 Vue 3 项目特点添加：
- `<script setup>` 语法
- `defineProps<>()` 泛型定义
- `defineEmits<>()` 泛型定义
- 组件命名：PascalCase + Mfw 前缀
- CSS 类名：BEM 命名

### 4. NestJS 规范 ✅

强调 NestJS 最佳实践：
- 分层架构（Controller-Service-Repository）
- 依赖注入（Constructor Injection）
- 装饰器使用（@Module, @Controller, @Injectable）
- DTO 验证（class-validator）
- 实体继承 BaseEntity

### 5. Schema 验证 ✅

为每个配置文件提供 JSON Schema，确保格式正确。
