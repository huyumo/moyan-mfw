# `.harness/.harness` 嵌套目录问题分析

## 问题现象

在执行 harness hooks 后，会在 `.harness` 目录下创建嵌套的 `.harness/.harness` 目录。

## 根本原因

### 问题链

```
1. .claude/settings.json 执行 hooks 时切换目录
   → "cd .harness && pnpm run hook:xxx"
   
2. Hook 脚本中的 findProjectRoot() 以 process.cwd() 为起点
   → process.cwd() = .harness (已切换)
   
3. findProjectRoot() 查找失败后返回 process.cwd()
   → 没找到 TASK.md，返回 .harness
   
4. paths.json 的相对路径被错误解析
   → path.join('.harness', '.harness/output')
   → .harness/.harness/output ❌
```

### 问题代码

**1. `settings.json` hooks 配置（第 70、76 行）**
```json
"command": "cd .harness && pnpm run hook:identity"
```

**2. `paths.ts` 第 71-76 行**
```typescript
export function findProjectRoot(): string {
  let currentDir = process.cwd();  // ← 已经是 .harness
  const maxDepth = 5;
  let depth = 0;

  while (depth < maxDepth) {
    const pathsConfig = path.join(currentDir, '.harness', 'config', 'paths.json');
    if (fs.existsSync(pathsConfig)) {
      return currentDir;  // ← 返回 .harness
    }
    // ...
  }
  return process.cwd();  // ← 最终还是返回 .harness
}
```

**3. `paths.json` 配置**
```json
{
  "output": {
    "directory": ".harness/output"  // ← 相对路径
  }
}
```

当 `projectRoot = .harness` 时：
```typescript
path.join('.harness', '.harness/output') 
// = .harness/.harness/output ❌
```

## 修复方案

### 方案 A：修改 settings.json（推荐）

不使用 `cd .harness`，而是直接执行命令：

```json
"hooks": {
  "SessionStart": [
    {
      "matcher": "*",
      "hooks": [
        {
          "type": "command",
          "command": "pnpm --filter @claude-code/harness run hook:identity",
          "timeout": 30,
          "statusMessage": "🔔 运行 Harness 身份报告检查..."
        }
      ]
    }
  ]
}
```

**优点**：
- 不切换工作目录
- projectRoot 正确指向项目根目录

### 方案 B：修改 findProjectRoot() 逻辑

增加对嵌套目录的检测和避免：

```typescript
export function findProjectRoot(): string {
  let currentDir = process.cwd();
  
  // 如果当前目录就是 .harness，向上退一级
  if (path.basename(currentDir) === '.harness') {
    const parentDir = path.dirname(currentDir);
    if (fs.existsSync(path.join(parentDir, '.harness', 'config', 'paths.json'))) {
      return parentDir;  // 返回项目根目录
    }
  }
  
  // ... 原有逻辑
}
```

### 方案 C：paths.json 使用绝对路径占位符

```json
{
  "output": {
    "directory": "${PROJECT_ROOT}/.harness/output"
  }
}
```

然后在 `paths.ts` 中解析占位符。

## 本次修复

采用 **方案 A + 方案 B 组合**：

1. ✅ 修改 `settings.json` 不使用 `cd .harness`
2. ✅ 修改 `findProjectRoot()` 增加嵌套检测

## 验证方法

```bash
# 清理嵌套目录
rm -rf .harness/.harness

# 执行 hooks
cd .harness && pnpm run hook:identity

# 检查是否再次创建嵌套目录
ls -la .harness/.harness 2>&1  # 应该返回 "No such file or directory"
```
