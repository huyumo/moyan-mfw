## 修复内容

### P1 - 路径配置不一致问题

**问题描述**: `paths.json` 中定义的路径是 `.harness/config/xxx.json`，但多个 hooks 中使用的是 `.claude/harness/config/xxx.json`。

**修复的文件**:

| 文件 | 修复内容 | 状态 |
|------|----------|------|
| `.harness/utils/paths.ts` | 修复 `loadPathsConfig()` 和 `getDefaultPathsConfig()` 的路径 | ✅ |
| `.harness/hooks/docs-template-hook.ts` | 使用 `loadPathsConfig()` 获取路径 | ✅ |
| `.harness/hooks/backend-security-hook.ts` | 使用 `loadPathsConfig()` 获取路径 | ✅ |
| `.harness/hooks/frontend-guidelines-hook.ts` | 使用 `loadPathsConfig()` 获取路径 | ✅ |
| `.harness/hooks/architect-context-hook.ts` | 使用 `loadPathsConfig()` 获取路径 | ✅ |
| `.harness/hooks/review-checklist-hook.ts` | 使用 `loadPathsConfig()` 获取路径 | ✅ |
| `.harness/hooks/session-start-hook.ts` | 移除重复的 `findProjectRoot` | ✅ |
| `.harness/hooks/session-end-hook.ts` | 移除重复的 `findProjectRoot` | ✅ |
| `.harness/hooks/pm-context-hook.ts` | 移除重复的 `findProjectRoot` | ✅ |

### 修复后的代码模式

```typescript
import { findProjectRoot, loadPathsConfig } from '../utils/paths';

function loadConfig(projectRoot: string): { config: any | null; path: string | null } {
  const paths = loadPathsConfig(projectRoot);
  const configPath = paths.input.config.backend;  // 使用统一路径配置

  if (fs.existsSync(configPath)) {
    try {
      const content = fs.readFileSync(configPath, 'utf-8');
      return { config: JSON.parse(content), path: configPath };
    } catch (e) {
      // 继续尝试示例配置
    }
  }

  // 尝试示例配置
  const exampleConfigPath = configPath.replace('.json', '.example.json');
  if (fs.existsSync(exampleConfigPath)) {
    try {
      const content = fs.readFileSync(exampleConfigPath, 'utf-8');
      return { config: JSON.parse(content), path: exampleConfigPath };
    } catch (e) {
      // 继续
    }
  }

  return { config: null, path: null };
}
```
