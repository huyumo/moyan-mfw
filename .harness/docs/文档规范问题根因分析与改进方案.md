# Harness 配置问题根因分析与改进方案

**分析日期**: 2026-04-12  
**问题**: 文档不规范（存放位置错误、命名错误）  
**根因**: AI 助手绕过 harness 工作流，直接创建文档

---

## 一、问题根因

### 1.1 配置状态

| 配置项 | 状态 | 说明 |
|--------|------|------|
| `.claude/settings.json` hooks | ✅ 正确 | 包含完整的 SubagentStart hooks |
| `docs-architect` hook | ✅ 正确 | `hook:docs-template` 已配置 |
| `PreToolUse` hook | ⚠️ 不足 | 只检查编码，不检查文档规范 |

### 1.2 问题流程

```
用户请求：组织团队摸底测试
       ↓
AI 助手（我）直接执行  ←── 问题在此：没有调用 DOC-001
       ↓
直接使用 Write 工具创建文档
       ↓
文档没有遵循规范（位置、命名）
```

### 1.3 为什么 harness 没有阻止？

| Hook | 触发条件 | 实际触发 |
|------|----------|----------|
| `SubagentStart: docs-architect` | 调用 `Agent(subagent_type="docs-architect")` | ❌ 未触发（我没有调用 Agent） |
| `PreToolUse: Write|Edit` | 调用 `Write`或`Edit` | ✅ 触发，但没有检查文档规范 |

**结论**: harness 配置正确，但**检查范围不足**——没有强制"创建文档前必须调用 DOC-001"

---

## 二、彻底解决方案

### 方案 A：增强 PreToolUse Hook（推荐）

**目的**: 在写入 `.md` 文件前，检查是否遵循了文档规范

**检查逻辑**:
1. 如果写入的是 `docs/` 目录下的 `.md` 文件
2. 检查当前会话是否调用过 `docs-architect` 智能体
3. 如果没有，返回错误并拒绝写入

**实现位置**: `.harness/hooks/pre-code-hook.ts` 或新建 `.harness/hooks/docs-norm-hook.ts`

```typescript
// 伪代码示例
if (filePath.startsWith('docs/') && filePath.endsWith('.md')) {
  const sessionLog = readSessionLog();
  const hasCalledDocsArchitect = sessionLog.some(
    call => call.tool === 'Agent' && call.subagent_type === 'docs-architect'
  );
  
  if (!hasCalledDocsArchitect) {
    return {
      passed: false,
      message: '❌ 创建文档前必须先调用 DOC-001 (docs-architect) 进行规划',
      suggestion: '请使用 Agent(subagent_type="docs-architect") 进行文档规划'
    };
  }
}
```

### 方案 B：增强 CLAUDE.md 规则（立即生效）

在 `CLAUDE.md` 中添加强制性规则：

```markdown
## 文档创建规范（强制执行）

⛔ 创建任何 `docs/` 目录下的 `.md` 文件前，必须先调用 DOC-001 智能体：

```
Agent({
  subagent_type: "docs-architect",
  prompt: "规划文档结构和存放位置..."
})
```

**违规后果**: 代码将被拒绝提交，必须重新执行规范流程
```

### 方案 C：添加 SessionStart Hook 提醒（辅助）

在 `session-start-hook.ts` 中添加提醒：

```typescript
if (currentBranch.startsWith('docs/')) {
  console.log('⚠️ 文档分支提醒：创建文档前请先调用 DOC-001 智能体');
}
```
