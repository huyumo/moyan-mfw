# 需求分析强制检查 Hook

## 目的

确保 AI 在实现任何需求前，都进行充分的思考和分析。

---

## 检查要素

每次用户提出需求时，AI 必须分析以下内容：

### 1. 需求理解（必需）
- [ ] 用户的核心需求是什么
- [ ] 期望的最终结果

### 2. 优缺点分析（必需）
- [ ] 方案的优点
- [ ] 方案的缺点/权衡

### 3. 边界条件（必需）
- [ ] 能做什么
- [ ] 不能做什么
- [ ] 例外情况

### 4. 实现思路（必需）
- [ ] 技术选型
- [ ] 实现步骤
- [ ] 替代方案

### 5. 文件影响（必需）
- [ ] 会编辑哪些文件
- [ ] 会创建哪些文件
- [ ] 会删除哪些文件

### 6. 风险评估（推荐）
- [ ] 潜在风险
- [ ] 回滚方案

---

## Hook 检查逻辑

```
用户提出需求
    │
    ▼
AI 输出分析（本文件检查的内容）
    │
    ▼
PreCode Hook 检查
    │
    ├── ✅ 有需求理解 → 通过
    ├── ✅ 有优缺点分析 → 通过
    ├── ✅ 有边界条件 → 通过
    ├── ✅ 有实现思路 → 通过
    ├── ✅ 有文件列表 → 通过
    └── 全部通过 → 开始编码
```

## 分析输出位置

### 方式 1：输出到分析文件

`.harness/output/analysis-[timestamp].md`

### 方式 2：输出到 TASK.md

在 TASK.md 中添加章节：
```markdown
## 需求分析
### 优缺点分析
...
### 边界条件
...
### 实现思路
...
### 文件影响
...
```

### 方式 3：直接在对话中输出

AI 在开始实现前，先在对话中输出分析结果

## Hook 检查实现

### 检查关键词

```typescript
const requiredSections = [
  // 需求理解
  '需求理解', '核心需求', '用户需要',
  
  // 优缺点分析
  '优缺点', '优点', '缺点', '权衡',
  
  // 边界条件
  '边界条件', '限制', '适用范围', '不适用范围',
  
  // 实现思路
  '实现思路', '技术方案', '实现步骤',
  
  // 文件影响
  '文件影响', '会编辑', '会创建', '修改文件'
];
```

### 检查逻辑

```typescript
// 检查分析文档中是否包含必要章节
const hasSection = (content: string, keywords: string[]) => {
  return keywords.some(keyword => 
    content.toLowerCase().includes(keyword.toLowerCase())
  );
};

// 缺少任何必要章节都阻塞
if (!hasRequirementAnalysis) {

```

## 配置选项

在 `config.json` 中配置：

```json
{
  "hooks": {
    "pre-code": {
      "enabled": true,
      "requireAnalysis": true,
      "requiredSections": [
        "需求理解",
        "优缺点分析",
        "边界条件",
        "实现思路",
        "文件影响"
      ],
      "optionalSections": [
        "风险评估",
        "替代方案"
      ],
      "onFailure": "block"
    }
  }
}
```

## 使用示例

### 用户说：
> 帮我添加一个用户登录功能

### AI 应该先输出：

```markdown
## 需求理解
用户需要在系统中添加登录功能，包括用户名密码验证、会话管理。

## 优缺点分析
### 优点
- 提供用户认证机制
- 保护系统安全

### 缺点
- 需要额外的数据库表
- 增加代码复杂度

## 边界条件
### 支持的
- 用户名密码登录
- 会话保持

### 不支持的
- 第三方登录（如需要可扩展）
- 双因素认证

## 实现思路
1. 创建用户表
2. 实现密码加密
3. 添加登录接口
4. 实现会话管理

## 文件影响
### 会编辑
- src/routes/auth.ts
- src/middleware/auth.ts

### 会创建
- src/types/user.ts
- src/db/migrations/add-users.sql
```

### 然后 PreCode Hook 检查通过 → 开始编码
