# 分页查询规范

> 状态：**已生效** | 最后更新：2026-04-19 | 版本：v2.0

---

## 概述

本规范统一项目中所有分页查询接口的编写方式，基于原生 SQL 实现分页查询，通过 `PaginationX` + `WhereBuilder` 工具类实现参数化查询，防止 SQL 注入。

**核心原则**：
1. **DRY 原则**：不重复定义分页参数和返回格式
2. **继承复用**：通过继承 `PaginationQueryDto` 获得分页能力
3. **原生 SQL**：使用 `PaginationX` 执行原生 SQL 分页查询
4. **参数化查询**：使用 `WhereBuilder` 构建参数化 WHERE 条件，防止 SQL 注入
5. **统一返回**：使用 `PaginationResult<T>` 统一分页响应格式

## 核心 API

### 1. PaginationQueryDto（分页参数 DTO）

**位置**：`packages/base-backend/src/common/utils/pagination-x.util.ts`

**功能**：通用的分页请求参数，所有分页查询 DTO 应继承此类

**属性**：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | `number` | `1` | 当前页码（从 1 开始） |
| `pageSize` | `number` | `10` | 每页数量 |
| `sortField` | `string` | - | 排序字段（可选） |
| `sortOrder` | `'ASC' \| 'DESC'` | - | 排序方向（可选） |

### 2. PaginationResult<T>（分页结果类）

**位置**：`packages/base-backend/src/common/utils/pagination-x.util.ts`

**功能**：通用的分页响应结果

**属性**：

| 属性 | 类型 | 说明 |
|------|------|------|

| `page` | `number` | 当前页码 |

### 3. PaginationX（分页处理器）

**位置**：`packages/base-backend/src/common/utils/pagination-x.util.ts`

**功能**：基于原生 SQL 的分页查询处理器

**构造函数**：

```typescript
constructor(dataSource: DataSource, query: T, noTotal?: boolean)
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `dataSource` | `DataSource` | TypeORM 数据源 |
| `query` | `T extends PagerReq` | 分页请求参数 |
| `noTotal` | `boolean` | 是否不统计总数（默认 false） |

**链式方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `where(name, builder)` | `name: string`, `builder: WhereBuilder` | `this` | 设置 WHERE 条件构建器 |
| `sql(fn)` | `(params) => string` | `this` | 设置 SQL 生成函数 |
| `select(sql)` | `string` | `this` | 设置 SELECT 字段 |
| `defaultOrderBy(order)` | `string` | `this` | 设置默认排序 |
| `printSql()` | - | `this` | 打印 SQL（调试用） |
| `getData()` | - | `Promise<PaginationResult<T>>` | 执行查询并返回结果 |

### 4. WhereBuilder（WHERE 条件构建器）

**位置**：`packages/base-backend/src/common/utils/sql.util.ts`

**功能**：构建参数化 WHERE 条件，自动过滤 undefined 值

**链式方法**：

| 方法 | 参数 | 说明 |
|------|------|------|
| `eq(field, value)` | `field: string`, `value?: any` | 等于条件 |
| `neq(field, value)` | `field: string`, `value?: any` | 不等于条件 |
| `gt(field, value)` | `field: string`, `value?: any` | 大于条件 |
| `gte(field, value)` | `field: string`, `value?: any` | 大于等于条件 |
| `lt(field, value)` | `field: string`, `value?: any` | 小于条件 |
| `lte(field, value)` | `field: string`, `value?: any` | 小于等于条件 |
| `like(field, value)` | `field: string`, `value?: string` | 模糊查询（%value%） |
| `likeLeft(field, value)` | `field: string`, `value?: string` | 左模糊（value%） |
| `likeRight(field, value)` | `field: string`, `value?: string` | 右模糊（%value） |
| `in(field, values)` | `field: string`, `values?: any[]` | IN 条件 |
| `notIn(field, values)` | `field: string`, `values?: any[]` | NOT IN 条件 |
| `between(field, start, end)` | `field: string`, `start?: any`, `end?: any` | BETWEEN 条件 |
| `isNull(field)` | `field: string` | IS NULL 条件 |
| `isNotNull(field)` | `field: string` | IS NOT NULL 条件 |
| `group(builder, op)` | `builder: WhereBuilder`, `op?: 'AND'\|'OR'` | 条件分组（括号） |
| `addParam(name, value)` | `name: string`, `value: any` | 手动添加参数 |

**build() 方法**：

```typescript
build(): { where: string; params: Record<string, any> }
```

返回 WHERE 子句和参数对象。

### 场景 2：Service 层 - 基础分页查询

```typescript
import { PaginationX, WhereBuilder, PaginationResult } from '../../../common';

async findAll(query: QueryUserDto): Promise<PaginationResult<any>> {
  const { username, phone, userStatus } = query;

  const whereBuilder = new WhereBuilder();
  whereBuilder
    .like('user.username', username)
    .like('user.phone', phone)
    .eq('user.userStatus', userStatus);

  const pager = new PaginationX(this.dataSource, query);
  return await pager
    .where('main', whereBuilder)
    .sql(({ select, wheres, orderBy, limit }) => {
      const whereClause = wheres?.main || '';
      return `SELECT ${select} FROM sys_users user ${whereClause} ${orderBy} ${limit}`;
    })
    .select('user.*')
    .defaultOrderBy('user.createdAt DESC')
    .getData();
}
```

**说明**：
- `whereBuilder` 自动过滤 undefined 值，无需手动判断
- `sql()` 函数接收 `{ select, wheres, orderBy, limit }` 参数
- `getData()` 直接返回 `PaginationResult`，无需手动构造

### 场景 3：复杂查询（多表 JOIN）

```typescript
async findAll(query: QueryAppDto): Promise<PaginationResult<any>> {
  const { appName, appCode, appTypeId, ownerId, appStatus } = query;

  const whereBuilder = new WhereBuilder();
  whereBuilder
    .like('app.appName', appName)
    .like('app.appCode', appCode)
    .eq('app.appTypeId', appTypeId)
    .eq('app.ownerId', ownerId)
    .eq('app.appStatus', appStatus);

  const pager = new PaginationX(this.dataSource, query);
  return await pager
    .where('main', whereBuilder)
    .sql(({ select, wheres, orderBy, limit }) => {
      const whereClause = wheres?.main || '';
      return `
        SELECT ${select} FROM sys_apps app
        LEFT JOIN sys_app_types at ON app.appTypeId = at.id
        ${whereClause}
        ${orderBy}
        ${limit}
      `;
    })
    .select('app.*, at.typeName')
    .defaultOrderBy('app.sortOrder ASC, app.createdAt DESC')
    .getData();
}
```

### 场景 4：复杂 WHERE 条件（条件分组）

**场景**：`(role.isBuiltin = 1 AND role.appId IS NULL) OR role.appId = :appId`

```typescript
async findAll(query: QueryRoleDto): Promise<PaginationResult<any>> {
  const { roleCode, roleName, roleStatus, appTypeId, appId } = query;

  // 构建子条件组 1：(role.isBuiltin = 1 AND role.appId IS NULL)
  const subBuilder1 = new WhereBuilder();
  subBuilder1
    .eq('role.isBuiltin', 1)
    .isNull('role.appId');

  // 构建子条件组 2：role.appId = :appId
  const subBuilder2 = new WhereBuilder();
  subBuilder2.eq('role.appId', appId);

  // 构建外层条件组：(subBuilder1) OR (subBuilder2)
  const outerBuilder = new WhereBuilder();
  outerBuilder
    .group(subBuilder1)
    .group(subBuilder2, 'OR');

  // 最终 WHERE 条件
  const whereBuilder = new WhereBuilder();
  whereBuilder
    .like('role.roleCode', roleCode)
    .like('role.roleName', roleName)
    .eq('role.roleStatus', roleStatus)
    .eq('role.appTypeId', appTypeId)
    .group(outerBuilder);

  const pager = new PaginationX(this.dataSource, query);
  return await pager
    .where('main', whereBuilder)
    .sql(({ select, wheres, orderBy, limit }) => {
      const whereClause = wheres?.main || '';
      return `SELECT ${select} FROM sys_roles role ${whereClause} ${orderBy} ${limit}`;
    })
    .select('role.*')
    .defaultOrderBy('role.sortOrder ASC, role.createdAt DESC')
    .getData();
}
```

**生成的 SQL**：

```sql
WHERE role.roleCode LIKE :roleCode_0 
  AND role.roleName LIKE :roleName_1 
  AND role.roleStatus = :roleStatus_2 
  AND role.appTypeId = :appTypeId_3 
  AND ((role.isBuiltin = :isBuiltin_4 AND role.appId IS NULL) OR role.appId = :appId_5)
ORDER BY role.sortOrder ASC, role.createdAt DESC
LIMIT 10 OFFSET 0
```

### 场景 5：子查询

```typescript
async getMembers(appId: string, query: QueryMemberDto): Promise<PaginationResult<any>> {
  const { nickname, username } = query;
  const app = await this.appRepository.findOne({ where: { id: appId } });
  if (!app) {
    throw new NotFoundException('应用不存在');
  }

  const whereBuilder = new WhereBuilder();
  whereBuilder
    .eq('am.appId', appId)
    .like('u.nickname', nickname)
    .like('u.username', username)
    .addParam('appId', appId)
    .addParam('appTypeId', app.appTypeId);

  const pager = new PaginationX(this.dataSource, query);
  return await pager
    .where('main', whereBuilder)
    .sql(({ select, wheres, orderBy, limit }) => {
      const whereClause = wheres?.main || '';
      return `
        SELECT ${select} FROM (
          SELECT 
            am.id,
            am.userId,
            am.appId,
            am.createdAt,
            u.nickname,
            u.username,
            JSON_ARRAYAGG(
              JSON_OBJECT(
                'roleCode', r.roleCode,
                'roleName', r.roleName
              )
            ) as roles
          FROM sys_app_members am
          INNER JOIN sys_users u ON u.id = am.userId
          LEFT JOIN sys_user_roles ur ON ur.userId = am.userId 
            AND (ur.roleId IN (
              SELECT r2.id FROM sys_roles r2 
              WHERE (r2.appId = :appId OR r2.appTypeId = :appTypeId) AND r2.isOwner = 0
            ))
          LEFT JOIN sys_roles r ON ur.roleId = r.id
          ${whereClause}
          GROUP BY am.userId
        ) AS sub
        ${orderBy}
        ${limit}
      `;
    })
    .select('*')
    .defaultOrderBy('createdAt DESC')
    .getData();
}
```

## WhereBuilder 操作符对照表

| 方法 | 操作符 | SQL 示例 | 说明 |
|------|--------|----------|------|
| `eq()` | = | `field = :value` | 等于 |
| `neq()` | != | `field != :value` | 不等于 |
| `gt()` | > | `field > :value` | 大于 |
| `gte()` | >= | `field >= :value` | 大于等于 |
| `lt()` | < | `field < :value` | 小于 |
| `lte()` | <= | `field <= :value` | 小于等于 |
| `like()` | LIKE | `field LIKE '%value%'` | 模糊查询 |
| `likeLeft()` | LIKE | `field LIKE 'value%'` | 左模糊 |
| `likeRight()` | LIKE | `field LIKE '%value'` | 右模糊 |
| `in()` | IN | `field IN (:v1, :v2)` | IN 条件 |
| `notIn()` | NOT IN | `field NOT IN (:v1, :v2)` | NOT IN |
| `between()` | BETWEEN | `field BETWEEN :start AND :end` | 范围查询 |
| `isNull()` | IS NULL | `field IS NULL` | 空值判断 |
| `isNotNull()` | IS NOT NULL | `field IS NOT NULL` | 非空判断 |
| `group()` | () | `(condition1 OR condition2)` | 条件分组 |

## 自动过滤 undefined

`WhereBuilder` 所有方法自动过滤 undefined 值：

```typescript
const { username, phone, status } = query; // 可能都是 undefined

const whereBuilder = new WhereBuilder();
whereBuilder
  .like('user.username', username)   // 如果 undefined，不添加条件
  .like('user.phone', phone)         // 如果 undefined，不添加条件
  .eq('user.status', status);        // 如果 undefined，不添加条件

// 最终生成的 WHERE 只包含有值的条件
```

## 常见问题

### Q1: 为什么使用原生 SQL 而不是 TypeORM QueryBuilder？

**A**: 原生 SQL 更灵活，支持复杂查询（如子查询、JSON 函数），且 `WhereBuilder` 提供参数化查询防止 SQL 注入。

### Q2: 如何调试 SQL？

**A**: 使用 `printSql()` 方法：

```typescript
const pager = new PaginationX(this.dataSource, query);
return await pager
  .where('main', whereBuilder)
  .sql(...)
  .printSql()  // 打印 SQL 到控制台
  .getData();
```

### Q3: 如何自定义排序？

**A**: 使用 `defaultOrderBy()` 或在 DTO 中设置 `sortField` 和 `sortOrder`：

```typescript
pager.defaultOrderBy('user.createdAt DESC, user.id ASC');
```

### Q4: 如何处理复杂条件？

**A**: 使用 `group()` 方法构建嵌套条件组，参考场景 4。

### Q5: 返回类型为什么是 `PaginationResult<any>`？

**A**: 因为原生 SQL 返回的是原始数据，类型推断需要根据实际查询结果确定。如需强类型，可在 Controller 层使用 DTO 转换。
