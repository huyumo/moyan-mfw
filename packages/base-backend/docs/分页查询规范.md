# 分页查询规范

> 状态：**已生效** | 最后更新：2026-03-31 | 版本：v1.1

---

## 概述

本规范统一项目中所有分页查询接口的编写方式，通过复用通用的分页工具类，减少代码重复，提高可维护性。

**核心原则**：
1. **DRY 原则**：不重复定义分页参数和返回格式
2. **继承复用**：通过继承 `PaginationQueryDto` 获得分页能力
3. **工具封装**：使用 `PaginationHelper` 封装 TypeORM 分页查询
4. **统一返回**：使用 `PaginationResult<T>` 统一分页响应格式
5. **简洁优先**：使用 `@ApiPaginatedResponse` 装饰器简化 Swagger 文档

---

## 核心 API

### 1. PaginationQueryDto（分页参数 DTO）

**位置**：`packages/base-backend/src/common/utils/pagination.util.ts`

**功能**：通用的分页请求参数，所有分页查询 DTO 应继承此类

**属性**：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | `number` | `1` | 当前页码（从 1 开始） |
| `pageSize` | `number` | `10` | 每页数量（最大 100） |
| `sortField` | `string` | `'created_at'` | 排序字段 |
| `sortOrder` | `'ASC' \| 'DESC'` | `'DESC'` | 排序方向 |

**方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getSkip()` | - | `number` | TypeORM 的 skip 参数（自动计算） |
| `getTake()` | - | `number` | TypeORM 的 take 参数（等于 pageSize） |
| `buildQueryOptions<T>(where?)` | `where?: FindOptionsWhere<T>` | `FindManyOptions<T>` | 构建 TypeORM 查询选项 |

### 2. PaginationResult<T>（分页结果类）

**位置**：`packages/base-backend/src/common/utils/pagination.util.ts`

**功能**：通用的分页响应结果

**属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `list` | `T[]` | 数据列表 |
| `total` | `number` | 总数量 |
| `page` | `number` | 当前页码 |
| `pageSize` | `number` | 每页数量 |
| `totalPages` | `number` | 总页数 |
| `hasNext` | `boolean` | 是否有下一页 |
| `hasPrev` | `boolean` | 是否有上一页 |

**静态方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `fromQuery(items, total, pageQuery)` | `items: T[]`, `total: number`, `pageQuery: PaginationQueryDto` | `PaginationResult<T>` | 从查询结果创建分页对象 |

### 3. PaginationHelper（分页辅助工具）

**位置**：`packages/base-backend/src/common/utils/pagination.util.ts`

**功能**：提供静态方法封装 TypeORM 分页查询

**方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `applyPagination(qb, pageQuery)` | `qb: SelectQueryBuilder<T>`, `pageQuery: PaginationQueryDto` | `SelectQueryBuilder<T>` | 应用分页到 QueryBuilder |
| `applyOrder(qb, pageQuery, alias?)` | `qb: SelectQueryBuilder<T>`, `pageQuery: PaginationQueryDto`, `alias?: string` | `SelectQueryBuilder<T>` | 应用排序到 QueryBuilder |
| `executeQuery(qb, pageQuery)` | `qb: SelectQueryBuilder<T>`, `pageQuery: PaginationQueryDto` | `Promise<PaginationResult<T>>` | 执行分页查询并返回结果 |

### 4. QueryBuilderHelper（查询条件构建工具）

**位置**：`packages/base-backend/src/common/utils/query-builder.util.ts`

**功能**：提供静态方法构建 TypeORM 查询条件

**方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `applyConditions(qb, conditions)` | `qb: SelectQueryBuilder<T>`, `conditions: QueryCondition[]` | `SelectQueryBuilder<T>` | 应用多个查询条件 |
| `applyAdvancedConditions(qb, group)` | `qb: SelectQueryBuilder<T>`, `group: ConditionGroup` | `SelectQueryBuilder<T>` | 应用嵌套条件组（复杂查询） |

**快捷方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `eq(field, value)` | `field: string`, `value?: any` | `QueryCondition` | 等于条件 |
| `ne(field, value)` | `field: string`, `value?: any` | `QueryCondition` | 不等于条件 |
| `gt(field, value)` | `field: string`, `value?: number` | `QueryCondition` | 大于条件 |
| `lt(field, value)` | `field: string`, `value?: number` | `QueryCondition` | 小于条件 |
| `gte(field, value)` | `field: string`, `value?: number` | `QueryCondition` | 大于等于条件 |
| `lte(field, value)` | `field: string`, `value?: number` | `QueryCondition` | 小于等于条件 |
| `like(field, value)` | `field: string`, `value?: string` | `QueryCondition` | 模糊查询（%value%） |
| `likeLeft(field, value)` | `field: string`, `value?: string` | `QueryCondition` | 左模糊（value%） |
| `likeRight(field, value)` | `field: string`, `value?: string` | `QueryCondition` | 右模糊（%value） |
| `in(field, value)` | `field: string`, `value?: any[]` | `QueryCondition` | IN 条件 |
| `notIn(field, value)` | `field: string`, `value?: any[]` | `QueryCondition` | NOT IN 条件 |
| `between(field, start, end)` | `field: string`, `start?: any`, `end?: any` | `QueryCondition` | BETWEEN 条件 |
| `isNull(field)` | `field: string` | `QueryCondition` | IS NULL 条件 |
| `isNotNull(field)` | `field: string` | `QueryCondition` | IS NOT NULL 条件 |
| `or(condition)` | `condition: QueryCondition` | `QueryCondition` | OR 条件包装器 |

## 使用指南

### 场景 1：创建分页查询 DTO

**❌ 错误做法**（重复定义分页参数）：

```typescript
import { ApiProperty } from '@nestjs/swagger';
import { IsOptional, IsInt, Min } from 'class-validator';
import { Type } from 'class-transformer';

export class QueryUserDto {
  @ApiProperty({ description: '用户名', required: false })
  @IsOptional()
  @IsString()
  username?: string;

  // ❌ 不应该重复定义分页参数
  @ApiProperty({ description: '当前页码', default: 1, required: false })
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  page?: number = 1;

  @ApiProperty({ description: '每页数量', default: 10, required: false })
  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  pageSize?: number = 10;
}
```

**✅ 正确做法**（继承 PaginationQueryDto）：

```typescript
import { ApiProperty } from '@nestjs/swagger';
import { IsOptional, IsString } from 'class-validator';
import { PaginationQueryDto } from '../../../../common';

export class QueryUserDto extends PaginationQueryDto {
  @ApiProperty({ description: '用户名', required: false })
  @IsOptional()
  @IsString()
  username?: string;

  @ApiProperty({ description: '邮箱', required: false })
  @IsOptional()
  @IsString()
  email?: string;
}
```

### 场景 4：Service 层 - Repository 模式

**❌ 错误做法**（手动构造分页返回）：

```typescript
async findAll(query: QueryUserDto) {
  const { page = 1, pageSize = 10, username } = query;

  const [list, total] = await this.userRepository.findAndCount({
    where: username ? { username: Like(`%${username}%`) } : {},
    skip: (page - 1) * pageSize, // ❌ 手动计算
    take: pageSize,
  });

  // ❌ 手动构造返回对象
  return {
    list,
    total,
    page,
    pageSize,
    totalPages: Math.ceil(total / pageSize),
  };
}
```

**✅ 正确做法**（使用 PaginationHelper）：

```typescript
import { PaginationHelper, PaginationResult } from '../../../../common';

async findAll(query: QueryUserDto): Promise<PaginationResult<User>> {
  const { username } = query;

  const [list, total] = await this.userRepository.findAndCount({
    where: username ? { username: Like(`%${username}%`) } : {},
    ...query.buildQueryOptions(), // ✅ 使用 DTO 方法
  });

  return PaginationResult.fromQuery(list, total, query);
}
```

### 场景 5：Service 层 - QueryBuilder 模式（推荐）

**❌ 错误做法**（手动写 skip/take + 冗长的 if 判断）：

```typescript
async findAll(query: QueryUserDto) {
  const { page, pageSize, username, email, phone, status, deptId } = query;

  const qb = this.userRepository.createQueryBuilder('user');

  // ❌ 如果有 10 个查询条件，这里会有 10 个 if，代码冗长
  if (username) {
    qb.andWhere('user.username LIKE :username', { username: `%${username}%` });
  }
  if (email) {
    qb.andWhere('user.email LIKE :email', { email: `%${email}%` });
  }
  // ... 更多 if

  const [list, total] = await qb
    .skip((page - 1) * pageSize)
    .take(pageSize)
    .getManyAndCount();

  return { /* 手动构造返回 */ };
}
```

**✅ 正确做法**（使用 QueryBuilderHelper + PaginationHelper）：

```typescript
import { QueryBuilderHelper, PaginationHelper } from '../../../../common';

async findAll(query: QueryUserDto): Promise<PaginationResult<User>> {
  const qb = this.userRepository.createQueryBuilder('user');

  // ✅ 使用 QueryBuilderHelper，一行搞定所有条件（支持 10+ 条件）
  QueryBuilderHelper.applyConditions(qb, [
    { field: 'user.username', value: query.username, operator: 'like' },
    { field: 'user.email', value: query.email, operator: 'like' },
    { field: 'user.phone', value: query.phone, operator: 'like' },
    { field: 'user.status', value: query.status, operator: '=' },
    { field: 'user.deptId', value: query.deptId, operator: '=' },
    // 添加更多条件...
  ]);

  // ✅ 使用 PaginationHelper 执行分页查询
  return PaginationHelper.executeQuery(
    qb.orderBy('user.createdAt', 'DESC'),
    query,
  );
}
```

**对比优势**：

| 方式 | 代码行数 | 可读性 | 可维护性 |
|------|----------|--------|----------|
| 传统 if 判断 | 20+ 行 | 差（条件多时） | 差 |
| QueryBuilderHelper | 5 行 | 优 | 优 |

### 场景 6：使用快捷方法（更简洁）

```typescript
import { QueryBuilderHelper as QB } from '../../../../common';

async findAll(query: QueryUserDto) {
  const qb = this.userRepository.createQueryBuilder('user');

  // ✅ 使用快捷方法，代码更简洁
  QB.applyConditions(qb, [
    QB.like('user.username', query.username),
    QB.like('user.email', query.email),
    QB.eq('user.status', query.status),
    QB.in('user.roleId', query.roleIds),
  ]);

  return PaginationHelper.executeQuery(qb, query);
}
```

**QueryBuilderHelper 支持的操作符**：

| 方法 | 操作符 | SQL | 示例 |
|------|--------|-----|------|
| `eq()` | = | `field = :value` | `QB.eq('user.status', 1)` |
| `ne()` | != | `field != :value` | `QB.ne('user.deleted', 1)` |
| `gt()` | > | `field > :value` | `QB.gt('user.age', 18)` |
| `gte()` | >= | `field >= :value` | `QB.gte('user.score', 60)` |
| `lt()` | < | `field < :value` | `QB.lt('user.score', 100)` |
| `lte()` | <= | `field <= :value` | `QB.lte('user.age', 60)` |
| `like()` | LIKE | `field LIKE %:value%` | `QB.like('user.username', '张三')` |
| `likeLeft()` | LIKE | `field LIKE :value%` | `QB.likeLeft('user.username', '张')` |
| `likeRight()` | LIKE | `field LIKE %:value` | `QB.likeRight('user.username', '三')` |
| `in()` | IN | `field IN (:...value)` | `QB.in('user.roleId', [1,2,3])` |
| `notIn()` | NOT IN | `field NOT IN (:...value)` | `QB.notIn('user.roleId', [4,5])` |
| `between()` | BETWEEN | `field BETWEEN :start AND :end` | `QB.between('user.createdAt', '2026-01-01', '2026-12-31')` |
| `isNull()` | IS NULL | `field IS NULL` | `QB.isNull('user.deletedAt')` |
| `isNotNull()` | IS NOT NULL | `field IS NOT NULL` | `QB.isNotNull('user.phone')` |
| `or()` | OR | `OR condition` | `QB.or(QB.eq('user.role', 'admin'))` |

**OR 条件示例**：

```typescript
QueryBuilderHelper.applyConditions(qb, [
  QB.eq('user.status', 1),
  QB.or(QB.eq('user.role', 'admin')), // OR user.role = 'admin'
]);
```

### 场景 7：复杂查询（嵌套条件组）

**场景**：`(status=1 OR status=2) AND (username LIKE '%张%' OR nickName LIKE '%张%')`

```typescript
import { QueryBuilderHelper as QB, type ConditionGroup } from '../../../../common';

async findAll(query: QueryUserDto) {
  const qb = this.userRepository.createQueryBuilder('user');

  // ✅ 使用 applyAdvancedConditions 处理嵌套条件
  QB.applyAdvancedConditions(qb, {
    logicalOperator: 'AND',
    conditions: [
      {
        // (status=1 OR status=2)
        logicalOperator: 'OR',
        conditions: [
          QB.eq('user.status', 1),
          QB.eq('user.status', 2),
        ],
      },
      {
        // (username LIKE '%张%' OR nickName LIKE '%张%')
        logicalOperator: 'OR',
        conditions: [
          QB.like('user.username', query.keyword),
          QB.like('user.nickName', query.keyword),
        ],
      },
    ],
  });

  return PaginationHelper.executeQuery(qb, query);
}
```

**生成的 SQL**：

```sql
WHERE (user.status = 1 OR user.status = 2)
  AND (user.username LIKE '%张%' OR user.nickName LIKE '%张%')
```

**三层嵌套示例**：

```typescript
// 场景：(status=1 AND role='admin') OR (status=2 AND vip=true)
QB.applyAdvancedConditions(qb, {
  logicalOperator: 'OR',
  conditions: [
    {
      logicalOperator: 'AND',
      conditions: [
        QB.eq('user.status', 1),
        QB.eq('user.role', 'admin'),
      ],
    },
    {
      logicalOperator: 'AND',
      conditions: [
        QB.eq('user.status', 2),
        QB.eq('user.vip', true),
      ],
    },
  ],
});
```

**混合使用示例**（简单条件 + 嵌套组）：

```typescript

  QB.eq('user.deleted', false),

  logicalOperator: 'AND',

    {
      logicalOperator: 'OR',
      conditions: [
        {
          logicalOperator: 'AND',

            QB.eq('user.status', 1),
```

### Service

```typescript
// src/modules/sys/user/user.service.ts
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, Like } from 'typeorm';
import { User } from './entities/user.entity';
import { QueryUserDto } from './dto/req/query-user.dto';
import { PaginationHelper, PaginationResult } from '../../../common';

@Injectable()
export class UserService {
  constructor(
    @InjectRepository(User)
    private userRepository: Repository<User>,
  ) {}

  async findAll(query: QueryUserDto): Promise<PaginationResult<User>> {
    const { username, email, phone } = query;

    const qb = this.userRepository.createQueryBuilder('user');

    if (username) {
      qb.andWhere('user.username LIKE :username', { username: `%${username}%` });
    }

    if (email) {
      qb.andWhere('user.email LIKE :email', { email: `%${email}%` });
    }

    if (phone) {
      qb.andWhere('user.phone LIKE :phone', { phone: `%${phone}%` });
    }

    return PaginationHelper.executeQuery(
      qb.orderBy('user.createdAt', 'DESC'),
      query,
    );
  }
}
```

## 常见问题

### Q1: 为什么 Swagger 没有显示分页参数？

**A**: `PaginationQueryDto` 的字段都有 `@ApiProperty` 装饰器，继承后会自动显示。如果没有显示，检查：
1. 是否正确继承了 `PaginationQueryDto`
2. 是否安装了 `@nestjs/swagger`
3. 是否正确使用了 `class-validator` 和 `class-transformer`

### Q2: 如何自定义分页参数默认值？

**A**: 在继承的 DTO 中重写默认值：

```typescript
export class QueryUserDto extends PaginationQueryDto {
  // 重写默认值
  page?: number = 1;
  pageSize?: number = 20; // 改为 20
}
```

### Q3: 如何实现多字段排序？

**A**: 暂时不支持。如需要多字段排序，可在 Service 层手动处理：

```typescript
async findAll(query: QueryUserDto) {
  const qb = this.userRepository.createQueryBuilder('user');
  // ... 条件查询

  // 手动添加排序
  qb.orderBy('user.sortOrder', 'ASC')
    .addOrderBy('user.createdAt', 'DESC');

  return PaginationHelper.executeQuery(qb, query);
}
```

### Q4: 如何添加额外的返回字段？

**A**: 继承 `PaginationResult<T>` 并添加字段：

```typescript
export class UserPaginationResult extends PaginationResult<User> {
  @ApiProperty({ description: '活跃用户数' })
  activeCount: number;

  constructor(data: User[], total: number, page: number, pageSize: number, activeCount: number) {
    super(data, total, page, pageSize);
    this.activeCount = activeCount;
  }
}
```

### Q5: @ApiPaginatedResponse 的原理是什么？

**A**: `@ApiPaginatedResponse` 是一个装饰器组合函数，内部使用 `applyDecorators` 将 `@ApiResponse` 的复杂 schema 配置封装起来。它接受一个 DTO 类作为参数，自动生成该类型的分页响应 Schema。
