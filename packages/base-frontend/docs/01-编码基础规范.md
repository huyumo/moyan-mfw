## 1.1 命名规范

### 1.1.1 文件命名

| 类型 | 规则 | 示例 | 等级 |
|------|------|------|------|
| 组件文件 | PascalCase | `UserCard.vue`, `OrderList.vue` | 🔴 |
| 页面文件 | PascalCase + Page | `UserListPage.vue` | 🔴 |
| 组合式函数 | camelCase + use | `useUserList.ts`, `useOrderLogic.ts` | 🔴 |
| 工具函数 | camelCase | `utils.ts`, `formatters.ts` | 🔴 |
| API 文件 | camelCase + api | `user-api.ts`, `order-api.ts` | 🟢 |
| 类型定义 | camelCase + types | `user-types.ts` | 🟢 |
| 样式文件 | kebab-case | `base-styles.scss` | 🟢 |

### 1.1.2 目录命名

```
✅ 推荐：
business/views/          # 业务视图
business/components/     # 业务组件
business/composables/    # 组合式函数

❌ 避免：
business/Views/          # 不要大写
business/view/           # 不要单数（除非语义需要）
```

### 1.1.3 代码命名

```typescript
// 变量 - camelCase
const userName = 'test';
const isLoading = ref(false);

// 常量 - UPPER_SNAKE_CASE
const MAX_RETRY_COUNT = 3;
const API_TIMEOUT = 10000;

// 组件 - PascalCase
defineOptions({ name: 'UserListCard' });

// 函数 - camelCase
const handleSubmit = () => {};
const fetchData = async () => {};

// 类/接口 - PascalCase
interface UserInfo { ... }
class DataProcessor { ... }

// 枚举 - PascalCase + 成员 UPPER_SNAKE_CASE
enum OrderStatus {
  PENDING = 'pending',
  COMPLETED = 'completed',
}
```

### 1.1.4 Props/Emits 命名

```typescript
// Props - camelCase
interface Props {
  userData?: UserInfo;
  isLoading?: boolean;
}

// Emits - 动词开头，camelCase
const emit = defineEmits<{
  updateModelValue: [value: string];
  itemClick: [id: number];
  confirm: [data: any];
}>();
```

### 1.2.3 页面级目录结构与路由配置

```
# 路由和菜单自动生成机制
# 框架使用 import.meta.glob 扫描 views/**/index.{ts,tsx} 配置文件
# 自动生成路由和菜单，无需手动配置路由表

# 配置文件格式 (index.ts)
export default {
  page: () => import('./Index.vue'),  // 或静态导入
  path: 'orders',                     // 路由路径
  name: '订单中心',                    // 页面名称/菜单名称
  icon: 'Tickets',                    // 菜单图标（Element Plus 图标）
  auth: true,                         // 是否需要认证
  order: 1,                           // 菜单排序（数字越小越靠前）
  hidden: false,                      // 是否隐藏菜单
};

# 示例：带子路由的页面
business/
├── Index.vue              # 父级页面（可选）
├── index.ts               # 父级配置（可选）
├── orders/
│   ├── Index.vue          # 子页面组件
│   └── index.ts           # 子页面配置
└── reports/
    ├── Index.vue
    └── index.ts

# 示例：简单页面
dashboard/
├── Index.vue              # 仪表盘页面
└── index.ts               # 配置文件

# 完整目录结构示例
views/
├── dashboard/
│   ├── Index.vue
│   └── index.ts           # 首页（order: 1）
├── business/
│   ├── orders/
│   │   ├── Index.vue
│   │   └── index.ts       # 订单中心
│   └── reports/
│       ├── Index.vue
│       └── index.ts       # 报表中心
├── monitor/
│   └── overview/
│       ├── Index.vue
│       └── index.ts       # 运行概览
├── system/
│   ├── user/
│   │   ├── Index.vue
│   │   └── index.ts       # 用户管理
│   └── role/
│       ├── Index.vue
│       └── index.ts       # 角色管理
├── login/
│   ├── Index.vue
│   └── index.ts           # 登录页（hidden: true, auth: false）
└── not-found/
    ├── Index.vue
    └── index.ts           # 404 页（hidden: true, auth: false）

# 路由生成规则：
# 1. path: 自动生成路由路径（支持嵌套）
# 2. name: 生成路由名称和菜单标签
# 3. icon: 菜单图标
# 4. order: 菜单排序
# 5. auth: 路由守卫（需要认证）
# 6. hidden: 是否在侧边栏隐藏
```

**规范要求：**
- 每个页面必须使用独立目录包裹
- 页面组件必须命名为 `Index.vue`（首字母大写）
- 每个页面目录必须包含 `index.ts` 配置文件
- 配置文件必须导出 `page`、`path`、`name` 属性
- 路由和菜单从配置文件自动生成，无需手动维护路由表
- 登录页、404、403 等系统页面需设置 `hidden: true`

## 1.4 导入导出规范

### 1.4.1 导入顺序

```typescript
// 1. Vue 核心
import { ref, computed } from 'vue';

// 2. 第三方库
import axios from 'axios';
import { ElMessage } from 'element-plus';

// 3. 基础框架
import { useLayoutStore } from 'moyan-mfw-base-frontend';

// 4. 项目内部 - 绝对路径
import type { UserInfo } from '@/types/user-types';

// 5. 项目内部 - 相对路径
import { useUserList } from './composables/use-user-list';
import UserCard from './components/UserCard.vue';
```

```typescript
// ✅ 推荐：统一导出
// index.ts
export * from './use-user-list';

export type * from './user-types';

// ❌ 避免：分散导出
```

## 1.5 代码格式化规范

### 1.5.1 Prettier 配置

项目使用 Prettier 进行代码格式化，配置文件位于根目录 `prettier.config.cjs`：

```javascript
module.exports = {
  printWidth: 120,        // 单行最大字符数
  singleQuote: true,      // 使用单引号
  trailingComma: 'all',   // 尾随逗号
  semi: true,             // 语句结尾分号
  arrowParens: 'always',  // 箭头函数参数括号
};
```

### 1.5.2 ESLint 配置

项目使用 ESLint 进行代码检查，配置文件位于根目录 `eslint.config.mjs`：

关键规则：
- `moyan/comment-compliance`: 强制注释规范
- `vue/multi-word-component-names`: 关闭（允许单组件名）
- `max-lines`: 文件最大行数 1000（类型文件 200）

### 1.5.3 格式化命令

```bash
# 检查代码格式
pnpm run format:check

# 自动格式化
pnpm run format

# ESLint 检查
pnpm run lint

# ESLint 自动修复
pnpm run lint:fix
```
