# 路由配置指南

> 状态：**已完成** | 版本：1.0.0

## 概述

本框架采用**约定优于配置**的原则，通过自动扫描页面配置文件生成路由和菜单，无需手动维护复杂的路由表。

## 快速开始

### 1. 创建页面目录

在 `views/` 目录下创建新的页面目录：

```bash
views/
  my-page/           # 新建页面目录
    ├── Index.vue    # 页面组件（必须）
    └── index.ts     # 配置文件（必须）
```

### 2. 编写配置文件

在 `index.ts` 中导出页面配置：

```typescript
import MyPage from './Index.vue';

export default {
  page: MyPage,
  path: 'my-page',
  name: '我的页面',
  icon: 'User',
  auth: true,
  order: 10,
};
```

### 3. 完成

框架会自动：
- 生成路由配置
- 注册到路由系统
- 生成侧边栏菜单项
- 设置路由守卫

## 配置文件属性

### 必需属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `page` | `Component` | Vue 页面组件，支持动态导入 |
| `path` | `string` | 路由路径（相对于父级） |
| `name` | `string` | 页面名称，用于菜单显示和路由命名 |

### 可选属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `icon` | `string` | - | Element Plus 图标名称 |
| `auth` | `boolean` | `true` | 是否需要认证 |
| `order` | `number` | `50` | 菜单排序，数字越小越靠前 |
| `hidden` | `boolean` | `false` | 是否在侧边栏隐藏 |
| `children` | `PageConfig[]` | - | 子页面配置 |

## 使用示例

### 基础页面

```typescript
// views/dashboard/index.ts
import Dashboard from './Index.vue';

export default {
  page: Dashboard,
  path: 'dashboard',
  name: '首页',
  icon: 'DataBoard',
  auth: true,
  order: 1,
};
```

### 带子路由的页面

```typescript
// views/business/index.ts
import Business from './Index.vue';

export default {
  page: Business,
  path: 'business',
  name: '业务中心',
  icon: 'Briefcase',
  auth: true,
  order: 10,
  children: [
    {
      page: () => import('./orders/Index.vue'),
      path: 'orders',
      name: '订单管理',
      icon: 'Tickets',
      auth: true,
    },
    {
      page: () => import('./reports/Index.vue'),
      path: 'reports',
      name: '报表中心',
      icon: 'PieChart',
      auth: true,
    },
  ],
};
```

### 动态导入组件

```typescript
// views/system/user/index.ts
export default {
  page: () => import('./Index.vue'),
  path: 'user',
  name: '用户管理',
  icon: 'User',
  auth: true,
};
```

### 隐藏页面（不显示在菜单）

```typescript
// views/profile/edit/index.ts
export default {
  page: EditProfile,
  path: 'edit',
  name: '编辑资料',
  auth: true,
  hidden: true,  // 不显示在侧边栏
};
```

## 路由生成规则

### 路径生成

- 嵌套目录自动生成嵌套路由
- `path` 属性相对于父级路由
- 根路径自动添加 `/`

```
views/
  business/
    orders/
      index.ts  →  /business/orders
```

### 路由名称生成

- 使用 `name` 属性作为路由名称
- 嵌套路由名称自动添加父级前缀

### 路由守卫

- `auth: true` 的路由需要认证令牌
- 未认证用户自动重定向到登录页
- 登录成功后自动跳转回原目标页

## 菜单生成规则

### 菜单树构建

框架会自动从路由配置生成菜单树：

1. 扫描所有 `views/**/index.{ts,tsx}` 文件
2. 读取 `export default` 配置
3. 根据 `path` 构建层级关系
4. 按 `order` 排序

### 菜单显示

| 条件 | 结果 |
|------|------|
| `hidden: true` | 不显示在菜单 |
| 无 `children` 且无 `path` | 不显示 |
| 有 `children` | 显示为分组/子菜单 |

### 图标

使用 Element Plus 图标：

```typescript
icon: 'DataBoard'      // 📊
icon: 'User'           // 👤
icon: 'Setting'        // ⚙️
icon: 'Briefcase'      // 💼
```

## 高级用法

### 页面级组件复用

```typescript
// views/system/user/index.ts
import UserPage from './Index.vue';

export default {
  page: UserPage,
  path: 'user',
  name: '用户管理',
  icon: 'User',
  auth: true,
};

// views/system/role/index.ts
// 复用同一个页面组件，不同的数据源
export default {
  page: UserPage,  // 复用
  path: 'role',
  name: '角色管理',
  icon: 'UserFilled',
  auth: true,
};
```

### 登录页特殊处理

登录页通常不需要认证，且应该隐藏在菜单外：

```typescript
// views/login/index.ts
import LoginPage from './Index.vue';

export default {
  page: LoginPage,
  path: 'login',
  name: '登录',
  auth: false,      // 不需要认证
  hidden: true,     // 不显示在菜单
};
```

### 404 页面

```typescript
// views/not-found/index.ts
import NotFoundPage from './Index.vue';

export default {
  page: NotFoundPage,
  path: '404',
  name: '页面不存在',
  auth: false,
  hidden: true,
};
```

## 注意事项

### 1. 配置文件必须

每个页面目录必须包含 `index.ts` 或 `index.tsx` 配置文件，否则：
- 路由不会注册
- 菜单不会生成
- 访问该路径会跳转到 404

### 2. 路径唯一性

确保 `path` 在全局范围内唯一，避免路由冲突。

### 3. 组件命名

- 页面组件使用 `Index.vue`（首字母大写）
- 目录内的子组件使用 PascalCase 命名

### 4. 认证令牌

- 认证令牌存储在 `localStorage` 中
- 键名为 `mfw:admin:token`
- 路由守卫自动检查

## 目录结构示例

完整的目录结构参考：

```
views/
├── dashboard/
│   ├── Index.vue
│   └── index.ts         # 首页
├── business/
│   ├── Index.vue
│   ├── index.ts         # 业务中心（父级）
│   ├── orders/
│   │   ├── Index.vue
│   │   └── index.ts     # 订单管理
│   └── reports/
│       ├── Index.vue
│       └── index.ts     # 报表中心
├── system/
│   ├── Index.vue
│   ├── index.ts         # 系统管理（父级）
│   ├── user/
│   │   ├── Index.vue
│   │   └── index.ts     # 用户管理
│   └── role/
│       ├── Index.vue
│       └── index.ts     # 角色管理
├── login/
│   ├── Index.vue
│   └── index.ts         # 登录页（隐藏）
└── not-found/
    ├── Index.vue
    └── index.ts         # 404 页（隐藏）
```
