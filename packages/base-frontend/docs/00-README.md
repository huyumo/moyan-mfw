## 与其他文档的关系

```
moyan-mfw/
├── docs/                      # 产品设计文档（只读）
│   ├── core/                  # 核心概念、架构
│   ├── database/              # 数据库设计
│   └── api/                   # API 高层设计
│
├── packages/
│   ├── base-frontend/docs/    # 前端开发规范（本文档）
│   └── base-backend/docs/     # 后端开发规范
│
└── dev/                       # 项目实施文档
    ├── 02-前端开发文档.md     # 业务前端开发指南
    └── 03-后端开发文档.md     # 业务后端开发指南
```

**区分原则**：
- `docs/` - 产品设计，描述"是什么"和"为什么"
- `packages/*/docs/` - 框架规范，描述"怎么写"和"写什么"
- `dev/` - 项目实施，描述"做哪个"和"何时做"
