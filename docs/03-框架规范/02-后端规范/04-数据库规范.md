## 4.1 实体设计规范

### 4.1.1 实体命名规范

| 类型 | 命名规则 | 示例 | 等级 |
|------|----------|------|------|
| 表名 | 小写 + 下划线 + 复数 | `sys_users`, `sys_roles` | 🔴 |
| 主键 | `id` | `id` | 🔴 |
| 外键 | `关联表名单数_id` | `user_id`, `role_id` | 🔴 |
| 字段名 | 小写 + 下划线 | `user_name`, `created_at` | 🔴 |
| 索引 | `idx_表名_字段名` | `idx_users_username` | 🟢 |
| 唯一索引 | `uk_表名_字段名` | `uk_users_username` | 🟢 |
| 外键约束 | `fk_表名_关联表名` | `fk_users_roles` | 🟢 |

### 分表分库命名规范

**水平分表命名（按数据量分表）：**

```
格式：{原表名}_{分表标识}

示例：
- sys_users_0000, sys_users_0001, ... sys_users_9999    # 按 ID 尾号分表（10000 个表）
- sys_orders_2024, sys_orders_2025, ...                 # 按年份分表
- sys_logs_01, sys_logs_02, ... sys_logs_12             # 按月份分表
```

**垂直分表命名（按字段拆分）：**

```
格式：{原表名}_{扩展标识}

示例：
- sys_users_base      # 用户基础信息表（id, username, password 等核心字段）
- sys_users_extend    # 用户扩展信息表（user_id, preferences, settings 等）
- sys_users_profile   # 用户 Profile 表（user_id, avatar, bio, social_links 等）
```

**分库命名（按业务/功能拆分）：**

```
格式：{业务前缀}_db

示例：
- user_db, order_db, product_db      # 按业务分库
- mfw_core_db, mfw_log_db            # 按功能分库
```

| 类型 | 命名规则 | 示例 | 等级 |
|------|----------|------|------|
| 水平分表 | `{原表名}_{序号/日期}` | `sys_users_0000`, `sys_orders_2024` | 🟢 |
| 垂直分表 | `{原表名}_{base/extend/profile}` | `sys_users_base`, `sys_users_extend` | 🟢 |
| 分库 | `{业务前缀}_db` | `user_db`, `order_db` | 🟢 |

**🔴 强制：**
- 分表规则必须在迁移文件中明确定义
- 分表键 (Sharding Key) 必须在 ER 图中标注
- 禁止在代码中硬编码分表表名，必须使用路由中间件（如 TypeORM Sharding）

### 4.1.2 通用字段规范

> **🔴 强制：TypeORM 注解规范详见本章节下方说明**

所有业务表必须包含以下审计字段：

```sql
-- 创建时间（自动填充）
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

-- 更新时间（自动更新）
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

-- 创建人 ID
created_by BIGINT COMMENT '创建人 ID'

-- 更新人 ID
updated_by BIGINT COMMENT '更新人 ID'

-- 逻辑删除标记（可选）
deleted_at TIMESTAMP NULL COMMENT '逻辑删除时间'
```

### 4.1.3 实体类 TypeORM 注解规范

**🔴 强制：`@Column({})` 中必须编写 `comment` 和 `type`，`comment` 必须详细描述字段作用、用途等细节**

```typescript
// 文件：user.entity.ts
@Entity('sys_users')
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({
    type: 'varchar',
    length: 50,
    comment: '用户昵称 - 用于前台展示的用户名称，支持中文、字母、数字，长度 2-50 字符'
  })
  nickname: string;

  @Column({
    type: 'varchar',
    length: 100,
    comment: '登录账号 - 用户登录系统的唯一标识，全局唯一，支持邮箱/手机号/自定义用户名'
  })
  username: string;

  @Column({
    type: 'varchar',
    length: 255,
    comment: '密码哈希 - 使用 bcrypt 加密存储，盐值 rounds=10'
  })
  passwordHash: string;

  @Column({
    type: 'tinyint',
    default: 1,
    comment: '状态 - 1:启用 0:禁用 - 控制用户是否可登录系统'
  })
  status: number;

  @Column({
    type: 'timestamp',
    default: () => 'CURRENT_TIMESTAMP',
    comment: '创建时间 - 记录创建时的时间戳，首次插入时自动设置'
  })
  createdAt: Date;

  @Column({
    type: 'timestamp',
    default: () => 'CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP',
    comment: '更新时间 - 记录最后修改时的时间戳，每次更新自动刷新'
  })
  updatedAt: Date;
}
```

**comment 编写规范：**

| 要求 | 说明 | 示例 |
|------|------|------|
| 格式 | `字段名称 - 用途说明 - 约束/规则` | `'用户昵称 - 用于前台展示 - 长度 2-50 字符'` |
| 必填 | 所有字段必须填写 comment | 不允许空白或省略 |
| 详细 | 描述字段用途、约束、默认值等 | 避免简单如'用户 ID' |
| 枚举值 | 有枚举值时必须列出 | `'状态 - 1:启用 0:禁用 2:锁定'` |

```typescript
// ❌ 避免：comment 过于简单
@Column({ type: 'varchar', comment: '用户昵称' })
nickname: string;

// ✅ 推荐：comment 详细完整
@Column({
  type: 'varchar',
  length: 50,
  comment: '用户昵称 - 用于前台展示的用户名称，支持中文、字母、数字，长度 2-50 字符'
})
nickname: string;
```

### 4.1.4 实体类定义

```typescript
// entity/User.entity.ts
import {
  Entity,
  PrimaryGeneratedColumn,
  Column,
  CreateDateColumn,
  UpdateDateColumn,
  DeleteDateColumn,
  ManyToMany,
  JoinTable,
} from 'typeorm';

@Entity('sys_users')
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ name: 'username', length: 50, unique: true })
  username: string;

  @Column({ name: 'password_hash', length: 255 })
  passwordHash: string;

  @Column({ name: 'nickname', length: 50, nullable: true })
  nickname?: string;

  @Column({ name: 'email', length: 100, nullable: true })
  email?: string;

  @Column({ name: 'phone', length: 20, nullable: true })
  phone?: string;

  @Column({ name: 'status', type: 'tinyint', default: 1 })
  status: number; // 1-正常，0-禁用

  @CreateDateColumn({ name: 'created_at' })
  createdAt: Date;

  @UpdateDateColumn({ name: 'updated_at' })
  updatedAt: Date;

  @DeleteDateColumn({ name: 'deleted_at' })
  deletedAt?: Date;

  @Column({ name: 'created_by', nullable: true })
  createdBy?: string;

  @Column({ name: 'updated_by', nullable: true })
  updatedBy?: string;

  @ManyToMany(() => Role)
  @JoinTable({
    name: 'sys_user_roles',
    joinColumn: { name: 'user_id', referencedColumnName: 'id' },
    inverseJoinColumn: { name: 'role_id', referencedColumnName: 'id' },
  })
  roles: Role[];
}
```

### 关系实体注解规范

**🔴 强制：关系注解必须明确连接字段**

| 关系类型 | 注解组合 | 拥有方 | 说明 |
|----------|----------|--------|------|
| 一对一 | `@OneToOne` + `@JoinColumn` | 外键拥有方 | 必须指定 `joinColumn` |
| 一对多 | `@OneToMany` + `@JoinColumn` | "一"方 | 外键在"多"方表中 |
| 多对一 | `@ManyToOne` + `@JoinColumn` | "多"方 | 外键在"多"方表中 |
| 多对多 | `@ManyToMany` + `@JoinTable` | 双方 | 必须指定中间表 |

**一对一示例：**
```typescript
// User 和 UserProfile 一对一
@OneToOne(() => UserProfile, (profile) => profile.user)
@JoinColumn({ name: 'user_id', referencedColumnName: 'id' })
profile?: UserProfile;
```

**一对多示例：**
```typescript
// Dept 和 User 一对多
@OneToMany(() => User, (user) => user.dept)
users: User[];

// User 端（多对一）
@ManyToOne(() => Dept, (dept) => dept.users)
@JoinColumn({ name: 'dept_id', referencedColumnName: 'id' })
dept?: Dept;
```

**多对多示例：**
```typescript
// User 和 Role 多对多
@ManyToMany(() => Role)
@JoinTable({
  name: 'sys_user_roles',
  joinColumn: { name: 'user_id', referencedColumnName: 'id' },
  inverseJoinColumn: { name: 'role_id', referencedColumnName: 'id' },
})
roles: Role[];
```

**🟢 推荐：配置级联操作和延迟加载**

```typescript
// 级联操作：持久化 User 时自动保存 Profile
@OneToOne(() => UserProfile, { cascade: ['insert', 'update', 'remove'] })
@JoinColumn()
profile?: UserProfile;

// 延迟加载：访问时才查询关联数据
@OneToMany(() => Order, (order) => order.user, { lazy: true })
orders: Promise<Order[]>;  // 注意返回类型是 Promise
```

**⚠️ 注意：避免循环依赖**

```typescript
// ❌ 避免：双向关系导致循环引用
@Entity('sys_users')
export class User {
  @ManyToOne(() => Dept, dept => dept.users)
  dept?: Dept;
}

@Entity('sys_depts')
export class Dept {
  @OneToMany(() => User, user => user.dept)
  users: User[];  // 循环依赖风险
}

// ✅ 推荐：使用 `eager: false` 或 DTO 投影
@ManyToOne(() => Dept, { eager: false })
@JoinColumn()
dept?: Dept;
```

### 4.1.4 字段类型选择

| 业务场景 | 推荐类型 | 说明 |
|----------|----------|------|
| 主键 ID | `BIGINT` 或 `VARCHAR(36)` (UUID) | UUID 更安全和分布式友好 |
| 金额 | `DECIMAL(10,2)` | 避免浮点数精度问题 |
| 状态/枚举 | `TINYINT` | 节省空间 |
| 短文本 (<100) | `VARCHAR(100)` | 可变长度 |
| 长文本 | `TEXT` | 文章、描述等 |
| 布尔值 | `TINYINT(1)` | 0/1 |
| 时间戳 | `TIMESTAMP` | 自动时区转换 |
| 日期 | `DATE` | 仅日期 |

### 4.1.5 数据库视图规范

**视图使用场景：**

- 简化复杂查询（多表 JOIN、聚合计算）
- 数据权限隔离（行级/列级权限控制）
- 报表统计查询
- 数据脱敏展示

**🔴 强制：视图命名规范**

| 视图类型 | 命名前缀 | 示例 | 说明 |
|----------|----------|------|------|
| 普通视图 | `v_` | `v_user_profiles` | 简单查询视图 |
| 复杂视图 | `vw_` | `vw_order_statistics` | 多表 JOIN/聚合视图 |
| 权限视图 | `v_auth_` | `v_auth_user_dept` | 带权限控制的视图 |
| 物化视图 | `mv_` | `mv_daily_stats` | 物化视图（缓存结果） |

**视图定义规范：**

```sql
-- 普通视图示例
CREATE OR REPLACE VIEW v_user_profiles AS
SELECT
    u.id,
    u.username,
    u.nickname,
    u.email,
    -- 手机号脱敏展示
    CONCAT(LEFT(u.phone, 3), '****', RIGHT(u.phone, 4)) AS masked_phone,
    -- 状态中文描述
    CASE u.status
        WHEN 1 THEN '正常'
        WHEN 0 THEN '禁用'
        ELSE '未知'
    END AS status_text
FROM sys_users u
WHERE u.deleted_at IS NULL;

-- 复杂视图示例（带权限控制）
CREATE OR REPLACE VIEW v_auth_user_dept AS
SELECT
    u.id AS user_id,
    u.username,
    u.nickname,
    d.id AS dept_id,
    d.dept_name,
    -- 数据权限级别
    CASE
        WHEN ur.role_code = 'ADMIN' THEN 1  -- 管理员：全部数据
        WHEN ur.role_code = 'DEPT_MANAGER' THEN 2  -- 部门经理：本部门 + 下级
        WHEN ur.role_code = 'DEPT_STAFF' THEN 3  -- 部门员工：仅本部门
        ELSE 4  -- 普通用户：仅本人
    END AS permission_level
FROM sys_users u
LEFT JOIN sys_user_roles ur ON u.id = ur.user_id
LEFT JOIN sys_depts d ON u.dept_id = d.id
WHERE u.deleted_at IS NULL;
```

**TypeORM 视图实体定义：**

```typescript
// entities/view/user-profile.view.entity.ts
import { ViewColumn, ViewEntity } from 'typeorm';
import { User } from '../user.entity';

@ViewEntity('v_user_profiles', { synchronize: false })
export class UserProfileView {
  @ViewColumn()
  id: string;

  @ViewColumn()
  username: string;

  @ViewColumn()
  nickname: string;

  @ViewColumn()
  email: string;

  @ViewColumn({ name: 'masked_phone' })
  maskedPhone: string;

  @ViewColumn({ name: 'status_text' })
  statusText: string;
}

// 使用示例
const profiles = await userRepository
  .createQueryBuilder('user')
  .innerJoin('v_user_profiles', 'profile', 'profile.id = user.id')
  .getMany();
```

**🟢 推荐：视图使用最佳实践**

1. **视图更新限制**
   - 避免在视图上进行 UPDATE/DELETE 操作
   - 需要更新时，直接操作基表

2. **物化视图刷新策略**
   ```sql
   -- 定时刷新（PostgreSQL 示例）
   REFRESH MATERIALIZED VIEW CONCURRENTLY mv_daily_stats;

   -- 或通过应用层定时任务刷新
   ```

3. **视图文档**
   - 在迁移文件中注释说明视图用途
   - 在 ER 图中标注视图依赖关系

**🟢 推荐：视图索引优化**

```sql
-- 为视图创建索引（物化视图）
CREATE UNIQUE INDEX idx_mv_stats_date ON mv_daily_stats (stat_date);
CREATE INDEX idx_mv_stats_dept ON mv_daily_stats (dept_id, stat_date);
```

## 4.2 迁移管理

### 4.2.1 迁移文件命名

```
格式：YYYYMMDDHHMMSS_描述.ts

示例：
20260330120000_create_users_table.ts
20260330120100_create_roles_table.ts
20260330120200_add_username_index_to_users.ts
```

### 4.2.2 迁移文件结构

```typescript
// migrations/20260330120000_create_users_table.ts
import { MigrationInterface, QueryRunner, Table, TableIndex } from 'typeorm';

export class CreateUsersTable1711771200000 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.createTable('sys_users', new Table({
      name: 'sys_users',
      columns: [
        {
          name: 'id',
          type: 'varchar',
          length: '36',
          isPrimary: true,
        },
        {
          name: 'username',
          type: 'varchar',
          length: '50',
          isUnique: true,
        },
        {
          name: 'password_hash',
          type: 'varchar',
          length: '255',
        },
        {
          name: 'created_at',
          type: 'timestamp',
          default: 'CURRENT_TIMESTAMP',
        },
        {
          name: 'updated_at',
          type: 'timestamp',
          default: 'CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP',
        },
      ],
    }));

    // 创建索引
    await queryRunner.createIndex(
      'sys_users',
      new TableIndex({
        name: 'idx_users_username',
        columnNames: ['username'],
      }),
    );
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.dropTable('sys_users');
  }
}
```

### 4.2.4 迁移回滚规范

**🔴 强制：down() 方法必须实现且可逆**

```typescript
// ✅ 推荐：完整的 down() 方法
export class CreateUsersTable1711771200000 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.createTable('sys_users', new Table({
      name: 'sys_users',
      columns: [
        { name: 'id', type: 'varchar', length: '36', isPrimary: true },
        { name: 'username', type: 'varchar', length: '50', isUnique: true },
      ],
    }));

    await queryRunner.createIndex(
      'sys_users',
      new TableIndex({
        name: 'idx_users_username',
        columnNames: ['username'],
      }),
    );
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    // 先删除索引，再删除表（依赖顺序）
    await queryRunner.dropIndex('sys_users', 'idx_users_username');
    await queryRunner.dropTable('sys_users');
  }
}
```

**⚠️ 注意：数据迁移回滚策略**

```typescript
// 添加字段的迁移
export class AddNicknameToUsers1711771300000 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.addColumn('sys_users', new TableColumn({
      name: 'nickname',
      type: 'varchar',
      length: '50',
      isNullable: true,
      comment: '用户昵称',
    }));
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    // 回滚前需确认是否有数据依赖
    // 如有数据依赖，应在文档中说明回滚影响
    await queryRunner.dropColumn('sys_users', 'nickname');
  }
}
```

**🔴 强制：破坏性变更需额外审批**

| 变更类型 | 风险等级 | 回滚策略 | 审批要求 |
|----------|----------|----------|----------|
| 添加表/字段 | 🟢 低风险 | 直接删除 | 开发负责人 |
| 修改字段类型 | 🟡 中风险 | 需数据迁移 | 技术负责人 |
| 删除表/字段 | 🔴 高风险 | 无法恢复数据 | CTO/架构师 |
| 重命名表/字段 | 🔴 高风险 | 需双写过渡 | CTO/架构师 |

```typescript
describe('Migration: CreateUsersTable', () => {
  let dataSource: DataSource;

  beforeAll(async () => {
    dataSource = new DataSource({ /* 测试数据库配置 */ });
    await dataSource.initialize();
  });

  it('should run migration successfully', async () => {
    const migration = new CreateUsersTable1711771200000();
    const queryRunner = dataSource.createQueryRunner();

    await migration.up(queryRunner);

    const tableExists = await queryRunner.hasTable('sys_users');
    expect(tableExists).toBe(true);

    expect(tableExists).toBe(false);
  });

  afterAll(async () => {
    await dataSource.destroy();
  });
});
```

## 4.3 查询优化

### 4.3.1 索引使用原则

**应该创建索引的场景：**
- 主键（自动创建）
- 外键字段
- WHERE 子句中的字段
- ORDER BY 中的字段
- GROUP BY 中的字段
- 联合查询的连接字段

**不应创建索引的场景：**
- 表记录很少（<1000）
- 频繁更新的字段
- 区分度低的字段（如性别）
- 在函数中使用的字段

### 4.3.2 联合索引最左前缀原则

```typescript
// ✅ 推荐：联合索引 (a, b, c)
CREATE INDEX idx_abc ON table_name(a, b, c);

// 可以使用索引的查询
WHERE a = 1                    // ✅ 使用全部
WHERE a = 1 AND b = 2          // ✅ 使用前两个
WHERE a = 1 AND b = 2 AND c = 3 // ✅ 使用全部

// 不能使用索引的查询
WHERE b = 2                    // ❌ 跳过 a
WHERE c = 3                    // ❌ 跳过 a, b
WHERE b = 2 AND c = 3          // ❌ 跳过 a
```

### 4.3.3 避免 N+1 查询

```typescript
// ❌ 避免：N+1 查询问题
```

### 4.3.6 索引覆盖规范

**🔴 强制：高频查询必须使用索引覆盖**

**索引覆盖（Covering Index）：** 查询所需的所有字段都在索引中，无需回表查询。

```sql
-- ❌ 避免：查询字段不在索引中，需要回表
CREATE INDEX idx_username ON sys_users(username);
SELECT username, email, phone FROM sys_users WHERE username = 'test';
-- 需要回表查询 email 和 phone

-- ✅ 推荐：创建覆盖索引
CREATE INDEX idx_username_email_phone ON sys_users(username, email, phone);
SELECT username, email, phone FROM sys_users WHERE username = 'test';
-- 直接从索引返回数据，无需回表
```

**TypeORM 实体类索引定义：**

```typescript
@Entity('sys_users')
@Index(['username', 'email', 'phone'])  // 联合索引
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ length: 50, unique: true })
  username: string;

  @Column({ length: 100, nullable: true })
  email?: string;

  @Column({ length: 20, nullable: true })
  phone?: string;
}
```

**🟢 推荐：索引覆盖最佳实践**

| 场景 | 索引策略 | 示例 |
|------|----------|------|
| WHERE + ORDER BY | 索引包含 WHERE 和 ORDER BY 字段 | `idx_status_created(status, created_at)` |
| WHERE + GROUP BY | 索引包含 WHERE 和 GROUP BY 字段 | `idx_dept_status(dept_id, status)` |
| 高频查询 | 索引覆盖所有 SELECT 字段 | `idx_username_email(username, email)` |
| 大字段 | 避免在覆盖索引中包含 TEXT/BLOB | 使用延迟加载 |

**⚠️ 注意：覆盖索引的局限性**

```sql
-- ❌ 避免：覆盖索引无法优化的场景
SELECT * FROM sys_users WHERE username = 'test';
-- SELECT * 会包含所有字段，覆盖索引失效

-- ✅ 推荐：只查询需要的字段
SELECT id, username, email FROM sys_users WHERE username = 'test';
```

|------|------|----------|
| type | 连接类型 | 至少达到 `range`，推荐 `ref` 或 `eq_ref` |
| key | 使用的索引 | 必须有值，避免 `NULL` |
| rows | 扫描行数 | 越少越好 |

**常见问题及优化：**

```sql
-- 问题 2：Using temporary（临时表）

SELECT dept_id, COUNT(*) as cnt
FROM sys_users
GROUP BY dept_id;

SELECT dept_id, COUNT(*) as cnt
FROM sys_users
WHERE dept_id IS NOT NULL
GROUP BY dept_id;

-- 问题 3：全表扫描（type = ALL）
-- 原因：WHERE 字段没有索引
-- 优化：为 WHERE 字段创建索引

```

### 4.4.2 事务实现

```typescript
// ✅ 推荐：使用装饰器事务
@Injectable()
export class UserService {
  @Transactional()
  async register(dto: RegisterDTO): Promise<User> {
    // 1. 创建用户
    const user = await this.userDAO.create({

// ✅ 推荐：手动事务
async transfer(fromId: string, toId: string, amount: number) {
  const queryRunner = this.dataSource.createQueryRunner();
  await queryRunner.connect();
  await queryRunner.startTransaction();

  try {
    await queryRunner.manager.decrement(
      'accounts',
      { id: fromId },
      'balance',
      amount,
    );
    await queryRunner.manager.increment(
      'accounts',
      { id: toId },
      'balance',
      amount,
    );

    await queryRunner.commitTransaction();
  } catch (error) {
    await queryRunner.rollbackTransaction();
    throw error;
  } finally {
    await queryRunner.release();
  }
}
```

### 4.4.3 事务隔离级别

| 隔离级别 | 脏读 | 不可重复读 | 幻读 | 说明 |
|----------|------|------------|------|------|
| READ UNCOMMITTED | ✅ | ✅ | ✅ | 允许读取未提交事务（最少用） |
| READ COMMITTED (默认) | ❌ | ✅ | ✅ | 只允许读取已提交事务（Oracle/PG 默认） |
| REPEATABLE READ | ❌ | ❌ | ✅ | 同一查询多次读取结果一致（MySQL 默认） |
| SERIALIZABLE | ❌ | ❌ | ❌ | 完全串行化，性能最低 |

### 4.4.4 事务传播行为

**🔴 强制：嵌套事务必须明确传播行为**

事务传播行为（Propagation Behavior）定义了事务方法被另一个事务方法调用时的事务处理方式。

**TypeORM + NestJS 事务传播行为：**

```typescript
import { Transactional } from 'typeorm-transactional';
import { TransactionHost } from '@nestjs-cls/transactional';

@Injectable()
export class UserService {
  constructor(
    private readonly transactionHost: TransactionHost,
    private readonly userService: UserService,
    private readonly auditService: AuditService,
  ) {}

  // REQUIRED（默认）：如果当前有事务，则加入；如果没有，则新建事务
  @Transactional()
  async register(dto: RegisterDTO): Promise<User> {
    // 1. 创建用户（新建事务）
    const user = await this.transactionHost.tx.manager.save(User, dto);

    // 2. 调用其他服务方法（加入同一事务）
    await this.auditService.logRegister(user.id);  // REQUIRED

    return user;
  }

  // REQUIRES_NEW：总是新建事务，如果当前有事务则挂起
  @Transactional({ isolation: 'READ COMMITTED' })
  async logAudit(action: string, userId: string): Promise<void> {
    // 总是新建事务，即使被 @Transactional 方法调用
    await this.transactionHost.tx.manager.save(AuditLog, { action, userId });
  }

  // NESTED：在当前事务中创建嵌套事务（保存点）
  @Transactional({ connectionName: 'default' })
  async updateWithNested(): Promise<void> {
    // 外层事务
    await this.transactionHost.tx.manager.save(User, { id: 1, name: 'A' });

    try {
      // 嵌套事务（保存点）
      await this.transactionHost.tx.transaction(async (tx) => {
        await tx.save(User, { id: 1, name: 'B' });
        throw new Error('嵌套事务回滚');
      });
    } catch (e) {
      // 嵌套事务回滚，外层事务不受影响
      console.log('嵌套事务已回滚');
    }
  }
}
```

**传播行为类型说明：**

| 传播行为 | 当前有事务 | 当前无事务 | 使用场景 |
|----------|------------|------------|----------|
| REQUIRED (默认) | 加入现有事务 | 新建事务 | 大多数业务方法 |
| REQUIRES_NEW | 挂起现有事务，新建事务 | 新建事务 | 审计日志、独立操作 |
| NESTED | 创建嵌套事务（保存点） | 新建事务 | 部分回滚场景 |
| SUPPORTS | 加入现有事务 | 非事务执行 | 查询方法 |
| NOT_SUPPORTED | 挂起现有事务，非事务执行 | 非事务执行 | 批处理操作 |
| NEVER | 抛出异常 | 非事务执行 | 强制非事务方法 |
| MANDATORY | 加入现有事务 | 抛出异常 | 必须在事务中调用 |

**⚠️ 注意：循环依赖问题**

```typescript
// ❌ 避免：循环依赖导致事务失效
@ServiceA
class ServiceA {
  @Transactional()
  async methodA() {
    await this.serviceB.methodB();  // 事务传播失效
  }
}

@ServiceB
class ServiceB {
  @Transactional()
  async methodB() {
    await this.serviceA.methodA();  // 循环依赖
  }
}

// ✅ 推荐：使用事件或消息队列解耦
```

### 4.4.5 事务超时配置

**🔴 强制：长时间运行的事务必须配置超时**

```typescript
// TypeORM 事务超时配置
async processWithTimeout(): Promise<void> {
  const queryRunner = this.dataSource.createQueryRunner();
  await queryRunner.connect();

  // 设置事务超时（MySQL/PostgreSQL）
  await queryRunner.query('SET LOCAL statement_timeout = 30000'); // 30 秒

  await queryRunner.startTransaction();

  try {
    // 长时间运行的操作
    await this.longRunningOperation(queryRunner.manager);

    await queryRunner.commitTransaction();
  } catch (error) {
    await queryRunner.rollbackTransaction();
    throw error;
  } finally {
    await queryRunner.release();
    // 重置超时设置
    await queryRunner.query('SET LOCAL statement_timeout = DEFAULT');
  }
}
```

**NestJS + TypeORM 事务超时装饰器：**

```typescript
// 自定义事务超时装饰器
export function TransactionalWithTimeout(timeoutMs: number = 30000) {
  return (target: any, key: string, descriptor: PropertyDescriptor) => {
    const originalMethod = descriptor.value;

    descriptor.value = async function (...args: any[]) {
      const dataSource = this.dataSource;
      const queryRunner = dataSource.createQueryRunner();

      await queryRunner.connect();
      await queryRunner.query(`SET LOCAL statement_timeout = ${timeoutMs}`);
      await queryRunner.startTransaction();

      try {
        const result = await originalMethod.apply(this, [queryRunner, ...args]);
        await queryRunner.commitTransaction();
        return result;
      } catch (error) {
        await queryRunner.rollbackTransaction();
        throw error;
      } finally {
        await queryRunner.query('SET LOCAL statement_timeout = DEFAULT');
        await queryRunner.release();
      }
    };

    return descriptor;
  };
}

// 使用示例
@Injectable()
export class ReportService {
  @TransactionalWithTimeout(60000)  // 60 秒超时
  async generateReport(queryRunner: QueryRunner, options: ReportOptions) {
    // 生成报表的耗时操作
    await queryRunner.manager.query(/* 复杂查询 */);
  }
}
```

**🟢 推荐：事务超时最佳实践**

| 操作类型 | 推荐超时 | 说明 |
|----------|----------|------|
| 简单 CRUD | 5-10 秒 | 单表操作 |
| 关联更新 | 10-30 秒 | 多表关联 |
| 批量操作 | 30-60 秒 | 分批处理 |
| 报表生成 | 60-300 秒 | 复杂聚合查询 |

**⚠️ 注意：事务过长的危害**

1. **锁等待**：长时间持有锁，阻塞其他事务
2. **回滚段溢出**：长事务可能导致回滚段过大
3. **连接耗尽**：长时间占用数据库连接
4. **主从延迟**：大事务在主库执行后，从库回放耗时


**字段级加密（AES-256-GCM）：**

```typescript
import * as crypto from 'crypto';

const ALGORITHM = 'aes-256-gcm';
const IV_LENGTH = 16;
const AUTH_TAG_LENGTH = 16;
const KEY_LENGTH = 32; // 256 bits

// 从环境变量获取密钥（32 字节）
const ENCRYPTION_KEY = process.env.ENCRYPTION_KEY || '';

interface EncryptedData {
  ciphertext: string;
  iv: string;
  authTag: string;
}

/**
 * AES-256-GCM 加密
 */
function encrypt(plaintext: string): EncryptedData {
  const iv = crypto.randomBytes(IV_LENGTH);
  const cipher = crypto.createCipheriv(ALGORITHM, Buffer.from(ENCRYPTION_KEY), iv);

  let ciphertext = cipher.update(plaintext, 'utf8', 'hex');
  ciphertext += cipher.final('hex');

  const authTag = cipher.getAuthTag().toString('hex');

  return { ciphertext, iv: iv.toString('hex'), authTag };
}

/**
 * AES-256-GCM 解密
 */
function decrypt(encrypted: EncryptedData): string {
  const decipher = crypto.createDecipheriv(
    ALGORITHM,
    Buffer.from(ENCRYPTION_KEY),
    Buffer.from(encrypted.iv, 'hex'),
  );

  decipher.setAuthTag(Buffer.from(encrypted.authTag, 'hex'));

  let plaintext = decipher.update(encrypted.ciphertext, 'hex', 'utf8');
  plaintext += decipher.final('utf8');

  return plaintext;
}

// TypeORM 转换器
import { ValueTransformer } from 'typeorm';

export class EncryptedColumnTransformer implements ValueTransformer {
  to(value: string | null): string | null {
    if (!value) return value;
    const encrypted = encrypt(value);
    return JSON.stringify(encrypted);
  }

  from(value: string | null): string | null {
    if (!value) return value;
    const encrypted = JSON.parse(value) as EncryptedData;
    return decrypt(encrypted);
  }
}

// 实体类使用示例
@Entity('sys_users')
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({
    type: 'varchar',
    transformer: new EncryptedColumnTransformer(),
    comment: '身份证号 - 加密存储',
  })
  idCard: string;

  @Column({
    type: 'varchar',
    transformer: new EncryptedColumnTransformer(),
    comment: '手机号 - 加密存储',
```

```typescript
// 密钥轮换策略
interface KeyVersion {

  key: string;
  createdAt: Date;
  retiredAt?: Date;
}

// 多版本密钥支持
class KeyManager {
  private keys: Map<number, KeyVersion> = new Map();
  private currentVersion: number = 1;

  // 获取当前密钥
  getCurrentKey(): string {
    return this.keys.get(this.currentVersion)!.key;
  }

  // 根据版本获取密钥
  getKeyByVersion(version: number): string {
    return this.keys.get(version)!.key;
  }

  // 轮换密钥
  rotateKey(newKey: string): void {
    const oldVersion = this.currentVersion;
    this.currentVersion++;
    this.keys.set(this.currentVersion, {
      version: this.currentVersion,
      key: newKey,
      createdAt: new Date(),
    });
    // 标记旧密钥为已退役
    this.keys.get(oldVersion)!.retiredAt = new Date();
  }
}
```

```typescript
// 后端解密
function decryptWithPrivateKey(encryptedData: string, privateKey: string): string {
  const buffer = Buffer.from(encryptedData, 'base64');
  return crypto.privateDecrypt(
    {
      key: privateKey,
      padding: crypto.constants.RSA_PKCS1_OAEP_PADDING,
      oaepHash: 'sha256',
    },
    buffer,
  ).toString('utf8');
}
```
