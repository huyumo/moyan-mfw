## 2.1 分层架构

### 2.1.1 经典三层架构

```
┌─────────────────────────────────────────────────────────┐
│                    表现层 (Controller)                    │
│                    HTTP 接口、请求响应                      │
└────────────────────────┬────────────────────────────────┘
                         │ 依赖 ↓
┌─────────────────────────────────────────────────────────┐
│                    业务层 (Service)                       │
│                   业务逻辑、事务处理                       │
└────────────────────────┬────────────────────────────────┘
                         │ 依赖 ↓
┌─────────────────────────────────────────────────────────┐
│                  持久层 (DAO/Repository)                  │
│                   数据访问、CRUD 操作                        │
└────────────────────────┬────────────────────────────────┘
                         │ 依赖 ↓
┌─────────────────────────────────────────────────────────┐
│                    数据层 (Database)                      │
│                   MySQL/PostgreSQL 等                      │
└─────────────────────────────────────────────────────────┘
```

**依赖方向规则：**
- 上层可依赖下层，下层不可依赖上层
- Controller → Service → DAO → Database（单向依赖）
- 禁止跨层调用（如 Controller 直接调用 DAO）

### 2.1.2 各层职责

| 层级 | 职责 | 依赖方向 | 示例 |
|------|------|----------|------|
| Controller 层 | 接收 HTTP 请求、参数验证、调用 Service、返回响应 | → Service | `UserController.createUser()` |
| Service 层 | 业务逻辑、事务管理、调用 DAO | → DAO | `UserService.register()` |
| DAO 层 | 数据库操作、CRUD、查询构建 | → Database | `UserDAO.findById()` |
| Database 层 | 数据存储、索引、事务支持 | - | MySQL 表 |

### 2.1.3 分层示例

```typescript
// ========== Controller 层 ==========
@Controller('users')
export class UserController {
  constructor(private userService: UserService) {}

  @Post()
  async createUser(@Body() dto: CreateUserDTO): Promise<ApiResponse<UserDTO>> {
    const user = await this.userService.createUser(dto);
    return ApiResponse.success(user);
  }
}

// ========== Service 层 ==========
@Injectable()
export class UserService {
  constructor(
    private userDAO: UserDAO,
    private roleDAO: RoleDAO,
    private eventEmitter: EventEmitter,
  ) {}

  @Transactional()
  async createUser(dto: CreateUserDTO): Promise<UserDTO> {
    // 1. 检查用户名是否存在
    const exists = await this.userDAO.findByUsername(dto.username);
    if (exists) {
      throw new UserExistsError(dto.username);
    }

    // 2. 创建用户
    const user = await this.userDAO.create({
      username: dto.username,
      password: await bcrypt.hash(dto.password, 10),
    });

    // 3. 绑定默认角色
    const defaultRole = await this.roleDAO.getDefault();
    await this.userDAO.bindRole(user.id, defaultRole.id);

    // 4. 发送事件
    this.eventEmitter.emit('user.created', user);

    return this.toDTO(user);
  }
}

// ========== DAO 层 ==========
@Injectable()
export class UserDAO {
  constructor(
    @InjectRepository(UserEntity)
    private repo: Repository<UserEntity>,
  ) {}

  async findById(id: string): Promise<UserEntity | null> {
    return this.repo.findOne({ where: { id } });
  }

  async findByUsername(username: string): Promise<UserEntity | null> {
    return this.repo.findOne({ where: { username } });
  }

  async create(data: CreateUserData): Promise<UserEntity> {
    const user = this.repo.create(data);
    return this.repo.save(user);
  }
}
```

### 2.2.3 模块间通信

**原则：**
- 模块间通过 Service 接口通信
- 避免循环依赖
- 使用事件解耦

**🔴 强制：模块依赖规则**

| 规则 | 说明 | 示例 |
|------|------|------|
| 单向依赖 | 上层模块可依赖下层模块，反向禁止 | UserModule → AuthModule ✅，AuthModule → UserModule ❌ |
| 禁止循环 | 模块间禁止循环依赖 | UserModule → RoleModule → UserModule ❌ |
| 公共优先 | 公共模块可被所有模块依赖 | CommonModule → 所有模块 ✅ |
| 接口解耦 | 跨模块依赖通过接口解耦 | 使用 `IUserService` 而非 `UserService` |

```typescript
// ✅ 推荐：使用事件解耦
// user.module.ts
@Module({
  imports: [
    // 导入需要的模块
    RoleModule,
  ],
  providers: [UserService, UserDAO],
  controllers: [UserController],
  exports: [UserService],  // 导出服务供其他模块使用
})
export class UserModule {}

// ✅ 推荐：使用事件总线
// 在 Service 中发送事件
this.eventEmitter.emit('user.created', { user });

// 在另一个模块中监听
@OnEvent('user.created')
handleUserCreated(payload: { user: User }) {
  // 发送欢迎邮件
  this.emailService.sendWelcome(payload.user);
}
```

**循环依赖解决方案：**

1. **通过接口解耦**
```typescript
// 定义接口
export interface IUserService {
  getUserById(id: string): Promise<User>;
}

// 模块 A 实现接口
@Injectable()
export class UserService implements IUserService { ... }

// 模块 B 通过接口依赖
export class RoleService {
  constructor(
    @Inject('IUserService')
    private userService: IUserService
  ) {}
}
```

2. **通过事件总线解耦**
```typescript
// 模块 A 发送事件
this.eventEmitter.emit('user.created', user);

// 模块 B 监听事件（无需直接依赖模块 A）
@OnEvent('user.created')
handleUserCreated(user: User) { ... }
```

## 2.3 依赖注入

### 2.3.1 构造函数注入（推荐）

```typescript
// ✅ 推荐
@Injectable()
export class UserService {
  constructor(
    private userDAO: UserDAO,
    private roleDAO: RoleDAO,
    private logger: Logger,
  ) {}
}

// ❌ 避免：在类内部实例化依赖
export class UserService {
  private userDAO = new UserDAO();  // 难以测试
}
```

### 2.3.2 接口注入

**使用场景：**
- 跨模块依赖，避免循环依赖
- 需要多个实现类，运行时切换
- 单元测试时需要 Mock 依赖

```typescript
// 定义接口
export interface IUserService {
  getUserById(id: string): Promise<User>;
}

// 实现类
@Injectable()
export class UserService implements IUserService {
  async getUserById(id: string): Promise<User> {
    // ...
  }
}

// 使用接口注入
@Controller('users')
export class UserController {
  constructor(
    @Inject('IUserService')
    private userService: IUserService,  // 通过接口依赖，而非具体实现
  ) {}
}
```

### 2.3.3 模块导出与导入

```typescript
// user.module.ts
@Module({
  imports: [TypeOrmModule.forFeature([User])],
  providers: [UserService, UserDAO],

// role.module.ts - 使用 UserModule 导出的服务
@Module({
  imports: [UserModule],  // 导入 UserModule 使用其导出的服务
```

## 2.5 中间件规范

### 2.5.1 中间件类型

| 中间件 | 用途 | 执行时机 |
|--------|------|----------|
| 认证中间件 | 验证用户身份 | 路由处理前 |
| 日志中间件 | 记录请求日志 | 请求开始/结束 |
| 异常处理中间件 | 统一异常处理 | 控制器异常抛出后 |
| CORS 中间件 | 跨域处理 | 响应前 |
| 限流中间件 | 请求频率限制 | 路由处理前 |

### 2.5.2 中间件实现

```typescript
// 认证中间件
@Injectable()
export class AuthMiddleware implements NestMiddleware {
  constructor(private jwtService: JwtService) {}

  use(req: Request, res: Response, next: NextFunction) {
    const token = this.extractToken(req);
    if (!token) {
      throw new UnauthorizedError();
    }

    try {
      const payload = this.jwtService.verify(token);
      req.user = payload;
      next();
    } catch (error) {
      throw new UnauthorizedError();
    }
  }
}
```

## 2.6 异常处理

### 2.6.1 自定义异常

```typescript
// 业务异常基类
export class BusinessException extends HttpException {
  constructor(
    message: string,
    code: number,
    status: HttpStatus,
  ) {
    super({ code, message, data: null }, status);
  }
}

// 具体业务异常
export class UserNotFoundError extends BusinessException {
  constructor(id: string) {
    super(`用户 ${id} 不存在`, 10001, HttpStatus.NOT_FOUND);
  }
}

export class UserExistsError extends BusinessException {
  constructor(username: string) {
    super(`用户名 ${username} 已存在`, 10002, HttpStatus.CONFLICT);
  }
}
```

### 2.6.2 全局异常过滤器

```typescript
@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  private readonly logger = new Logger();

  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();

    let status = HttpStatus.INTERNAL_SERVER_ERROR;
    let code = 50000;
    let message = '服务器内部错误';

    if (exception instanceof HttpException) {
      const resp = exception.getResponse() as any;
      status = exception.getStatus();
      code = resp.code || status;
      message = resp.message || exception.message;
    }

    // 记录日志
    this.logger.error(
      `请求失败：${request.method} ${request.url}`,
      exception instanceof Error ? exception.stack : '',
    );

    // 返回统一格式
    response.status(status).json({
      code,
      message,
      data: null,
    });
  }
}
```

### 2.6.3 全局异常过滤器注册

**方式一：在 main.ts 中全局注册**

```typescript
// main.ts
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import { AllExceptionsFilter } from './common/filters/all-exceptions.filter';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // 全局注册异常过滤器
  app.useGlobalFilters(new AllExceptionsFilter());

  await app.listen(3000);
}
bootstrap();
```

**方式二：在 AppModule 中注册**

```typescript
// app.module.ts
import { Module } from '@nestjs/common';
import { APP_FILTER } from '@nestjs/core';
import { AllExceptionsFilter } from './common/filters/all-exceptions.filter';

@Module({
  imports: [...],
  providers: [
    {
      provide: APP_FILTER,
      useClass: AllExceptionsFilter,
    },
  ],
})
export class AppModule {}
```

**方式三：在控制器中局部注册**

```typescript
// UserController.ts
@Controller('users')
@UseFilters(new AllExceptionsFilter())
export class UserController {
  // 仅对该控制器生效
}
```

## 2.8 代码复杂度控制

### 2.8.1 代码行数限制

| 指标 | 限制 | 等级 | 说明 |
|------|------|------|------|
| 函数行数 | ≤ 50 行 | 🟢 推荐 | 超过需考虑拆分 |
| 类行数 | ≤ 300 行 | 🟢 推荐 | 超过需考虑拆分 |
| 文件行数 | ≤ 500 行 | 🔴 强制 | 超过必须拆分 |
| 方法参数 | ≤ 4 个 | 🟢 推荐 | 超过需使用对象参数 |

### 2.8.2 嵌套层级限制

**🔴 强制：嵌套层级不得超过 4 层**

```typescript
// ❌ 避免：过深的嵌套（5 层）
async function processUser(user: User) {
  if (user) {                    // 1 层
    if (user.isActive) {         // 2 层
      if (user.hasPermission) {  // 3 层
        for (const item of user.items) {  // 4 层
          if (item.valid) {      // 5 层 - 过深
            // ...
          }
        }
      }
    }
  }
}

// ✅ 推荐：使用卫语句减少嵌套
async function processUser(user: User) {
  if (!user || !user.isActive) return;
  if (!user.hasPermission) return;

  for (const item of user.items) {
    if (!item.valid) continue;
    // ...
  }
}
```

### 2.8.3 圈复杂度控制

**🟢 推荐：圈复杂度 ≤ 10**

圈复杂度（Cyclomatic Complexity）是代码中独立执行路径的数量，计算方式：
- 基础复杂度 = 1
- 每个 `if`/`else if`/`for`/`while`/`case` +1
- 每个 `catch`/`? :`（三元运算符）+1
- 每个 `&&`/`||`（逻辑运算符）+1

```typescript
// ❌ 避免：圈复杂度过高（>10）
function calculatePrice(order: Order): number {
  let price = order.basePrice;

  if (order.user.isVip) {        // +1
    price *= 0.9;
  } else if (order.user.isMember) {  // +1
    price *= 0.95;
  }

  if (order.items.length > 10) { // +1
    price *= 0.9;
  } else if (order.items.length > 5) {  // +1
    price *= 0.95;
  }

  if (order.coupon) {            // +1
    price -= order.coupon.value;
  }

  if (order.shipping === 'express') {  // +1
    price += 20;
  } else if (order.shipping === 'standard') {  // +1
    price += 10;
  }

  if (price < 0) price = 0;      // +1
  if (price > 10000) price = 10000;  // +1

  return Math.round(price * 100) / 100;  // 圈复杂度 = 10
}

// ✅ 推荐：拆分为多个函数
function applyUserDiscount(price: number, user: User): number {
  if (user.isVip) return price * 0.9;
  if (user.isMember) return price * 0.95;
  return price;
}

function applyQuantityDiscount(price: number, quantity: number): number {
  if (quantity > 10) return price * 0.9;
  if (quantity > 5) return price * 0.95;
  return price;
}

function calculatePrice(order: Order): number {
  let price = order.basePrice;
  price = applyUserDiscount(price, order.user);
  price = applyQuantityDiscount(price, order.items.length);
  price = applyCoupon(price, order.coupon);
  price = applyShipping(price, order.shipping);
  return clampPrice(price);
}
```
