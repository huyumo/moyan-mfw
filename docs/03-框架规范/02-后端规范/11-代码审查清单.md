## API 审查

### RESTful 规范

- [ ] 路径是否使用复数名词（`/users` 而非 `/user` 或 `/getUserList`）
- [ ] 是否使用 HTTP 方法表达操作（GET/POST/PUT/DELETE）
- [ ] 路径是否包含版本号（`/api/v1/users`）
- [ ] 资源嵌套是否不超过三级

### 请求参数

- [ ] 请求参数是否有完整的验证
- [ ] 必填参数是否有校验
- [ ] 参数类型是否正确
- [ ] 是否有合理的默认值
- [ ] 分页参数是否规范（page, pageSize）

### 响应格式

- [ ] 响应结构是否统一（code, data, message）
- [ ] 成功响应 code 是否为 0
- [ ] 错误响应是否包含有意义的 message
- [ ] 分页响应是否包含 list, total, page, pageSize
- [ ] 敏感数据是否脱敏（密码、手机号等）

## 安全审查

### SQL 注入防护

- [ ] 是否使用参数化查询/ORM
- [ ] 是否避免拼接 SQL 字符串
- [ ] 用户输入是否经过验证和转义

```typescript
// ✅ 推荐
const user = await repository
  .createQueryBuilder('user')
  .where('user.name = :name', { name })
  .getOne();

// ❌ 避免
const user = await query(`SELECT * FROM users WHERE name = '${name}'`);
```

### XSS 防护

- [ ] 输出到前端的数据是否进行 HTML 转义
- [ ] 是否避免直接返回用户输入的 HTML 内容
- [ ] 富文本内容是否进行白名单过滤

### 认证与授权

- [ ] 需要认证的接口是否有 `@UseGuards(AuthGuard)`
- [ ] 需要权限的接口是否有 `@RequirePermission`
- [ ] 是否验证了用户身份与资源的所属关系
- [ ] 是否避免越权访问

### 敏感数据加密

- [ ] 密码是否使用 bcrypt 加密存储
- [ ] Token 是否使用 JWT 签名
- [ ] 传输敏感数据是否使用 HTTPS
- [ ] 日志中是否避免输出敏感信息

### 速率限制

- [ ] 登录接口是否有速率限制
- [ ] 短信/邮件发送接口是否有频率限制
- [ ] 是否有防刷机制

## 数据审查

### 数据库操作

- [ ] 是否正确使用了事务
- [ ] 事务粒度是否合适
- [ ] 是否有事务超时设置
- [ ] 是否正确处理了并发冲突

```typescript
// ✅ 推荐
await this.dataSource.transaction(async (manager) => {
  await manager.save(user);
  await manager.save(role);
});

// ❌ 避免
await this.userRepository.save(user);
await this.roleRepository.save(role);  // 前面失败时数据不一致
```

### 索引使用

- [ ] 查询条件字段是否有索引
- [ ] 联合索引是否符合最左前缀原则
- [ ] 是否避免在索引列上使用函数
- [ ] 是否使用 EXPLAIN 分析慢查询

### 查询优化

- [ ] 是否避免 SELECT *
- [ ] 是否只查询需要的字段
- [ ] 是否正确使用了 JOIN
- [ ] 是否有限制返回数量（LIMIT）
- [ ] 是否有 N+1 查询问题

### 数据验证

- [ ] 创建/更新操作是否有 DTO 验证
- [ ] 数据类型是否正确
- [ ] 字符串长度是否有限制
- [ ] 枚举值是否在校验范围内
- [ ] 外键关联是否存在

## 代码质量审查

### 代码结构

- [ ] 是否遵循分层架构（Controller → Service → DAO）
- [ ] 类职责是否单一
- [ ] 方法长度是否适中（不超过 50 行）
- [ ] 类长度是否适中（不超过 300 行）

### 命名规范

- [ ] 类名是否使用 PascalCase
- [ ] 方法名是否使用 camelCase
- [ ] 常量是否使用 UPPER_SNAKE_CASE
- [ ] 文件命名是否符合规范

### 错误处理

- [ ] 是否有 try-catch 处理异步操作
- [ ] 是否捕获具体的错误类型
- [ ] 错误日志是否包含足够的上下文
- [ ] 是否向用户返回友好的错误消息

```typescript
// ✅ 推荐
try {
  await userService.create(dto);
} catch (error) {
  if (error instanceof UserExistsError) {
    throw new HttpException('用户已存在', 409);
  }
  logger.error('创建用户失败:', { error, dto });
  throw new HttpException('服务器错误', 500);
}

// ❌ 避免
try {
  await userService.create(dto);
} catch (e) {
  console.log(e);  // 日志信息不足
}
```

### 日志记录

- [ ] 关键操作是否有日志记录
- [ ] 日志级别是否合适（info/warn/error）
- [ ] 日志是否包含追踪 ID
- [ ] 是否避免记录敏感信息

- [ ] 频繁查询的数据是否使用缓存

### 并发处理

- [ ] 是否可以并发执行的异步操作是否使用了 Promise.all
- [ ] 是否有潜在的竞态条件
- [ ] 是否正确处理了并发请求

```typescript
// ✅ 推荐
const [user, roles] = await Promise.all([
  this.userRepository.findById(id),
  this.roleRepository.findByUserId(id),
]);

// ❌ 避免
const user = await this.userRepository.findById(id);
const roles = await this.roleRepository.findByUserId(id);  // 串行执行
```
