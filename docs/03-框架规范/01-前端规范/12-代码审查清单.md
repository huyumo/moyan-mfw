# 代码审查清单

> 状态：**已完成** | 版本：1.0.0

---

## 组件审查

### 结构与命名

- [ ] 组件命名是否使用 PascalCase
- [ ] 组件文件命名是否与组件名一致
- [ ] 组件职责是否单一（是否做过的事情太多）
- [ ] 是否使用了 `defineOptions({ name: 'ComponentName' })`

### Props/Emits

- [ ] Props 是否使用 TypeScript 接口定义
- [ ] Props 是否有合理的默认值
- [ ] Props 命名是否使用 camelCase
- [ ] Emits 是否使用动词开头（如 `submit`, `update:modelValue`）
- [ ] Emits 是否有清晰的参数类型定义

### 模板

- [ ] 是否使用 `v-if` 而非 `v-show` 处理条件渲染（不需要频繁切换时）
- [ ] `v-for` 是否使用了唯一的 `key`
- [ ] 是否避免在 `v-for` 中使用 `v-if`
- [ ] 是否使用语义化 HTML 标签

---

## 逻辑审查

### 状态管理

- [ ] 状态是否放在合适的位置（组件内 vs Store）
- [ ] 是否避免不必要的响应式数据
- [ ] 大型对象是否使用 `shallowRef`
- [ ] 计算属性是否有缓存价值

### 异步处理

- [ ] 异步操作是否有错误处理
- [ ] 是否处理了加载状态
- [ ] 是否避免竞态条件
- [ ] 组件卸载时是否清理异步操作
- [ ] try-catch 范围是否合理（不会吞掉业务错误）
- [ ] ElMessageBox 取消操作是否单独 catch 并 return

### 代码复用

- [ ] 是否有重复代码可以提取为 Composable
- [ ] 是否正确使用了现有的 Composables
- [ ] 工具函数是否放在合适的位置

### 质量检查

- [ ] 无重复代码
- [ ] 无魔法值
- [ ] 无死代码（未使用变量/函数/import）
- [ ] 无空回调函数
- [ ] 无调试代码（console.log/console.debug）
- [ ] 有完整的错误处理
- [ ] 有必要的注释

---

## 安全审查

### 输入验证

- [ ] 用户输入是否有验证
- [ ] 表单提交前是否验证
- [ ] 是否正确处理了空值/null

### XSS 防护

- [ ] 是否避免使用 `v-html`（除非必要）
- [ ] 使用 `v-html` 时是否进行了 HTML 转义/过滤
- [ ] URL 参数是否进行了验证

### 权限控制

- [ ] 需要认证的页面是否设置了 `auth: true`
- [ ] 敏感操作是否有权限检查
- [ ] 按钮/菜单是否根据权限正确隐藏

---

## 样式审查

### CSS 规范

- [ ] 组件是否使用了 `scoped` 样式
- [ ] 类名命名是否遵循规范（BEM 或短横线）
- [ ] 是否使用了 CSS 变量
- [ ] 嵌套深度是否不超过 3 层

### 响应式

- [ ] 是否适配了移动端
- [ ] 是否使用了相对单位（rem/em/%）
- [ ] 图片是否有固定宽度或最大宽度

---

## 可访问性审查

- [ ] 图片是否有 `alt` 属性
- [ ] 表单控件是否有 `label`
- [ ] 按钮是否有清晰的文本
- [ ] 颜色对比度是否足够
- [ ] 是否可以使用键盘导航

---

## 性能审查

- [ ] 列表项是否使用了正确的 `key`
- [ ] 大型列表是否使用了虚拟滚动
- [ ] 图片是否使用了懒加载
- [ ] 是否合理使用了 `keep-alive`
- [ ] 是否有不必要的重新渲染

---

## 测试审查

- [ ] 核心功能是否有单元测试
- [ ] 边界条件是否有测试
- [ ] 错误场景是否有测试
- [ ] 测试是否通过

---

## 文档审查

- [ ] 代码是否有必要的注释
- [ ] 复杂逻辑是否有说明
- [ ] 公共 API 是否有文档
- [ ] 变更是否更新了更新历史

---

## Git 审查

- [ ] Commit 信息是否清晰
- [ ] 是否有不必要的文件提交（如 node_modules、dist）
- [ ] 是否有敏感信息（如密钥、密码）
- [ ] 分支命名是否符合规范
