## 1.1 命名规范

### 1.1.1 文件命名

| 类型 | 规则 | 示例 | 等级 |
|------|------|------|------|
| 组件文件 | PascalCase | `UserCard.vue`, `OrderList.vue` | 🔴 |
| 页面文件 | PascalCase + Page | `UserListPage.vue` | 🔴 |
| 组合式函数 | camelCase + use | `useUserList.ts`, `useOrderLogic.ts` | 🔴 |
| 工具函数 | camelCase | `utils.ts`, `formatters.ts` | 🔴 |
| API 文件 | camelCase + api | `user-api.ts`, `order-api.ts` | 🟢 |
| 类型定义 | camelCase + types | `user-types.ts` | 🟢 |
| 样式文件 | kebab-case | `base-styles.scss` | 🟢 |

### 1.1.2 目录命名

```
✅ 推荐：
frontend/views/          # 业务视图
frontend/components/     # 业务组件
frontend/types/          # 业务类型

❌ 避免：
frontend/Views/          # 不要大写
frontend/view/           # 不要单数（除非语义需要）
```

### 1.1.3 代码命名

```typescript
// 变量 - camelCase
const userName = 'test';
const isLoading = ref(false);

// 常量 - UPPER_SNAKE_CASE
const MAX_RETRY_COUNT = 3;
const API_TIMEOUT = 10000;

// 组件 - PascalCase
defineOptions({ name: 'UserListCard' });

// 函数 - camelCase
const handleSubmit = () => {};
const fetchData = async () => {};

// 类/接口 - PascalCase
interface UserInfo { ... }
class DataProcessor { ... }

// 枚举 - PascalCase + 成员 UPPER_SNAKE_CASE
enum OrderStatus {
  PENDING = 'pending',
  COMPLETED = 'completed',
}
```

### 1.1.4 Props/Emits 命名

```typescript
// Props - camelCase
interface Props {
  userData?: UserInfo;
  isLoading?: boolean;
}

// Emits - 动词开头，camelCase
const emit = defineEmits<{
  updateModelValue: [value: string];
  itemClick: [id: number];
  confirm: [data: any];
}>();
```

**菜单结构（自动生成）：**

```
- 首页                    # 根级页面（dashboard/index.ts）
- 业务中心                # 模块分组（business/index.ts）
  ├─ 订单中心             # 子页面（business/orders/index.ts）
  └─ 报表中心             # 子页面（business/reports/index.ts）
- 运维中心                # 模块分组（monitor/index.ts）
  └─ 运行概览             # 子页面（monitor/overview/index.ts）
- 系统管理                # 模块分组（system/index.ts）
  ├─ 用户管理             # 子页面（system/user/index.ts）
  └─ 角色管理             # 子页面（system/role/index.ts）
```

**路由生成规则：**

1. 模块配置：`type='module'` 标识为菜单分组，不生成独立路由，仅作为父级菜单
2. 页面配置：生成实际路由和子菜单项
3. 嵌套路由：子路由路径自动生成（如 `business/orders` → path: 'orders'，嵌套在 business 下）
4. `name`: 生成路由名称和菜单标签
5. `icon`: 菜单图标
6. `order`: 菜单排序（数字越小越靠前）
7. `auth`: 路由守卫（需要认证）
8. `hidden`: 是否在侧边栏隐藏

**规范要求：**
- 每个页面必须使用独立目录包裹
- 页面组件必须命名为 `Index.vue`（首字母大写）
- 每个页面目录必须包含 `index.ts` 配置文件
- 配置文件必须导出 `page`、`path`、`name` 属性
- 路由和菜单从配置文件自动生成，无需手动维护路由表
- 登录页、404、403 等系统页面需设置 `hidden: true`

## 1.6 代码组织结构规范

### 6.1 文件内部组织

**🔴 强制：文件必须按功能模块组织，避免代码混乱**

```typescript
// ✅ 推荐：清晰的功能分区
// composables/useUserList.ts

// 1. 类型定义区
interface UserListState {
  list: User[];
  loading: boolean;
  total: number;
}

interface UserListOptions {
  initialPage?: number;
  initialLimit?: number;
}

// 2. 常量定义区
const DEFAULT_PAGE_SIZE = 20;
const MAX_RETRY_COUNT = 3;

// 3. 响应式状态区
const createState = (): UserListState => ({
  list: [],
  loading: false,
  total: 0,
});

// 4. 核心逻辑区
const createLoadFunction = (state: UserListState) => {
  const loadList = async () => { /* ... */ };
  return { loadList };
};

// 5. Composable 入口
export function useUserList(options: UserListOptions = {}) {
  const state = createState();
  const { loadList } = createLoadFunction(state);
  return { ...state, loadList };
}
```

```typescript
// ❌ 避免：代码混杂无组织
export function useUserList(options: any = {}) {
  const list = ref([]);
  const loading = ref(false);
  const DEFAULT_PAGE_SIZE = 20;
  const loadList = async () => { /* ... */ };
  const total = ref(0);
  // 代码顺序混乱，难以阅读
  return { list, loading, total, loadList };
}
```

### 6.2 组件/函数复杂度控制

| 指标 | 限制 | 等级 | 说明 |
|------|------|------|------|
| 函数行数 | ≤ 50 行 | 🟢 推荐 | 超过需考虑拆分 |
| 组件行数 | ≤ 300 行 | 🟢 推荐 | 超过需考虑拆分 |
| 文件行数 | ≤ 500 行 | 🔴 强制 | 超过必须拆分 |
| 函数参数 | ≤ 4 个 | 🟢 推荐 | 超过需使用对象参数 |
| 嵌套层级 | ≤ 4 层 | 🔴 强制 | if/for/while 嵌套 |
| 圈复杂度 | ≤ 10 | 🟢 推荐 | 分支路径数量 |

**示例：函数拆分**

```typescript
// ❌ 避免：单个函数过长
const processUserData = async () => {
  // 100+ 行代码...
  // 验证数据
  // 处理数据
  // 保存数据
  // 发送通知
  // 记录日志
};

// ✅ 推荐：拆分为多个职责单一的函数
const validateUserData = (data: UserData) => { /* ... */ };
const transformUserData = (data: UserData) => { /* ... */ };
const saveUserData = (data: UserData) => { /* ... */ };
const sendNotification = (userId: number) => { /* ... */ };

const processUserData = async () => {
  validateUserData(data);
  const transformed = transformUserData(data);
  await saveUserData(transformed);
  await sendNotification(data.id);
};
```

### 6.3 代码复用检查

**🔴 强制：发现重复代码必须抽取复用**

```typescript
// ❌ 避免：重复代码（相同逻辑出现 2 次以上）
// component-a.vue
const formatDate = (date: Date) => {
  return new Intl.DateTimeFormat('zh-CN').format(date);
};

// component-b.vue
const formatDate = (date: Date) => {
  return new Intl.DateTimeFormat('zh-CN').format(date);
};

// ✅ 推荐：抽取为工具函数
// utils/formatter.ts
export const formatDate = (date: Date) => {
  return new Intl.DateTimeFormat('zh-CN').format(date);
};
```

**复用抽取优先级：**
1. 完全相同代码 → 立即抽取
2. 逻辑相同仅参数不同 → 参数化后抽取
3. 相似逻辑 → 抽象为通用模式

### 6.4 模块依赖管理

|------|------|----------|
| 语义化命名 | 变量/函数名称清晰表达意图 | 代码审查 |
| 单一职责 | 每个函数只做一件事 | 函数长度检查 |
| 消除魔法值 | 使用常量替代硬编码值 | 代码审查 |
| 消除死代码 | 删除未使用的代码 | ESLint 检查 |

```typescript
// ❌ 避免：魔法值
const status = 1;
if (status === 1) { /* ... */ }

// ✅ 推荐：使用常量
const USER_STATUS = {
  ACTIVE: 1,
  INACTIVE: 0,
  BANNED: -1,
} as const;
const status = USER_STATUS.ACTIVE;
if (status === USER_STATUS.ACTIVE) { /* ... */ }
```

### 7.2 健壮性

**🔴 强制：必须处理边界情况和错误**

```typescript
// ❌ 避免：无错误处理
const fetchData = async () => {
  const result = await api.getUserList();
  return result.rows;
};

// ✅ 推荐：完整的错误处理
const fetchData = async () => {
  try {
    const result = await api.getUserList();
    return result?.rows ?? [];
  } catch (error) {
    console.error('获取用户列表失败:', error);
    throw error;
  }
};
```

### 7.3 可测试性

**🟢 推荐：代码应该易于测试**

```typescript
// ❌ 避免：难以测试（隐式依赖）
let globalCounter = 0;
const increment = () => {
  globalCounter++;
  return globalCounter;
};

// ✅ 推荐：可测试（纯函数）
const increment = (counter: number): number => {
  return counter + 1;
};
```

**可测试代码特征：**
- 纯函数优先（相同输入=相同输出）
- 避免全局状态
- 依赖注入而非硬编码
- 副作用隔离

### 7.4 性能意识

**🟢 推荐：编写高性能代码**

| 场景 | 建议 | 说明 |
|------|------|------|
| 大列表渲染 | 使用虚拟滚动 | 避免 DOM 过多 |
| 频繁计算 | 使用 `computed` 缓存 | 避免重复计算 |
| 事件监听 | 使用防抖/节流 | 避免频繁触发 |
| 图片加载 | 使用懒加载 | 减少首屏加载 |

```typescript
// ❌ 避免：重复计算
const filteredList = () => {
  return props.list.filter(item => {
    // 每次都重新计算
    return item.status === getActiveStatus();
  });
};

// ✅ 推荐：使用 computed 缓存
const filteredList = computed(() => {
  return props.list.filter(item => {
    return item.status === getActiveStatus();
  });
});
```
