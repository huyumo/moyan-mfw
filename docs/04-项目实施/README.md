## 文档与现有 docs 目录的关系

```
moyan-mfw/
├── docs/           # 产品文档（只读，不可污染）
│   ├── core/       # 核心概念（权限、角色、架构）
│   ├── database/   # 数据库设计
│   ├── flows/      # 业务流程
│   ├── pages/      # 页面设计
│   └── api/        # API 文档（高层设计）
│
└── dev/            # 开发文档（新增目录）
    ├── 00-项目综合分析与发展规划.md  # 项目分析报告
    ├── 01-实施方案.md               # 实施计划
    ├── 02-前端开发文档.md           # 前端开发指南
    ├── 02-API 开发文档.md           # API 接口定义
    ├── 03-后端开发文档.md           # 后端开发指南
    └── README.md                    # 本文档
```

**区分原则**:
- `docs/` - 产品设计文档，描述"是什么"和"为什么"
- `dev/` - 开发实施文档，描述"怎么做"和"何时做"

## 文档维护

### 添加新文档

1. 在 `dev/` 目录下创建新文件
2. 命名格式：`编号 - 文档名称.md`
3. 更新本文档的文档清单

### 更新现有文档

1. 更新文档末尾的"更新历史"表格
2. 如影响其他文档，同步更新相关引用

### 版本控制

文档版本与项目版本关联：
- 主版本：x.0.0 - 架构性变更
- 次版本：0.x.0 - 功能增改
- 修订版：0.0.x - 文字修订
