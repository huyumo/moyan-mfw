## 1. 项目概述

### 1.1 项目定位

墨言权限管理系统（Moyan MFW）是一款面向多租户应用的**统一权限管理平台**，采用创新的**应用类型中心模式**，为企业提供细粒度、可配置的权限控制解决方案。

### 1.2 当前项目状态

- ✅ 后端核心架构设计完成（五层架构、权限池隔离）
- ✅ 数据库实体设计完成（10 张核心表）
- ✅ 基础前端框架搭建完成（packages/base-frontend）
- ✅ 业务前端应用框架整合完成（frontend/）
- ✅ E2E 测试框架搭建完成（Playwright）
- ⚠️ 业务功能模块待开发（订单中心、报表中心为演示模块）
- ⚠️ 权限管理核心页面待开发

## 3. 前端框架分析

### 3.1 基础框架（packages/base-frontend）

**核心能力**：
- 应用创建入口：`createBaseAdminApp()`
- 布局系统：`AdminLayout` + 5 个面板组件
  - `HeaderPanel` - 顶栏
  - `AsidePanel` - 侧边栏
  - `MainPanel` - 主内容区（含 Tabs）
  - `NavigationPanel` - 导航面板
  - `SettingsPanel` - 设置面板
- 路由系统：`createBaseAdminRouter()`
- 状态管理：`useLayoutStore()` (Pinia)
- 主题系统：支持多主题注册和切换

**可扩展点**：
```typescript
createBaseAdminApp({
  layout: { layoutMode, showTabs, isDark },
  navigation: { brandName, homePath },
  themes: ThemeRegistry,
  layoutExtensions: {
    headerCommon: Component,
    headerAvatar: Component,
  },
  businessRoutes: RouteRecordRaw[],
})
```

## 4. 数据库设计分析

### 4.1 核心实体（10 张表）

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| sys_app_type | 应用类型 | typeCode, multiAppEnabled |
| sys_app | 应用实例 | appTypeId, ownerId |
| sys_role | 角色 | appId, appTypeId, isBuiltin, isOwner |
| sys_permission | 权限 | permissionType, nodeType, permissionValue |
| sys_user | 用户 | username, isDeveloper |
| sys_role_permission | 角色权限关联 | roleId, permissionId, permissionValue |
| sys_app_type_permission | 权限池 | appTypeId, permissionId, permissionValue |
| sys_user_app | 用户应用关联 | userId, appId |
| sys_user_role | 用户角色关联 | userId, roleId |

### 4.2 位运算权限设计

```
权限定义 (7n = ADD|EDIT|DELETE)
    ↓ 权限池配置选择子集
权限池 (3n = ADD|EDIT)
    ↓ 角色分配选择子集
角色权限 (1n = ADD)
    ↓ 运行时验证
用户权限 = role1Value | role2Value | ...
```

**权限位定义**：
| 权限位 | 值 | 说明 |
|--------|-----|------|
| ADD | 1n | 新增 |
| EDIT | 2n | 编辑 |
| DELETE | 4n | 删除 |
| EXPORT | 8n | 导出 |
| IMPORT | 16n | 导入 |
| VIEW | 32n | 查看 |
| APPROVE | 64n | 审批 |

## 5. 核心架构分析

### 5.1 五层架构

```
┌─────────────────────────────────────────────────────────┐
│ 第一层：用户层 (User Layer)                             │
│ - sys_user, sys_user_role, sys_user_app                │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│ 第二层：应用实例选择层 (Application Instance Layer)     │
│ - sys_app                                                │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│ 第三层：权限计算层 (Permission Calculation Layer)       │
│ - 收集关联角色，计算权限并集                             │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│ 第四层：权限池层 (Permission Pool Layer)                │
│ - sys_app_type_permission                               │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│ 第五层：权限定义层 (Permission Definition Layer)        │
│ - sys_permission                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.2 应用类型中心模式

```
                    ┌─────────────────┐
                    │   AppType       │
                    │  (应用类型)      │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│      App        │ │      Role       │ │AppTypePermission│
│  (应用实例)     │ │  (应用类型角色) │ │  (权限池)       │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```
