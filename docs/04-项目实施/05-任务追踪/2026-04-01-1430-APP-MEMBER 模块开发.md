## 技术方案

### 架构设计
采用与现有模块一致的分层架构：
```
Controller → Service → Repository (TypeORM)
    ↓           ↓
  Guard      Entity
    ↓
  Decorator
```

### 关键技术点

#### 1. 应用创建时的负责人绑定
- 创建应用时，同时向 `sys_app_members` 表插入记录
- 负责人自动绑定"拥有者角色"（需要查询应用类型的拥有者角色）
- 使用事务保证原子性

#### 2. 负责人变更的级联处理
```typescript
async changeOwner(appId: string, newOwnerId: string) {
  const queryRunner = this.dataSource.createQueryRunner();
  await queryRunner.connect();
  await queryRunner.startTransaction();

  try {
    // 1. 更新 app.ownerId
    // 2. 查询原拥有者角色
    // 3. 移除原拥有者的成员记录
    // 4. 添加新拥有者的成员记录（绑定拥有者角色）
    await queryRunner.commitTransaction();
  } catch (error) {
    await queryRunner.rollbackTransaction();
    throw error;
  }
}
```

#### 3. 成员角色分配
- 全量替换模式：提交的角色 ID 列表替换原有所有角色
- 需要验证角色是否属于该应用（或应用类型的内置角色）
- 使用事务保证 `sys_user_role` 表的原子性

#### 4. 审计日志
- APP 模块：CREATE_APP、UPDATE_APP、DELETE_APP、CHANGE_OWNER
- MEMBER 模块：ADD_MEMBER、REMOVE_MEMBER、UPDATE_MEMBER_ROLES

### DTO 设计

#### CreateAppDto
```typescript
export class CreateAppDto {
  @IsString()
  @IsNotEmpty()
  appTypeId: string;

  @IsString()
  @IsNotEmpty()
  appName: string;

  @IsString()
  @IsNotEmpty()
  appCode: string;

  @IsString()
  @IsNotEmpty()
  @IsUUID()
  ownerId: string;

  @IsString()
  @IsOptional()
  appDesc?: string;

  @IsString()
  @IsOptional()
  icon?: string;

  @IsInt()
  @IsOptional()
  sortOrder?: number;
}
```

#### AddMemberDto
```typescript
export class AddMemberDto {
  @IsUUID()
  @IsNotEmpty()
  userId: string;

  @IsUUID()
  @IsOptional()
  roleId?: string; // 可选，默认分配基础角色
}
```

### 接口设计

#### APP 模块接口
| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/apps` | 创建应用 | system:app ADD |
