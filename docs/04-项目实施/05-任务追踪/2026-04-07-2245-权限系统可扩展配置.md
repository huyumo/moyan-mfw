## 任务背景

### 问题发现

在修复 PermissionValueForm 组件时，用户提出两个关键问题：

1. **为什么定义了 PERMISSION_VALUES 却不使用，而是重新定义硬编码值？**
   - PermissionValueForm.vue 中硬编码了 6 项权限（查看、添加、编辑、删除、导出、导入）
   - 与 utils/permissions.ts 中定义的 PERMISSION_VALUES 不一致（后者有 10 项）

2. **base-frontend 是基础框架，业务项目在 frontend 目录，不同业务项目的 PERMISSION_VALUES 如何扩展？**
   - 基础框架定义 10 项默认权限
   - 业务项目可能需要自定义权限（如：审批、拒绝、发布、归档等）
   - 需要支持运行时配置覆盖

### 用户需求

- ✅ 移除无用的 mergePermissions 工具函数
- ✅ PermissionValueForm 使用 PERMISSION_VALUES 动态生成权限选项
- ✅ 设计可扩展的权限配置方案，支持业务项目覆盖/扩展

## 设计方案

### 架构原则

| 原则 | 说明 |
|------|------|
| 分离默认与扩展 | DEFAULT_PERMISSION_VALUES (6 项基础) + EXTENSION_PERMISSION_VALUES (4 项扩展) |
| 运行时可覆盖 | 前端使用 window 对象存储配置，后端使用环境变量 |
| 类型安全 | 保留 TypeScript 类型推断，支持自定义权限名称 |
| UI 友好 | getPermissionOptions() 支持 labels 和 icons 自定义 |

### 前端实现

```typescript
// packages/base-frontend/src/utils/permissions.ts

// 1. 定义默认和扩展权限
export const DEFAULT_PERMISSION_VALUES = ['查看', '添加', '编辑', '删除', '导出', '导入']
export const EXTENSION_PERMISSION_VALUES = ['审批', '拒绝', '发布', '归档']

// 2. 运行时可变的 PERMISSION_VALUES
export let PERMISSION_VALUES: string[] = [...DEFAULT_PERMISSION_VALUES, ...EXTENSION_PERMISSION_VALUES]

// 3. 配置接口
export interface PermissionConfig {
  values: readonly string[]
  labels?: Record<string, string>  // 自定义显示名称
  icons?: Record<string, any>      // 自定义图标
}

// 4. 配置 API
export function setPermissionConfig(config: PermissionConfig) {
  PERMISSION_VALUES = [...config.values]
  ;(window as any).__PERMISSION_CONFIG__ = config
}

export function getPermissionConfig(): PermissionConfig {
  return (window as any).__PERMISSION_CONFIG__ || createPermissionConfig()
}

// 5. UI 渲染支持
export function getPermissionOptions(): Array<{
  name: string
  label: string
  value: number
  icon?: any
}> {
  const config = getPermissionConfig()
  return PERMISSION_VALUES.map((name) => ({
    name,
    label: config.labels?.[name] || name,
    value: Number(1n << BigInt(PERMISSION_VALUES.indexOf(name))),
    icon: config.icons?.[name],
  }))
}
```

### 后端实现

```typescript
// packages/base-backend/src/common/constants/permissions.ts

// 环境变量支持
export const PERMISSION_VALUES = process.env.PERMISSION_VALUES
  ? (process.env.PERMISSION_VALUES.split(',') as readonly string[])
  : [...DEFAULT_PERMISSION_VALUES, ...EXTENSION_PERMISSION_VALUES] as const;

// 配置对象
export const PERMISSION_CONFIG: PermissionConfig = {
  values: PERMISSION_VALUES,
};

// UI 渲染支持
export function getPermissionOptions(): Array<{
  name: string;
  label: string;
  value: number;
}> {
  return PERMISSION_VALUES.map((name) => ({
    name,
    label: name,
    value: Number(1n << BigInt(PERMISSION_VALUES.indexOf(name))),
  }));
}
```

### 组件使用


## 遇到的问题及解决

### 问题 1: PermissionGuard 500 错误

**错误现象**:
```json
{
  "code": 50000,
  "message": "requirePermissions is not iterable",
  "data": null,
  "path": "/api/permissions/tree/all?permissionType=PC"
}
```

**根本原因**:
`Reflector.getAllAndOverride()` 返回值类型可能是：
- `RequirePermissionOptions` (单个对象)
- `RequirePermissionOptions[]` (数组)
- `undefined` (无装饰器)

直接使用 `for...of` 遍历时，如果返回单个对象会报错 "not iterable"。

**修复方案**:
```typescript
// 1. 正确标注返回类型
const requirePermissions = this.reflector.getAllAndOverride<RequirePermissionOptions | RequirePermissionOptions[]>(
  REQUIRE_PERMISSION,
  [context.getHandler(), context.getClass()],
);

// 2. 统一转换为数组
const permissionsArray = Array.isArray(requirePermissions)
  ? requirePermissions
  : (requirePermissions ? [requirePermissions] : []);

// 3. 使用转换后的数组
for (const options of permissionsArray) { ... }
```

**文件**: `packages/base-backend/src/common/guards/permission.guard.ts`

## 权限位运算规则

| 索引 | 权限名称 | 位运算值 | 十进制 |
|------|----------|----------|--------|
| 0 | 查看 | 1n << 0 | 1 |
| 1 | 添加 | 1n << 1 | 2 |
| 2 | 编辑 | 1n << 2 | 4 |
| 3 | 删除 | 1n << 3 | 8 |
| 4 | 导出 | 1n << 4 | 16 |
| 5 | 导入 | 1n << 5 | 32 |
| 6 | 审批 | 1n << 6 | 64 |
| 7 | 拒绝 | 1n << 7 | 128 |
| 8 | 发布 | 1n << 8 | 256 |
| 9 | 归档 | 1n << 9 | 512 |

**API 传输格式**: 字符串（如 `"7"` 表示 `查看 + 添加 + 编辑`）
