## 根本原因分析

### 1. 文档与代码脱节

**现象**：文档定义的接口，代码未实现；代码实现的功能，文档未定义。

**原因**：
- 开发时未强制读取文档
- 文档更新滞后于代码变更
- 缺乏自动化验证机制

### 2. API 规范不统一

**现象**：API 路径缺少版本号，HTTP 方法不一致。

**原因**：
- 无全局 API 规范约束
- 后端 Controller 未使用统一前缀
- 文档设计（v1）与实际实现不一致

### 3. 类型定义不精确

**现象**：前端使用 `string` 代替枚举，`as any` 强制类型断言。

**原因**：
- `moyan-api` 生成工具类型映射错误
- 枚举类型被转换为 string
- bigint 类型未正确处理

### 4. 功能实现不完整

**现象**：核心功能标记 TODO，使用模拟数据。

**原因**：
- 开发优先级问题
- 后端接口未实现，前端用模拟数据
- 缺乏功能完整性检查
