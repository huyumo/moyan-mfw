## 场景 1：添加待开始任务（不中断）

**用户输入**：`顺便帮我做一下文档目录结构整理`

- [ ] MfwFormat 组件开发（P0）
  - 当前状态：编写代码中

## 待开始
- [ ] 文档目录结构整理（P2）← 新增 2026-03-30 14:30

- [ ] MfwUpload 组件开发（P0）

## 状态标记符号说明

| 符号 | 含义 | 使用场景 |
|------|------|----------|
| ⏸️ | 已暂停 | 任务被临时中断 |
| ▶️ | 已恢复 | 暂停的任务重新开始 |
| 🔥 | 插队 | 紧急插入的任务 |
| ← | 变更指示 | 标记变更内容 |
| ✅ | 已完成 | 任务完成 |
| ❌ | 已取消 | 任务取消 |

---

## 最佳实践建议

1. **添加任务时**：记录添加时间和原因，便于后续追溯
2. **删除任务时**：保留删除记录在"备注"中，不要完全抹去痕迹
3. **暂停任务时**：记录暂停原因，方便恢复时快速回忆上下文
4. **优先级调整时**：标记原优先级，便于了解变更历史
5. **定期归档**：任务完成后及时归档，保持 TASK.md 精简
