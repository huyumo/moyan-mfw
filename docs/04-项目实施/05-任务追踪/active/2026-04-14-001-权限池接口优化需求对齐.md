## 背景与目标

### 背景

当前权限池配置和角色权限分配两个功能使用相似的接口设计，但返回数据结构存在字段命名不一致的问题：
- 权限池接口返回体使用 `inPool: boolean` 字段
- 角色权限接口返回体使用 `assigned: boolean` 字段

这导致前端需要编写两套相似但不同的处理逻辑，增加维护成本。

### 目标

通过需求对齐会议，确保各方对优化方案达成一致，明确实施范围、技术方案和排期。

### 2. 确认数据结构变更

**讨论要点**：
- [x] `checked` 字段统一 - 返回体与请求体保持一致
- [x] `parentPermissionValue` 新增 - 用于子集验证
- [x] 类型定义更新范围

**新增字段说明**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `checked` | boolean | 统一选中状态字段（替代 inPool/assigned） |
| `parentPermissionValue` | string? | 父层位运算权限值，用于子集验证 |

**会议结论**：
- **过渡期策略**：本次迭代无需过渡期，直接替换字段名。原因：功能为内部管理后台，无外部依赖方，可一次性完成迁移
- **parentPermissionValue 实现**：由后端实现，在构建权限树时从 Permission 表或权限池配置中获取父层权限值
