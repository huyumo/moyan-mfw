## 背景与目标

### 背景

当前测试环境存在以下问题：

**问题 1：测试数据污染开发环境**

当前 `.env.test` 配置使用远程数据库：
```bash
# 当前配置（问题配置）
DB_HOST=47.109.38.249
DB_NAME=moyan_mfw
```

这导致：
- 测试运行时会删除并重建 `sys_*` 表
- 测试数据写入开发数据库
- 多人协作时数据冲突
- 开发环境数据被测试污染

**问题 2：测试完成后数据未清理**

当前 `jest.global-teardown.ts` 仅关闭数据库连接，未清理数据：
```typescript
// 当前 teardown（问题代码）
export default async function globalTeardown() {
  const connection = (global as any).TEST_DB_CONNECTION;
  if (connection && connection.isConnected) {
    await connection.close();  // 仅关闭连接，不清理数据
  }
}
```

**问题 3：缺少数据库空状态检测**

测试前未检测数据库状态，可能导致：
- 在已有数据的环境运行测试
- 测试结果不可预测
- 数据残留影响后续测试

**问题 4：缺少系统初始化流程**

数据库为空时未调用系统初始化接口：
- 无法验证真实业务流程
- 测试覆盖率受限

### 目标

建立本地隔离的测试环境，确保：
1. 测试使用本地数据库（localhost）
2. 测试数据库与开发数据库分离
3. 每次测试完成后清理所有数据
4. 测试前检测数据库是否为空
5. 数据库为空时调用系统初始化接口

### 阶段 3：测试 Teardown 脚本优化

**任务 3.1：添加数据清理逻辑**

修改 `jest.global-teardown.ts`：
```typescript
export default async function globalTeardown() {
  const connection = (global as any).TEST_DB_CONNECTION;
  
  if (connection && connection.isConnected) {
    try {
      // 新增：清理所有 sys_* 表数据
      const queryRunner = connection.createQueryRunner();
      await queryRunner.query('SET FOREIGN_KEY_CHECKS = 0');
      
      const tables = await queryRunner.query(`
        SELECT TABLE_NAME FROM information_schema.TABLES
        WHERE TABLE_SCHEMA = ? AND TABLE_NAME LIKE 'sys_%'
      `, [process.env.DB_NAME]);
      
      for (const table of tables) {
        await queryRunner.query(`TRUNCATE TABLE \`${table.TABLE_NAME}\``);
      }
      
      await queryRunner.query('SET FOREIGN_KEY_CHECKS = 1');
      await queryRunner.release();
      
      console.log('[GlobalTeardown] All test data cleared');
    } finally {
      await connection.close();
    }
  }
}
```
