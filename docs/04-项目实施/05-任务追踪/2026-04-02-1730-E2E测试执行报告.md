## 2. 失败原因分析

### 2.1 根本原因

| 类别 | 原因 | 影响范围 | 优先级 |
|------|------|----------|--------|
| **数据问题** | 测试数据库缺少用户数据 | 所有系统管理测试 | P0 |
| **路径错误** | `/sys/app-instance` 应为 `/sys/app` | 应用实例测试 | P1 |
| **选择器问题** | 登出按钮选择器不匹配 | AUTH-015 | P2 |
| **功能未开发** | 应用切换、无权限场景 | AUTH-012/13/14 | P3 |

### 2.2 详细分析

#### 2.2.1 数据问题（最关键）

**现象**：系统管理页面加载成功，但表格显示"暂无数据"

**根因**：
- 测试数据库 `sys_users` 表可能为空或数据未同步
- 后端服务在测试执行期间意外关闭（端口3000被占用）

**影响**：
- 所有依赖数据列表的测试失败
- 无法验证表格行、分页、筛选等功能

**解决方案**：
1. 确保测试数据库有足够的测试数据
2. 修复后端服务启动逻辑（端口冲突处理）
3. 添加测试数据初始化步骤

#### 2.2.2 路径错误

**现象**：应用实例管理测试无法找到页面

**根因**：测试用例使用 `/sys/app-instance`，实际路由为 `/sys/app`

**修复**：已修正 `tests/e2e/sys/app-instance.spec.ts`

#### 2.2.3 登出按钮选择器

**现象**：AUTH-015 测试失败，找不到登出按钮

**根因**：UI 中的登出按钮在选择器中未找到匹配

**当前 UI 结构**：
```yaml
button "Open user menu" [ref=e45] [cursor=pointer]:
  - generic [ref=e46]: A
  - strong [ref=e47]: Admin
```

需要点击用户菜单后再点击登出，但登出按钮元素未在快照中显示。
