# API 集成测试 - 用例

> **项目名称**: 墨焱框架 (Moyan MFW)
> **测试范围**: 基础设施权限系统 - 后端 API 接口
> **测试类型**: 集成测试 (Integration Test)
> **测试工具**: Jest + Supertest + TestContainer
> **数据库**: MySQL 8.0 (测试专用)
> **版本**: 1.0.0
> **创建日期**: 2026-04-01

---

## 测试说明

### 测试环境

| 配置项 | 值 |
|--------|-----|
| 基础 URL | `http://localhost:3000/api` |
| 测试数据库 | `test_moyan_mfw` |
| 测试用户账号 | 见各模块测试用例 |
| Token 类型 | JWT Bearer Token |

### 测试数据说明

- 本测试用例使用独立的测试数据库，不考虑数据污染问题
- 测试数据由 TestContainer 在每次测试前自动创建和清理
- 种子数据：包含 2 个应用类型、25 个权限、3 个角色、2 个用户

### 默认测试账号

| 账号 | 密码 | 角色 | 权限说明 |
|------|------|------|----------|
| `admin` | `Admin@123` | 超级管理员 | 拥有所有权限 |
| `test` | `Test@123` | 普通用户 | 仅查看权限 |

### 测试用例编号规则

```
[模块]-[序号]-[子序号]
示例：AUTH-01-01 = 认证模块第 1 个用例的第 1 个子用例
```

### 位运算权限值说明

| 权限位 | 十进制 | 二进制 | 说明 |
|--------|--------|--------|------|
| ADD | 1 | 00000001 | 新增 |
| EDIT | 2 | 00000010 | 编辑 |
| DELETE | 4 | 00000100 | 删除 |
| EXPORT | 8 | 00001000 | 导出 |
| IMPORT | 16 | 00010000 | 导入 |
| VIEW | 32 | 00100000 | 查看 |
| APPROVE | 64 | 01000000 | 审批 |
| REJECT | 128 | 10000000 | 拒绝 |

### AUTH-01: 用户登录

#### AUTH-01-01: 使用用户名登录成功

**场景**: 用户使用正确的用户名和密码登录

**前置条件**:
- 用户 `admin` 存在且状态正常

**请求**:
```http
POST /api/auth/login
Content-Type: application/json

{
  "identifier": "admin",
  "password": "Admin@123"
}
```

**预期响应**:
```json
{
  "code": 0,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
    "expiresIn": 7200,
    "user": {
      "id": "uuid",
      "username": "admin",
      "nickname": "管理员",
      "avatar": "https://..."
    }
  },
  "message": "登录成功"
}
```

**断言**:
- 状态码 = 200
- code = 0 (成功)
- token 存在且不为空
- refreshToken 存在且不为空
- expiresIn = 7200
- user.id 存在
- user.username = "admin"

**请求**:
```http

{
  "username": "newuser001",
  "password": "NewUser@123",
  "nickname": "新用户 001",
```

