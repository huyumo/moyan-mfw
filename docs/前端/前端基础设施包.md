# 前端基础设施包 (base-frontend)

## 概述

`moyan-mfw-base-frontend` 是框架的核心前端包，封装了管理后台的通用能力，包括组件库、布局系统、路由管理、状态管理、权限体系、主题系统等。业务前端应用通过消费此包来快速搭建管理后台。

- 包名：`moyan-mfw-base-frontend`
- 路径：`packages/base-frontend/`
- 导出方式：ESM + CJS + 类型声明

## 应用创建

### createBaseAdminApp

创建管理后台应用的核心函数，完成路由、状态、导航初始化。

```typescript
import { createBaseAdminApp } from 'moyan-mfw-base-frontend';

const admin = createBaseAdminApp({
  title: '应用标题',
  routes: businessRoutes,
  layout: {
    layoutMode: 'dual',     // sidebar | top | dual
    showTabs: true,
    colorMode: 'system',    // light | dark | system
    themePackage: 'default',
  },
  navigation: {
    brandName: '品牌名称',
    brandTagline: '品牌标语',
    homePath: '/dashboard',
  },
  layoutExtensions: {
    headerCommon: HeaderCommonComponent,
    headerAvatar: HeaderAvatarComponent,
  },
});

admin.mount('#app');
```

### 启动选项

| 参数 | 类型 | 说明 |
|------|------|------|
| `title` | string | 页面标题后缀 |
| `routes` | RouteRecordRaw[] | 业务路由配置 |
| `layout` | Partial\<LayoutStyleConfig\> | 布局样式配置（含 `searchTrigger`、`keepAlive` 等字段） |
| `navigation` | Partial\<AdminNavigationConfig\> | 导航配置 |
| `layoutExtensions` | LayoutExtensionComponents | 布局扩展组件 |
| `loginExtensions` | LoginExtensionComponents | 登录页扩展组件 |
| `pinia` | Pinia | 外部注入的状态实例 |

## 路由系统

### 自动扫描模式

框架使用 `import.meta.glob` 自动扫描 `views/` 目录下的 `index.{ts,tsx}` 配置文件生成路由。

#### 页面配置 (index.ts)

```typescript
import Dashboard from './dashboard.vue';
export default {
  page: Dashboard,
  path: 'dashboard',
  name: '看板',
  icon: 'DataBoard',
  auth: true,
  permissions: ['添加', '编辑'],
};
```

#### 模块配置 (index.ts)

```typescript
export default {
  type: 'module',
  name: '系统管理',
  icon: 'Setting',
  order: 10,
};
```

#### 权限自动计算

使用 `definePageConfig` 自动将权限名称列表转为位运算值：

```typescript
import { definePageConfig } from 'moyan-mfw-base-frontend';

export default definePageConfig({
  page: UserPage,
  path: 'user',
  name: '用户管理',
  permissions: ['添加', '编辑', '删除'],
  // permissionValue 自动计算为 7n (1|2|4)
});
```

### 路由守卫

`setupRouteGuard` 注册全局前置守卫：

1. 检查 Token 是否存在 → 无 Token 跳转 `/login`
2. 检查路由是否需要认证 (`meta.requiresAuth`)
3. 首次进入时初始化认证状态 (`initializeAuth`)
4. 检查权限菜单是否加载
5. 根据权限菜单判断路由是否可访问

### 路由合并

基包路由和业务路由自动合并，业务路由优先覆盖同名路径。

## 状态管理

### auth-store

认证状态管理，提供完整的登录/登出/权限菜单流程。

| 状态 | 类型 | 说明 |
|------|------|------|
| `token` | string | 访问令牌 |
| `refreshToken` | string | 刷新令牌 |
| `user` | UserInfo | 当前用户信息 |
| `apps` | AppInstance[] | 用户可访问的应用列表 |
| `currentApp` | AppInstance | 当前选中的应用 |
| `permissionMenu` | PermissionMenuItem[] | 权限菜单树 |
| `permissionValueMap` | Record\<string, string\> | 权限值映射 |
| `isAuthenticated` | boolean | 是否已认证（计算属性） |
| `needSelectApp` | boolean | 是否需要选择应用（计算属性） |

核心方法：`login`、`logout`、`refreshAccessToken`、`fetchUserInfo`、`fetchUserApps`、`selectApp`、`loadPermissions`、`initializeAuth`

### layout-store

布局状态管理，控制布局模式、侧边栏、标签页、导航等。

| 状态 | 类型 | 说明 |
|------|------|------|
| `styleConfig` | LayoutStyleConfig | 布局样式配置 |
| `navigation` | AdminNavigationConfig | 导航配置 |
| `showSidebar` | boolean | 侧边栏显示状态 |
| `showSettingsPanel` | boolean | 设置面板显示状态 |

## 权限体系

### 权限位运算

| 名称 | 位值 |
|------|------|
| 添加 | 1n (1 << 0) |
| 编辑 | 2n (1 << 1) |
| 删除 | 4n (1 << 2) |
| 导出 | 8n (1 << 3) |
| 导入 | 16n (1 << 4) |
| 审批 | 32n (1 << 5) |
| 拒绝 | 64n (1 << 6) |
| 发布 | 128n (1 << 7) |
| 归档 | 256n (1 << 8) |

### 权限工具函数

| 函数 | 说明 |
|------|------|
| `buildPerValue(names)` | 根据权限名称数组构建位运算值 |
| `parsePerValue(value)` | 根据位运算值解析为权限名称数组 |
| `hasPermission(value, name)` | 检查是否包含指定权限 |
| `getPermValue(name)` | 获取单个权限位值 |
| `getPermissionOptions(parentValue?)` | 获取权限选项（用于UI展示） |
| `registerPermissionValues(values)` | 注册额外的权限值 |
| `createBusinessPageConfigFn(permissions)` | 创建业务层页面配置函数 |

### 权限指令

```vue
<template>
  <button v-permission="'添加'">新增</button>
  <button v-permission="['编辑', '删除']">操作</button>
</template>
```

### 权限钩子

```typescript
import { usePermission } from 'moyan-mfw-base-frontend';

const { hasPermission, checkPermission } = usePermission();
```

## 业务前端应用

业务前端应用（`frontend/`）消费 `base-frontend` 包，结构精简：

```
frontend/src/
├── components/Layout/         # 布局扩展组件
├── views/
│   ├── business/              # 业务页面（订单、报表）
│   ├── dashboard/             # 看板
│   └── monitor/               # 监控
├── main.ts                    # 入口
├── router.ts                  # 路由（自动扫描）
└── themes.ts                  # 主题配置
```
