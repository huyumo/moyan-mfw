## 创建项目

```bash
# 在 monorepo 根目录下执行
cd e:\Moyan\moyan\moyan-mfw-workspace\workspace04\moyan-mfw

# 交互式创建业务项目（按提示输入项目信息）
node packages/cli/dist/index.js create business my-project
```

交互式提示及默认值：

| 步骤 | 提示 | 默认值 | 说明 |
|------|------|--------|------|
| 1 | 显示名称 | `MyProject` | PascalCase 自动推导 |
| 2 | 描述 | `MyProject 业务项目` | 项目的简要描述 |
| 3 | 后端端口 | `3000` | NestJS 服务监听端口 |
| 4 | 前端端口 | `5173` | Vite dev server 端口 |

生成完成后输出 28 个文件，目录结构：

```
my-project/
├── package.json              # 根聚合脚本
├── tsconfig.base.json        # 共享 TS 配置
├── backend/                  # 业务后端
│   ├── .env                  # 数据库/端口配置
│   └── src/
│       ├── main.ts           # createBaseBackendApp() 入口
│       ├── app.modules.ts    # 模块聚合（在此添加扩展）
│       └── permissions.ts    # 业务权限装饰器
├── frontend/                 # 业务前端
│   └── src/
│       ├── main.ts           # createBaseAdminApp() 入口
│       ├── router.ts         # 自动路由扫描
│       └── views/dashboard/  # 内置仪表盘
└── shared/                   # 业务共享层
    └── src/
        ├── index.ts
        └── permissions.ts    # {NAME}_PERMISSION_VALUES
```

## 配置数据库

### 1. 创建数据库

```bash
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS my_project CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 2. 编辑环境变量

修改 `backend/.env`：

```env
# 数据库配置
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=your-password
DB_NAME=my_project

# JWT 配置
JWT_SECRET=your-jwt-secret-change-in-production
JWT_EXPIRES_IN=7d

# 服务端口
PORT=3000
```

> **注意**：`DB_NAME` 必须与创建的数据库名一致（不含特殊字符，建议用下划线替代连字符）。

## 安装依赖与构建

```bash
# 安装所有依赖（首次使用 --no-frozen-lockfile）
pnpm install --no-frozen-lockfile

# 构建依赖链：shared → base → backend
pnpm --filter my-project-shared run build
pnpm --filter my-project-backend run build
```

构建成功后会生成 `backend/dist/` 输出目录。

> **提示**：`pnpm --filter my-project-backend run build` 会自动执行 `prebuild` 脚本，先构建 shared 和 base 包。

## 首次初始化

第一次启动时数据库无表，需要运行安装接口：

```bash
# 查看安装状态
curl http://localhost:3000/api/install/status

# 执行初始化安装（创建所有表 + 种子数据）
curl -X POST http://localhost:3000/api/install/init \
  -H "Content-Type: application/json" \
  -d '{
    "adminUsername": "admin",
    "adminPassword": "admin123",
    "adminNickname": "超级管理员"
  }'
```

初始化完成后会自动创建：

- 所有系统表（`sys_users`、`sys_roles`、`sys_permissions` 等）
- 超级管理员账号
- 默认角色和权限数据
- 应用类型配置

### 验证安装结果

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'
```

预期返回 JWT token：

```json
{
  "code": 0,
  "data": {
    "accessToken": "eyJhbGciOi...",
    "refreshToken": "eyJhbGciOi..."
  }
}
```

## 权限功能验证

### 1. 获取当前用户信息与权限

```bash
TOKEN="your-access-token"

curl -X POST http://localhost:3000/api/auth/userinfo \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN"
```

响应中包含 `permissions` 字段，验证权限树是否正确加载。

### 2. 查看权限值列表

```bash
curl http://localhost:3000/api/permission-values \
  -H "Authorization: Bearer $TOKEN"
```

### 3. 查看角色列表

```bash
curl http://localhost:3000/api/roles \
  -H "Authorization: Bearer $TOKEN"
```

### 4. 查看应用类型

```bash
curl http://localhost:3000/api/app-types \
  -H "Authorization: Bearer $TOKEN"
```

### 5. 同步权限标签

在 `shared/src/permissions.ts` 中添加自定义权限后，调用同步接口：

```bash
curl -X POST http://localhost:3000/api/auth/sync-permissions \
  -H "Authorization: Bearer $TOKEN"
```

此操作会将 `{NAME}_PERMISSION_VALUES` 中定义的权限标签同步到 `sys_permission_values` 表。

### 权限验证链路

```
shared/src/permissions.ts  ← 定义权限标签
    ↓
backend/src/permissions.ts ← createBusinessPermissionDecorator() 创建类型安全的 @Permission() 装饰器
    ↓
main.ts → permissionValues: [] ← 传入 createBaseBackendApp，自动同步到 DB
    ↓
Controller → @Permission('添加') ← 控制器方法上加权限守卫
    ↓
AuthGuard + PermissionGuard ← 请求时逐层校验 JWT → 位运算权限
```

## 集成扩展模块

在 `backend/src/app.modules.ts` 中引入扩展：

```typescript
import { Module } from '@nestjs/common';
import AdModule from 'moyan-mfw-extension-ad/backend';

@Module({
  imports: [AdModule],
})
export class AppModule {}
```

在 `backend/src/main.ts` 中注册 Swagger 分组和权限值：

```typescript
import { AdModule, AD_EXTENSION_PERMISSION_VALUES } from 'moyan-mfw-extension-ad/backend';

const swaggerGroups: SwaggerGroupConfig[] = [
  { name: 'ad-extension', title: '广告管理API文档', include: [AdModule] },
];

const app = await createBaseBackendApp({
  modules: [AppModule],
  swagger: swaggerGroups,
  permissionValues: [...AD_EXTENSION_PERMISSION_VALUES],
});
```

## 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| `TS5069: Option 'declarationMap' cannot be specified without 'declaration'` | shared tsconfig 缺少 declaration 字段 | 在 `shared/tsconfig.json` 的 `compilerOptions` 中添加 `"declaration": true` |
| `Table 'xxx.sys_app_types' doesn't exist` | 数据库未初始化 | 调用 `POST /api/install/init` 创建表结构 |
| `ERR_PNPM_OUTDATED_LOCKFILE` | lockfile 与实际依赖不匹配 | 使用 `pnpm install --no-frozen-lockfile` |
| 数据库连接失败 | MySQL 未启动或配置错误 | 检查 `backend/.env` 中的 DB_HOST/DB_PORT/DB_USERNAME/DB_PASSWORD |
| `test-shop-shared has no exported member 'BUSINESS_PERMISSION_VALUES'` | 权限导入名不匹配 | 应导入 `{NAME}_PERMISSION_VALUES`（如 `TESTSHOP_PERMISSION_VALUES`） |
| 端口被占用 | 3000 端口已被使用 | 修改 `backend/.env` 中的 `PORT` 值 |
| 权限同步失败 | `sys_permission_values` 表不存在 | 先执行 `POST /api/install/init` 创建表 |
