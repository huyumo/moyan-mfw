# API 接口文档

## 概述

所有 API 统一使用 `/api` 前缀，返回统一响应格式：

```json
{
  "code": 200,
  "data": {},
  "message": "success",
  "timestamp": "2026-04-25T10:00:00.000Z"
}
```

## 认证模块 (/api/auth)

| 方法 | 路径 | 认证 | 功能 |
|------|------|------|------|
| POST | `/login` | 否 | 用户登录 |
| POST | `/refresh` | 否 | 刷新 Token |
| POST | `/logout` | 否 | 退出登录 |
| POST | `/register` | 否 | 用户自注册 |
| GET | `/check-availability` | 否 | 检查用户名/邮箱/手机号可用性 |
| POST | `/userinfo` | 是 | 获取当前用户信息 |
| GET | `/apps` | 是 | 获取用户可访问的应用列表 |
| GET | `/permissions` | 是 | 获取用户权限菜单（需 appId） |
| POST | `/change-password` | 是 | 修改密码 |
| POST | `/sync-permissions` | 是 | 同步用户权限 |

## 用户管理 (/api/users)

权限编码：`pc_root:sys:user`

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/` | 添加 | 创建用户 |
| POST | `/admin-create` | 添加 | 管理员创建用户（默认密码） |
| GET | `/` | - | 分页查询用户列表 |
| GET | `/find-one` | - | 精确查找用户 |
| GET | `/:id` | - | 根据 ID 查询用户 |
| PUT | `/:id` | 编辑 | 更新用户 |
| DELETE | `/:id` | 删除 | 删除用户（软删除） |
| PUT | `/:id/status` | 编辑 | 更新用户状态 |
| POST | `/:id/reset-password` | 编辑 | 重置用户密码 |

## 权限管理 (/api/permissions)

权限编码：`pc_root:sys:permission-pc` OR `pc_root:sys:permission`

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/` | 添加 | 创建权限 |
| GET | `/` | - | 分页查询权限列表 |
| GET | `/tree/all` | - | 查询所有权限树 |
| GET | `/tree` | - | 获取指定父节点的权限树 |
| GET | `/:id` | - | 根据 ID 查询权限 |
| PUT | `/:id` | 编辑 | 更新权限 |

## 分页查询参数

所有列表查询接口支持统一的分页参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` | number | 页码（从1开始） |
| `pageSize` | number | 每页数量 |
| `sortField` | string | 排序字段 |
| `sortOrder` | string | 排序方向（ASC/DESC） |

分页响应格式：

```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 0,
    "page": 1,
    "pageSize": 10
  }
}
```
