# 系统架构核心概念

> 5 分钟理解应用类型中心模式与五层架构设计

---

## 1. 应用类型中心模式

**核心设计理念**: 应用类型 (`sys_app_type`) 是整个权限系统的中心节点。

```
                    ┌─────────────────┐
                    │   AppType       │
                    │  (应用类型)     │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│      App        │ │      Role       │ │AppTypePermission│
│  (应用实例)     │ │  (应用类型角色) │ │  (权限池)       │
└────────┬────────┘ └────────┬────────┘ └────────┬────────┘
         │                   │                   │
         ▼                   ▼                   │
┌─────────────────┐ ┌─────────────────┐         │
│    UserApp      │ │  RolePermission │         │
│  (用户绑定)     │ │  (角色权限)     │         │
└─────────────────┘ └────────┬────────┘         │
                             │                  │
                             └────────┬─────────┘
                                      │
                                      ▼
                             ┌─────────────────┐
                             │   Permission    │
                             │    (权限)       │
                             └─────────────────┘
```

**关键关系**:
- `AppType → App`: 1:N，一个应用类型可包含多个应用实例
- `AppType → Role`: 1:N，通过 `sys_role.appTypeId` + `isBuiltin=1` 关联内置角色
- `AppType → AppTypePermission`: 1:N，定义权限池

## 2. 五层架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│ 第一层：用户层 (User Layer)                                     │
│ - sys_user: 用户基本信息                                         │
│ - sys_user_role: 用户直接绑定的角色                              │
│ - sys_user_app: 用户绑定的应用                                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第二层：应用实例选择层 (Application Instance Layer)             │
│ - 用户选择要访问的具体应用实例                                    │
│ - sys_app: 应用实例信息                                          │
│ - sys_user_app: 用户可访问的应用范围                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第三层：权限计算层 (Permission Calculation Layer)               │
│ - 收集用户所有关联角色（直接绑定 + 应用绑定 + 应用类型绑定）      │
│ - 计算权限并集，合并 permissionValue（位运算 OR）                  │
│ - sys_role: 角色定义                                             │
│ - sys_role_permission: 角色权限关联                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第四层：权限池层 (Permission Pool Layer)                        │
│ - 定义应用类型可用的权限范围                                     │
│ - 约束角色权限的选择范围                                         │
│ - sys_app_type_permission: 应用类型权限池                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第五层：权限定义层 (Permission Definition Layer)                │
│ - 定义所有可用的权限节点和操作                                   │
│ - 权限树形结构（MENU → PAGE → permissionValue）                    │
│ - sys_permission: 权限定义表                                     │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 权限隔离机制

### 3.1 应用类型级别隔离

```
┌─────────────────────────────────────────────────────────────┐
│                      应用类型 A (OA 系统)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  权限池 A    │  │  内置角色   │  │  应用级角色  │         │
│  │ [P1,P2,P3]  │  │  A1,A2     │  │  A-app1    │         │
│  │ [pcA1,pcA2] │  │ [P1,P2]     │  │ [P2,P3]     │         │
│  │             │  │ [pcA1]      │  │ [pcA2]      │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                      应用类型 B (CRM 系统)                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  权限池 B    │  │  内置角色   │  │  应用级角色  │         │
│  │ [P4,P5,P6]  │  │  B1,B2     │  │  B-app1    │         │
│  │ [pcB1,pcB2] │  │ [P4,P5]     │  │ [P5,P6]     │         │
│  │             │  │ [pcB1]      │  │ [pcB2]      │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

**隔离规则**:
- 权限 P1,P2,P3 仅在应用类型 A 中可用
- 权限 P4,P5,P6 仅在应用类型 B 中可用
- 不同应用类型的角色权限互不影响

## 4. 数据流向总览

```
用户登录
    │
    ▼
选择应用实例 (sys_app)
    │
    ▼
收集关联角色 ─────────────────────────────────┐
    ├── 直接绑定的角色 (sys_user_role)        │
    ├── 通过应用绑定的角色                     │
    └── 通过应用类型绑定的角色 (内置角色)      │
    │                                         │
    ▼                                         │
计算权限并集                                  │
    │                                         │
    ▼                                         │
验证权限池约束 ◄──────────────────────────────┘
    │
    ▼
生成用户最终权限
    │
    ▼
前端菜单渲染 + 按钮显示控制
    │
    ▼
后端接口权限验证
```

## 5. 关键设计决策

| 设计决策 | 理由 |
|----------|------|
| 应用类型中心模式 | 支持多租户、权限隔离、统一配置 |
| 权限池隔离 | 不同应用类型的权限独立管理，避免权限污染 |
| permissionValue 三层数据流 | 权限定义 → 权限池配置 → 角色分配，层层约束 |
| 拥有者两处记录 | `sys_app.ownerId` 用于业务归属，`sys_user_app` 用于统一权限计算 |
