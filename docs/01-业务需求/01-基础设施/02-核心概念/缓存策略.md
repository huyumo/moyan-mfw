# 缓存策略设计

> 版本：1.0.0
> 最后更新：2026-03-28

---

## 1. 概述

缓存系统是提升莫言权限管理系统性能的关键组件，通过减少对数据库的重复查询，显著降低响应时间和数据库负载。

### 设计目标

- **高性能**：热点数据命中率 > 95%，平均响应时间 < 10ms
- **一致性**：缓存与数据库数据最终一致，不一致窗口 < 1 秒
- **高可用**：缓存故障不影响主业务流程，自动降级到数据库
- **易维护**：缓存键命名规范、过期策略清晰、监控完善

```
                              ▼
┌─────────────────────────────────────────────────────────────┐

│ - 权限树结构 (1 小时 TTL)                                   │
│ - 角色权限映射 (30 分钟 TTL)                                │

└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐

│ - 持久化存储                                                │

└─────────────────────────────────────────────────────────────┘
```

### 2.2 缓存选型

| 缓存类型 | 技术选型 | 适用场景 |
|----------|----------|----------|
| 客户端缓存 | localStorage | 用户专属、不常变更的数据 |
| 应用层缓存 | Redis | 共享数据、高频查询数据 |
| 进程内缓存 | Node.js Map / Java Caffeine | 配置类、极少变更的数据 |

## 3. 缓存策略

### 3.1 Cache-Aside Pattern（旁路缓存）

**读操作**:

```typescript
async function getUserPermissions(userId: string) {
  const cacheKey = `user:perm:${userId}`

  // 1. 先查缓存
  const cached = await redis.get(cacheKey)
  if (cached) {
    return JSON.parse(cached)
  }

  // 2. 缓存未命中，查数据库
  const dbResult = await db.query(
    `SELECT p.*, rp.permissionValue
     FROM sys_permission p
     JOIN sys_role_permission rp ON p.id = rp.permissionId
     JOIN sys_user_role ur ON ur.roleId = rp.roleId
     WHERE ur.userId = ? AND p.permStatus = 1`,
    [userId]
  )

  // 3. 写入缓存（30 分钟过期）
  await redis.setex(cacheKey, 1800, JSON.stringify(dbResult))

  return dbResult
}
```

**写操作**:

```typescript
async function assignRolePermission(roleId: string, permissions: any[]) {
  const transaction = await db.beginTransaction()

  try {
    // 1. 先更新数据库
    await transaction.query(
      'DELETE FROM sys_role_permission WHERE roleId = ?',
      [roleId]
    )
    await transaction.query(
      'INSERT INTO sys_role_permission (roleId, permissionId, permissionValue) VALUES ?',
      [permissions.map(p => [roleId, p.permissionId, p.permissionValue])]
    )

    await transaction.commit()

    // 2. 删除相关缓存（缓存失效）
    await invalidateRolePermissionCache(roleId)

  } catch (error) {
    await transaction.rollback()
    throw error
  }
}

async function invalidateRolePermissionCache(roleId: string) {
  // 删除角色权限缓存
  await redis.del(`role:perm:${roleId}`)

  // 删除绑定该角色的所有用户的权限缓存
  const users = await db.query(
    'SELECT userId FROM sys_user_role WHERE roleId = ?',
    [roleId]
  )
  for (const user of users) {
    await redis.del(`user:perm:${user.userId}`)
  }
}
```

### 3.2 缓存键设计

**命名规范**: `{业务}:{子业务}:{标识符}:{字段}`

| 缓存键 | 说明 | 过期时间 |
|--------|------|----------|
| `token:{token}` | Token 验证信息 | 2 小时 |
| `user:info:{userId}` | 用户基本信息 | 30 分钟 |
| `user:perm:{userId}` | 用户权限列表 | 30 分钟 |
| `user:apps:{userId}` | 用户可访问应用列表 | 30 分钟 |
| `role:perm:{roleId}` | 角色权限配置 | 30 分钟 |
| `perm:tree:{appTypeId}:{type}` | 应用类型权限树 | 1 小时 |
| `perm:id:{permCode}` | permCode→permissionId 映射 | 1 小时 |
| `pool:{appTypeId}` | 应用类型权限池配置 | 1 小时 |

### 3.3 过期策略

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| Token | 2 小时 | 与 Token 有效期一致 |
| RefreshToken | 7 天 | 与 RefreshToken 有效期一致 |
| 用户权限 | 30 分钟 | 平衡性能与一致性 |
| 权限树 | 1 小时 | 变更频率低，可较长 |
| permCode 映射 | 1 小时 | 极少变更 |
| 配置数据 | 24 小时 | 几乎不变更 |

**主动失效场景**:
- 权限配置变更 → 失效相关用户、角色缓存
- 角色分配变更 → 失效相关用户缓存
- 用户信息变更 → 失效该用户所有缓存

## 4. 缓存一致性

### 4.1 一致性保证

| 场景 | 一致性策略 | 不一致窗口 |
|------|-----------|-----------|
| 权限配置变更 | 主动失效 + 延迟双删 | < 1 秒 |
| 角色分配变更 | 主动失效 | < 1 秒 |
| 用户信息变更 | 主动失效 | < 1 秒 |
|  Token 刷新 | 原子替换 | 0 |

### 4.2 延迟双删策略

```typescript
async function updatePermissionPool(appTypeId: string, config: any) {
  // 1. 先删除缓存
  await redis.del(`pool:${appTypeId}`)

  // 2. 更新数据库
  await db.query('UPDATE sys_app_type SET permissionPool = ? WHERE id = ?', [config, appTypeId])

  // 3. 延迟 500ms 再次删除（应对主从同步延迟）
  setTimeout(async () => {
    await redis.del(`pool:${appTypeId}`)
  }, 500)
}
```

### 4.3 缓存降级

```typescript
async function getUserPermissionsWithFallback(userId: string) {
  try {
    // 尝试从缓存获取
    const cached = await redis.get(`user:perm:${userId}`)
    if (cached) {
      return JSON.parse(cached)
    }
  } catch (error) {
    // 缓存故障，记录日志，降级到数据库
    logger.error('Redis 故障，降级到数据库查询', { error, userId })
  }

  // 数据库查询
  const dbResult = await getUserPermissionsFromDB(userId)

  // 尝试回写缓存（失败不影响主流程）
  try {
    await redis.setex(`user:perm:${userId}`, 1800, JSON.stringify(dbResult))
  } catch (error) {
    logger.warn('缓存写入失败', { error, userId })
  }

  return dbResult
}
```

## 5. 性能优化

### 5.1 批量查询

```typescript
// 反模式：N+1 查询
for (const roleId of roleIds) {
  const perm = await redis.get(`role:perm:${roleId}`)
  // ...
}

// 推荐：批量查询
const keys = roleIds.map(id => `role:perm:${id}`)
const perms = await redis.mget(keys)
```

### 5.2 管道（Pipeline）

```typescript
// 反模式：串行操作
await redis.setex(`user:info:${userId}`, 1800, userInfo)
await redis.setex(`user:perm:${userId}`, 1800, userPerm)
await redis.setex(`user:apps:${userId}`, 1800, userApps)

// 推荐：管道批量操作
const pipeline = redis.pipeline()
pipeline.setex(`user:info:${userId}`, 1800, userInfo)
pipeline.setex(`user:perm:${userId}`, 1800, userPerm)
pipeline.setex(`user:apps:${userId}`, 1800, userApps)
await pipeline.exec()
```

### 5.3 热点数据预加载

```typescript
// 系统启动时预加载热点数据
async function preloadHotData() {
  const appTypes = await db.query('SELECT * FROM sys_app_type WHERE typeStatus = 1')

  for (const appType of appTypes) {
    const permTree = await buildPermissionTree(appType.id)
    await redis.setex(
      `perm:tree:${appType.id}:PC`,

      JSON.stringify(permTree.pcTree)
    )
  }
}
```

## 7. 最佳实践

### 7.1 缓存什么

✅ **适合缓存的数据**:
- 频繁读取、少变更的数据（权限树、配置数据）
- 计算成本高、结果稳定的数据（用户权限计算）
- 用户专属、隔离的数据（用户信息、Token）

❌ **不适合缓存的数据**:
- 实时性要求极高的数据（如库存）
- 变更频繁、缓存命中率低的数据
- 数据量大、访问频率低的数据

### 7.2 缓存陷阱

| 陷阱 | 说明 | 避免方式 |
|------|------|----------|
| 缓存穿透 | 查询不存在的数据，缓存不命中直接打到数据库 | 布隆过滤器、缓存空值 |
| 缓存击穿 | 热点数据失效瞬间，大量请求打到数据库 | 互斥锁、永不过期 + 逻辑过期 |
| 缓存雪崩 | 大量缓存同时失效 | 随机过期时间、分级缓存 |

### 7.3 代码示例

```typescript
// 防止缓存穿透：缓存空值
async function getPermissionById(permissionId: string) {
  const cacheKey = `perm:id:${permissionId}`

  const cached = await redis.get(cacheKey)
  if (cached !== null) {  // null 表示缓存了空值
    return cached ? JSON.parse(cached) : null
  }

  const dbResult = await db.query('SELECT * FROM sys_permission WHERE id = ?', [permissionId])

  // 缓存空值（5 分钟过期，防止穿透）
  const ttl = dbResult ? 3600 : 300
  await redis.setex(cacheKey, ttl, JSON.stringify(dbResult || ''))

  return dbResult
}

// 防止缓存击穿：互斥锁
async function getHotPermissionTree(appTypeId: string) {
  const cacheKey = `perm:tree:${appTypeId}:PC`

  const cached = await redis.get(cacheKey)
  if (cached) {
    return JSON.parse(cached)
  }

  // 尝试获取分布式锁
  const lockKey = `lock:${cacheKey}`
  const lockAcquired = await redis.set(lockKey, '1', 'NX', 'EX', 10)

  if (lockAcquired) {
    try {
      // 双重检查（防止锁等待期间已有其他请求更新了缓存）
      const doubleCheck = await redis.get(cacheKey)
      if (doubleCheck) {
        return JSON.parse(doubleCheck)
      }

      const tree = await buildPermissionTree(appTypeId)
      await redis.setex(cacheKey, 3600, JSON.stringify(tree))
      return tree
    } finally {
      await redis.del(lockKey)
    }
  } else {
    // 未获取到锁，等待后重试
    await sleep(100)
    return getHotPermissionTree(appTypeId)
  }
}
```
