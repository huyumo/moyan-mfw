# 权限池与角色权限接口统一优化方案

> **文档类型**: 技术方案
> **创建日期**: 2026-04-14
> **状态**: 待实施

---

## 1. 问题背景

当前权限池配置和角色权限分配两个功能使用相似的接口设计，但返回数据结构存在字段命名不一致的问题，导致：

1. 前端需要编写两套相似但不同的处理逻辑
2. 代码重复度高，维护成本增加
3. 概念上容易混淆，增加理解难度

---

## 2. 当前接口设计分析

### 2.1 权限池接口

**获取权限池配置**: `GET /api/v1/app-types/:appTypeId/permission-pool`

```typescript
// 返回数据结构
{
  code: number;
  data: {
    appTypeId: string;
    permissionTrees: {
      pcTree: PermissionTreeNode[];
      normalTree: PermissionTreeNode[];
    };
  };
}

// PermissionTreeNode 结构（权限池场景）
interface PermissionTreeNode {
  id: string;
  permName: string;
  permCode: string;
  // ... 其他权限节点字段
  inPool: boolean;              // 标识是否在权限池中
  permissionValue?: string;     // 位运算权限值
  children?: PermissionTreeNode[];
}
```

**更新权限池配置**: `PUT /api/v1/app-types/:appTypeId/permission-pool`

```typescript
// 请求数据结构
{
  permissionTrees: {
    pcTree: PermissionTreePayload[];
    normalTree: PermissionTreePayload[];
  };
}

interface PermissionTreePayload {
  permissionId: string;
  checked: boolean;             // 标识选中状态
  permissionValue?: string;
  children?: PermissionTreePayload[];
}
```

### 2.2 角色权限接口

**获取角色权限配置**: `GET /api/v1/app-types/roles/:id/permissions`

```typescript
// 返回数据结构
{
  code: number;
  data: {
    roleId: string;
    permissionTrees: {
      pcTree: PermissionTreeNode[];
      normalTree: PermissionTreeNode[];
    };
  };
}

// PermissionTreeNode 结构（角色权限场景）
interface PermissionTreeNode {
  id: string;
  permName: string;
  permCode: string;
  // ... 其他权限节点字段
  assigned: boolean;            // 标识是否已分配给角色
  permissionValue?: string;
  children?: PermissionTreeNode[];
}
```

**更新角色权限配置**: `PUT /api/v1/app-types/roles/:id/permissions`

```typescript
// 请求数据结构
{
  permissionTrees: {
    pcTree: PermissionTreePayload[];
    normalTree: PermissionTreePayload[];
  };
}

interface PermissionTreePayload {
  permissionId: string;
  checked: boolean;             // 标识选中状态
  permissionValue?: string;
  children?: PermissionTreePayload[];
}
```

### 2.3 问题总结

| 对比项 | 权限池接口 | 角色权限接口 |
|--------|-----------|--------------|
| 返回体选中字段 | `inPool: boolean` | `assigned: boolean` |
| 请求体选中字段 | `checked: boolean` | `checked: boolean` |
| 字段语义 | 是否在权限池中 | 是否已分配给角色 |

**核心问题**：
- 返回体使用不同的字段名（`inPool` vs `assigned`）
- 请求体统一使用 `checked`，但返回体不一致
- 前端需要编写两套字段转换逻辑

## 3. 优化后的接口设计

### 3.1 优化方案

统一返回体和请求体的字段命名，采用以下规则：

| 场景 | 返回体字段 | 请求体字段 | 说明 |
|------|-----------|-----------|------|
| 权限池配置 | `checked: boolean` | `checked: boolean` | 统一使用 checked |
| 角色权限分配 | `checked: boolean` | `checked: boolean` | 统一使用 checked |

**设计原则**：
1. 返回体与请求体使用相同的字段名
2. `checked` 表示"选中状态"，具体含义由上下文决定
3. 前端使用统一的处理逻辑

### 3.2 优化后的数据结构

**统一返回体结构**:

```typescript
// 权限池配置返回
interface PermissionPoolResponse {
  appTypeId: string;
  permissionTrees: {
    pcTree: PermissionTreeNode[];
    normalTree: PermissionTreeNode[];
  };
}

// 角色权限配置返回
interface RolePermissionResponse {
  roleId: string;
  permissionTrees: {
    pcTree: PermissionTreeNode[];
    normalTree: PermissionTreeNode[];
  };
}

// 统一的 PermissionTreeNode 结构
interface PermissionTreeNode {
  // 基础字段
  id: string;
  permName: string;
  permCode: string;
  permDesc?: string;
  permissionType: 'PC' | 'NORMAL';
  nodeType: 'MENU' | 'PAGE' | 'TAG';
  parentId?: string;
  
  // 路由相关
  routePath?: string;
  externalUrl?: string;
  iconName?: string;
  
  // 配置字段
  sortOrder: number;
  isVisible: number;
  isCache: number;
  showMode: 'NORMAL' | 'DEV';
  permStatus: number;
  
  // 统一的选中状态字段
  checked: boolean;              // 统一字段：选中状态
  permissionValue?: string;      // 位运算权限值（当前节点）
  parentPermissionValue?: string; // 位运算权限值（父层，用于子集验证）
  children?: PermissionTreeNode[];
}
```

**统一请求体结构**（保持不变）:

```typescript
interface PermissionTreePayload {
  permissionId: string;
  checked: boolean;              // 选中状态
  permissionValue?: string;      // 位运算权限值
  children?: PermissionTreePayload[];
}

interface UpdatePermissionConfigRequest {
  permissionTrees: {
    pcTree: PermissionTreePayload[];
    normalTree: PermissionTreePayload[];
  };
}
```

## 4. 接口返回结构对比

### 4.1 变更前后对比表

| 接口 | 变更前 | 变更后 |
|------|--------|--------|
| 权限池 GET | `{ ...node, inPool: boolean }` | `{ ...node, checked: boolean }` |
| 权限池 PUT | `{ checked: boolean }` | `{ checked: boolean }` (不变) |
| 角色权限 GET | `{ ...node, assigned: boolean }` | `{ ...node, checked: boolean }` |
| 角色权限 PUT | `{ checked: boolean }` | `{ checked: boolean }` (不变) |

### 4.2 字段语义对照

| 字段名 | 语义说明 | 适用场景 |
|--------|---------|---------|
| `checked` | 节点选中状态 | 权限池配置、角色权限分配 |
| `permissionValue` | 当前节点的位运算权限值 | PAGE/TAG 节点的权限值配置 |
| `parentPermissionValue` | 父层的位运算权限值 | 用于子集验证 |

**场景语义解释**：
- 权限池配置：`checked = true` 表示该权限已加入权限池
- 角色权限分配：`checked = true` 表示该权限已分配给角色

**parentPermissionValue 说明**：
- **用途**：前端渲染子集可选范围
- **权限池场景**：Permission 定义的 permissionValue，表示该权限可用的最大操作权限范围
- **角色权限场景**：权限池的 permissionValue，表示该角色可选的最大操作权限范围
- **前端使用**：根据 parentPermissionValue 渲染可选的权限位（ADD、EDIT、DELETE 等）
- **验证规则**：后端保存时验证 `(userPermissionValue & parentPermissionValue) === userPermissionValue`

**示例**：
```typescript
// 权限池场景
{
  id: "perm-001",
  permName: "用户管理",
  permissionValue: "7",        // 当前配置：ADD | EDIT | DELETE
  parentPermissionValue: "15"  // Permission 定义的最大范围：ADD | EDIT | DELETE | EXPORT
}

// 角色权限场景
{
  id: "perm-001",
  permName: "用户管理",
  permissionValue: "3",        // 当前分配：ADD | EDIT
  parentPermissionValue: "7"   // 权限池配置：ADD | EDIT | DELETE
  // 前端渲染时，只能勾选 ADD、EDIT、DELETE 三个选项
}
```

## 5. 前端代码简化方案

### 5.1 当前前端处理逻辑

**权限池面板** (`MfwPermissionPoolPanel.vue`):

```typescript
// 当前：需要从 inPool 字段收集已勾选的 ID
const collectInPoolIds = (nodes: PermissionTreeNodeDto[]): string[] => {
  const ids: string[] = []
  for (const node of nodes) {
    if (node.inPool) {           // 使用 inPool 字段
      ids.push(node.id)
    }
    if (node.children) {
      ids.push(...collectInPoolIds(node.children))
    }
  }
  return ids
}

checkedIds.value = [
  ...collectInPoolIds(pcTreeData.value),
  ...collectInPoolIds(normalTreeData.value),
]
```

**角色权限面板**（需要类似但不同的逻辑）:

```typescript
// 当前：需要从 assigned 字段收集已勾选的 ID
const collectAssignedIds = (nodes: PermissionTreeNodeDto[]): string[] => {
  const ids: string[] = []
  for (const node of nodes) {
    if (node.assigned) {         // 使用 assigned 字段
      ids.push(node.id)
    }
    if (node.children) {
      ids.push(...collectAssignedIds(node.children))
    }
  }
  return ids
}
```

### 5.2 优化后的前端处理逻辑

**统一处理函数**:

```typescript
// 优化后：统一的选中状态收集函数
const collectCheckedIds = (nodes: PermissionTreeNodeDto[]): string[] => {
  const ids: string[] = []
  for (const node of nodes) {
    if (node.checked) {          // 统一使用 checked 字段
      ids.push(node.id)
    }
    if (node.children) {
      ids.push(...collectCheckedIds(node.children))
    }
  }
  return ids
}

// 权限池面板和角色权限面板都使用相同的逻辑
checkedIds.value = [
  ...collectCheckedIds(pcTreeData.value),
  ...collectCheckedIds(normalTreeData.value),
]
```

### 5.3 类型定义简化

**当前类型定义**（需要两个不同的类型）:

```typescript
// 权限池场景
interface PermissionPoolTreeNode extends PermissionTreeNodeBase {
  inPool: boolean;
}

// 角色权限场景
interface RolePermissionTreeNode extends PermissionTreeNodeBase {
  assigned: boolean;
}
```

**优化后的统一类型定义**:

```typescript
// 统一类型，适用于两种场景
interface PermissionTreeNode {
  id: string;
  permName: string;
  permCode: string;
  // ... 其他字段
  checked: boolean;              // 统一字段
  permissionValue?: string;
  children?: PermissionTreeNode[];
}
```

### 5.4 组件复用改进

**优化后的 MfwPermissionTree 组件**:

```vue
<script setup lang="ts">
// 组件内部无需区分 inPool 或 assigned
// 统一读取 checked 字段
const collectCheckedIds = (nodes: PermissionTreeNode[]): string[] => {
  return nodes.reduce((acc, node) => {
    if (node.checked) acc.push(node.id)
    if (node.children) acc.push(...collectCheckedIds(node.children))
    return acc
  }, [] as string[])
}
</script>
```
