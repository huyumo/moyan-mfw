# 系统架构

## 整体架构

Moyan MFW 采用前后端分离的 Monorepo 架构，分为三层：

```
┌─────────────────────────────────────────────────────┐
│                  业务前端 (frontend)                  │
│          消费 base-frontend，提供业务页面             │
├─────────────────────────────────────────────────────┤
│             前端基础设施 (base-frontend)              │
│    组件库 + 布局系统 + 路由 + 状态管理 + 权限         │
├─────────────────────────────────────────────────────┤
│               业务后端 (backend)                      │
│    消费 moyan-base-backend，提供业务模块和 API        │
├─────────────────────────────────────────────────────┤
│           后端基础设施 (moyan-base-backend)            │
│    NestJS + TypeORM + MySQL + Redis + JWT            │
└─────────────────────────────────────────────────────┘
```

## 后端架构

### 模块划分

```
base-backend/src/
├── common/                    # 通用基础设施
│   ├── constants/             # 常量定义（权限位运算值）

│   ├── entities/              # 基类实体（Base）

│   ├── filters/               # 全局异常过滤器
│   ├── guards/                # 守卫（Auth、Permission）
│   ├── interceptors/          # 拦截器（Transform、Logging、Audit）
│   ├── types/                 # 通用类型定义

├── modules/
│   ├── health/                # 健康检查模块
│   └── sys/                   # 系统业务模块
│       ├── auth/              # 认证模块
```

### 请求处理流水线

```
HTTP 请求
  → CORS 中间件
  → AuthGuard（JWT 验证，解析用户信息）
  → PermissionGuard（位运算权限检查）
  → ValidationPipe（请求体校验，whitelist + transform）
  → LoggingInterceptor（请求日志）
  → Controller（业务处理）
  → AuditInterceptor（审计日志记录，通过 AuditLogService 写入数据库）
  → TransformInterceptor（统一响应格式）
  → AllExceptionsFilter（异常兜底）
  → HTTP 响应
```

### 统一响应格式

```typescript
{
  code: number;       // 业务状态码
  data: any;          // 业务数据
  message: string;    // 提示信息
  timestamp: string;  // 时间戳
}
```

### 布局系统

支持三种布局模式：

| 模式 | 说明 |
|------|------|
| `sidebar` | 侧边栏导航模式 |
| `top` | 顶部导航模式 |
| `dual` | 顶部+侧边栏双导航模式 |

### 路由与菜单

路由采用 **自动扫描模式**：

1. 页面组件放置在 `views/` 目录下
2. 每个页面是一个独立目录，包含 `index.ts` 配置文件
3. 配置文件导出 `page`（组件）、`path`（路径）、`name`（名称）
4. 框架通过 `import.meta.glob` 自动扫描生成路由和菜单
5. 支持模块分组（`type: 'module'`）
6. 支持权限位运算值自动计算（`definePageConfig` + `permissions`）

### 认证流程

```
1. 用户登录 → ApiAuthLogin → 获取 accessToken + refreshToken
2. Token 存储到 localStorage
3. 路由守卫检查 Token → 无 Token 跳转登录页
4. 认证初始化 → fetchUserInfo → fetchUserApps → autoSelectApp
5. 选择应用 → loadPermissions → 获取权限菜单
6. 权限菜单 → 转换为侧边栏菜单 → 同步到 layoutStore
7. 权限指令 v-permission → 检查按钮级权限
8. Token 过期 → 自动刷新 refreshToken
```

## 数据流

```
[前端业务应用]
    ↓ createBaseAdminApp()
[base-frontend 框架层]
    ↓ moyan-api (ApiCall 类)
    ↓ Axios HTTP Client
[backend 业务后端]
    ↓ createBaseBackendApp()
[moyan-base-backend 框架层]
    ↓ Controller → Service
    ↓ TypeORM QueryBuilder / PaginationX
[MySQL 数据库]
```
