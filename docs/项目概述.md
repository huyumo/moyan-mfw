# Moyan MFW - 墨焱管理框架

## 项目简介

Moyan MFW（墨焱管理框架）是一个面向后台管理场景的全栈框架，采用 Monorepo 架构组织前端基础设施包与后端服务，提供完整的权限管理、用户管理、应用管理等后台核心能力。

## 项目结构

```
moyan-mfw/
├── packages/
│   ├── base-backend/          # 后端基础设施包（NestJS，发布为 moyan-base-backend）

├── eslint.config.mjs          # ESLint 配置
└── .trae/                     # Trae IDE 配置与 Skill 文档
```

## 包说明

### base-backend

后端基础设施包，基于 NestJS 构建，提供 `createBaseBackendApp()` 工厂函数及 RESTful API 框架。

- 路径：`packages/base-backend/`
- 包名：`moyan-base-backend`
- 端口：3000
- API前缀：`/api`
- Swagger文档：`/api-docs`（支持分组）

### base-frontend

前端基础设施包，封装管理后台的通用组件、布局、路由、状态管理等，作为 npm 包供业务前端消费。

- 路径：`packages/base-frontend/`
- 包名：`moyan-mfw-base-frontend`
- 导出方式：ESM + CJS + 类型声明

### frontend

业务前端应用，消费 `moyan-mfw-base-frontend` 包，提供具体的业务页面和路由。

- 路径：`frontend/`
- 包名：`moyan-frontend`

### backend

业务后端应用，通过 `createBaseBackendApp()` 工厂函数创建 NestJS 实例，提供业务模块和 API。

- 路径：`backend/`
- 包名：`moyan-backend`
- 依赖：`moyan-base-backend`（workspace:*）

## 核心设计

### 权限体系

框架采用 **RBAC + 位运算** 的权限模型：

- 角色（Role）通过 `sys_role_permissions` 关联权限（Permission）
- 权限值使用 BigInt 位运算实现细粒度控制（添加/编辑/删除/导出/导入等）
- 支持多装饰器 OR 逻辑，一个接口可声明多个权限编码
- 开发者（isDeveloper）拥有全部权限，直接放行

### 多租户设计

- 应用类型（AppType）定义应用分类和权限池
- 应用实例（App）是应用类型的具体实例
- 角色可绑定到应用类型或应用实例，支持全局角色
- 权限池机制限定每种应用类型可用的权限范围

### 软删除

所有业务实体继承 `Base` 基类，支持 `createdAt`、`updateAt`、`deleteAt` 审计字段，删除操作为软删除。

## 开发命令

```bash
# 安装依赖
pnpm install

# 启动前端开发服务
pnpm dev:frontend

# 构建所有包
pnpm build

# 代码检查
pnpm lint

# 类型检查
pnpm typecheck

# 格式化
pnpm format
```
