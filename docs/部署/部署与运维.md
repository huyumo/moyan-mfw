# 部署与运维

## Docker Compose 部署

### 服务组成

docker-compose.yml 定义了三个服务：

| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| mysql | mysql:8.0 | 3306 | 数据库服务 |
| redis | redis:7-alpine | 6379 | 缓存服务 |
| app | node:20-alpine | 3000 | 应用服务 |

### 数据库配置

- 数据库名：`moyan_mfw`
- 用户名：`moyan`
- 密码：`moyan123456`
- 字符集：`utf8mb4`
- 时区：`+08:00`

### Redis 配置

- 地址：`localhost:6379`
- 前缀：`mfw:`
- TTL：1800秒
- 使用自定义 `redis.conf`

### 网络

- 网络名：`moyan-mfw-network`（bridge 模式）
- 数据卷：`mysql_data`、`redis_data`

## Dockerfile

采用三阶段构建：

| 阶段 | 基础镜像 | 说明 |
|------|---------|------|
| builder | node:20-alpine | pnpm install + build |
| production | 精简镜像 | 非 root 用户（nestjs:1001），健康检查 `/health`，`node dist/main.js` |
| development | node:20-alpine | 开发模式，`pnpm run start:dev`，挂载源码 |

## 本地开发

### 环境要求

- Node.js >= 20
- pnpm >= 8
- MySQL 8.0
- Redis 7（可选，目前未实际使用）

### 启动步骤

```bash
# 1. 安装依赖
pnpm install

# 2. 启动数据库（Docker）
cd packages/base-backend
docker-compose up -d mysql redis

# 3. 启动后端开发服务
cd packages/base-backend
pnpm run start:dev

# 4. 启动前端开发服务
pnpm dev:frontend
```

### 环境变量

在 `packages/base-backend/` 下创建 `.env` 文件：

```env
# 数据库
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=moyan
DB_PASSWORD=moyan123456
DB_DATABASE=moyan_mfw

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT
JWT_SECRET=your_jwt_secret_key
JWT_EXPIRES_IN=24h
JWT_REFRESH_EXPIRES_IN=7d

# 应用
APP_PORT=3000
```

### 系统初始化

首次部署后，访问 `/api/install/status` 检查系统是否已初始化。未初始化时，前端会自动引导至系统初始化页面。

初始化操作：

1. 访问 `/install` 页面
2. 填写管理员信息
3. 提交初始化请求到 `POST /api/install/init`
4. 种子数据自动创建（应用类型、管理员、权限、角色等）

## 健康检查

| 端点 | 说明 |
|------|------|
| `GET /api/health` | 综合健康检查（含DB延迟+内存使用） |
| `GET /api/health/ready` | 就绪检查 |
| `GET /api/health/live` | 存活检查 |

数据库健康检查服务每 30 秒执行 `SELECT 1`，失败时自动重连。

1. **JWT Secret**：生产环境务必修改 JWT Secret，当前为硬编码测试值
2. **数据库密码**：生产环境需修改默认数据库密码
3. **Redis**：配置已定义但业务层尚未使用，后续缓存功能需要激活
