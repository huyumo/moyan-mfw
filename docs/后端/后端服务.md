# 后端服务 (base-backend)

## 概述

`moyan-mfw-base-backend` 是框架的后端服务，基于 NestJS 构建，提供完整的后台管理 RESTful API。

- 包名：`moyan-mfw-base-backend`
- 路径：`packages/base-backend/`
- 端口：3000
- API前缀：`/api`
- Swagger文档：`/api-docs`

## 启动流程

入口文件 `src/main.ts` 的启动顺序：

1. 加载 `.env` 环境变量
2. 创建 NestJS 应用实例
3. 注册 BigInt 序列化中间件
4. 设置全局前缀 `/api`
5. 注册全局异常过滤器 `AllExceptionsFilter`
6. 注册全局拦截器：`LoggingInterceptor` → `TransformInterceptor` → `AuditInterceptor`
7. 注册全局验证管道 `ValidationPipe`（whitelist + transform）
8. 启用 CORS
9. 配置 Swagger 文档
10. 监听端口（默认 3000）

## 模块注册

根模块 `app.module.ts` 注册的模块：

### 全局模块

| 模块 | 说明 |
|------|------|
| ConfigModule (isGlobal) | 加载 database/app/redis/user 四组配置 |
| TypeOrmModule.forRootAsync | MySQL 连接（连接池100，自动重试10次，心跳30秒） |
| JwtModule.register (global) | JWT 认证（secret + 7200s过期） |

### 业务模块

| 模块 | 路径 | 功能 |
|------|------|------|
| AuthModule | modules/sys/auth | 认证登录 |
| UserModule | modules/sys/user | 用户管理 |
| RoleModule | modules/sys/role | 角色管理 |
| PermissionModule | modules/sys/permission | 权限管理 |
| AppTypeModule | modules/sys/app-type | 应用类型管理 |
| AppModule | modules/sys/app | 应用实例+成员管理 |
| AuditLogModule | modules/sys/audit-log | 审计日志 |
| HealthModule | modules/health | 健康检查 |
| InstallModule | modules/sys/install | 系统初始化 |

### 全局守卫

| 守卫 | 说明 |
|------|------|
| AuthGuard | JWT Token 验证，解析 `Authorization: Bearer <token>`，注入 `request.user = { id, username, roleIds }` |
| PermissionGuard | 位运算权限检查，查询 `sys_role_permissions` 表，支持多装饰器 OR 逻辑，开发者直接放行 |

## 通用基础设施 (common/)

### 装饰器

| 装饰器 | 文件 | 功能 |
|--------|------|------|
| `@Public()` | public.decorator.ts | 标记接口无需认证 |
| `@RequirePermission({ permCode, permissionValue? })` | require-permission.decorator.ts | 声明所需权限（支持多次注解，OR 逻辑） |
| `@AuditLog({ module, event, description? })` | audit-log.decorator.ts | 标记需审计的接口 |
| `@ApiPaginatedResponse()` | api-paginated-response.decorator.ts | Swagger 分页响应文档 |

### 自定义异常

| 异常类 | HTTP 状态码 | 说明 |
|--------|-----------|------|
| BusinessException | 400 | 业务异常基类 |
| UnauthorizedError | 401 | 未授权 |
| ForbiddenError | 403 | 禁止访问 |
| NotFoundError | 404 | 资源未找到 |

### 拦截器

| 拦截器 | 功能 |
|--------|------|
| TransformInterceptor | 统一响应格式 `{ code, data, message, timestamp }` |
| LoggingInterceptor | 请求日志（method, url, statusCode, 耗时） |
| AuditInterceptor | 审计日志拦截（目前仅日志输出，TODO: 写入数据库） |

### 工具函数

| 工具 | 功能 |
|------|------|
| `hashPassword` / `verifyPassword` | bcrypt 加密/验证（rounds=10） |
| `PaginationX` + `WhereBuilder` | 原生 SQL 分页查询构建器 |
| `executeRawSql` | 原生 SQL 执行封装 |
| `flatToTree` | 扁平列表转树形结构 |

## 权限常量

默认权限位：

| 名称 | 位值 |
|------|------|
| 添加 | 1n (1 << 0) |
| 编辑 | 2n (1 << 1) |
| 删除 | 4n (1 << 2) |
| 导出 | 8n (1 << 3) |
| 导入 | 16n (1 << 4) |

扩展权限位：

| 名称 | 位值 |
|------|------|
| 审批 | 32n (1 << 5) |
| 拒绝 | 64n (1 << 6) |
| 发布 | 128n (1 << 7) |
| 归档 | 256n (1 << 8) |

可通过环境变量 `PERMISSION_VALUES` 覆盖。提供 `buildPerValue`、`parsePerValue`、`hasPermission`、`getPermValue` 等工具函数。

## 种子数据

系统初始化时创建的种子数据：

1. 应用类型："系统管理"（typeCode: system）
2. 管理员账号：admin（默认密码: Admin@123）
3. 权限：2个根节点（pc_root, normal_root）+ 9个PC权限子节点
4. 角色：超级管理员（roleCode: super_admin, isBuiltin=1, isOwner=1）
5. 应用实例：系统管理后台（appCode: system-instance）
6. 权限池：所有PC权限添加到 system 应用类型
7. 角色权限：超级管理员绑定所有权限
8. 成员绑定：admin 加入 system-instance 成员 + super_admin 角色
