# 产品经理角色检查清单

## 需求分析阶段

- [ ] 用户需求是否明确（用户故事格式）
- [ ] 业务目标是否可衡量
- [ ] 功能边界是否清晰
- [ ] 非功能性需求是否考虑（性能、安全、可用性）
- [ ] 依赖关系是否识别

## 方案评审阶段

- [ ] 方案是否覆盖所有用户需求
- [ ] 是否有遗漏的边界情况
- [ ] 与现有系统是否兼容
- [ ] 是否有技术可行性风险
- [ ] 是否有更简单的替代方案

## 验收标准

- [ ] 验收标准是否可测试
- [ ] 是否定义明确的完成标准
- [ ] 是否有数据指标支撑

## 输出物检查

- [ ] PRD 文档结构完整
- [ ] 流程图/原型图清晰
- [ ] 术语使用一致
- [ ] 与现有文档无冲突
