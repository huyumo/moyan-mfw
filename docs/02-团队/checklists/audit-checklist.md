# 项目审查角色检查清单

## 需求评审

- [ ] 需求是否明确可执行
- [ ] 需求优先级是否合理
- [ ] 是否有镀金功能（过度设计）
- [ ] 是否有遗漏的核心功能

## 技术方案评审

- [ ] 技术选型是否合理
- [ ] 架构是否过度设计
- [ ] 是否有更简单的方案
- [ ] 风险是否识别
- [ ] 应急预案是否制定

## 进度评审

- [ ] 任务分解是否合理
- [ ] 工时估算是否准确
- [ ] 关键路径是否识别
- [ ] 里程碑是否可交付

## 质量评审

- [ ] 测试覆盖率是否达标
- [ ] 代码审查是否通过
- [ ] 文档是否完整
- [ ] 验收标准是否满足

## 风险评审

- [ ] 技术风险是否识别
- [ ] 依赖风险是否识别
- [ ] 人员风险是否有备份
- [ ] 延期风险是否有预案

## 交付评审

- [ ] 交付物是否完整
- [ ] 用户文档是否就绪
- [ ] 运维文档是否就绪
- [ ] 回滚方案是否就绪
