## 任务优先级定义

| 优先级 | 说明 | 响应时间 | 示例 |
|--------|------|----------|------|
| P0 | 紧急且重要 | 立即处理 | 线上 Bug、安全问题 |
| P1 | 重要不紧急 | 24 小时内 | 核心功能开发、关键文档 |
| P2 | 紧急不重要 | 48 小时内 | 一般功能优化、文档更新 |
| P3 | 不紧急不重要 | 待排期 | 技术债务、体验优化 |

## 任务状态定义

| 状态 | 说明 | 流转条件 |
|------|------|----------|
| 待开始 | 任务已创建，等待执行 | 负责人确认接收 |
| 进行中 | 任务正在执行 | 提交成果物 |
| 审查中 | 等待审查验收 | 审查通过/不通过 |
| 已完成 | 任务完成并验收 | - |
| 已关闭 | 任务取消或废弃 | - |

## 附录：会话流程规则（新增）

> 本章节定义会话开始/结束时的任务追踪流程，确保跨会话工作连续性。

### 会话开始流程

```
1. 读取 TASK.md
       ↓
2. 向用户确认任务状态
   - "我看到当前任务是 [X]，状态是 [Y]，是否正确？"
   - 如有变化，先更新 TASK.md
       ↓
3. 确认会话目标
   - "今天/这次会话计划完成什么？"
       ↓
4. 开始工作
```

**违规处理**：如未执行会话开始流程，用户应提醒"请先读取 TASK.md 并确认任务状态"

---

### 会话结束流程

```
1. 总结本次会话完成的工作
   - 列出所有完成项
       ↓
2. 更新 TASK.md
   - 标记已完成任务
   - 更新进行中任务状态
   - 记录下一步行动
       ↓
3. 记忆保存检查
   - 如有重要决策，询问"是否需要保存到记忆？"
       ↓
4. 确认会话结束
```

**违规处理**：用户应在会话结束前提醒"请先更新 TASK.md"

### 多会话锁机制

**问题**：同时打开多个 VSCode 窗口时，可能并发写入 TASK.md

**解决方案**：在 TASK.md 头部记录：

```yaml
session: be2d86c6-3b77-43f3-bfba-6fadbea86dfb  # 当前会话 ID
lock: 1711785600  # Unix 时间戳（秒）
```

**写入规则**：
1. 写入前检查 `lock` 时间戳
2. 如当前时间 - lock < 300 秒（5 分钟）→ 警告"有其他会话正在工作"
3. 写入时更新 `lock` 为当前时间戳
