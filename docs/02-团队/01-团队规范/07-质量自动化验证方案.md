# 质量自动化验证方案

> 版本：1.0.0 | 创建：2026-03-30 | 维护：@maintainer

---

## 概述

本方案定义自动化质量验证工具，用于在任务执行过程中自动检查规范符合性，减少人工检查成本，提高质量保障效率。

---

## 验证体系架构

```
┌─────────────────────────────────────────────────────────┐
│                    任务执行流程                          │
│                                                          │
│  任务开始 → 执行中 → 任务完成 → 自检 → 归档              │
│                          ↓         ↓                     │
│                   [自动验证]   [人工检查清单]             │
│                          ↓         ↓                     │
│                   [验证报告]   [检查记录]                 │
│                          ↓         ↓                     │
│                   全部通过 → 归档                         │
└─────────────────────────────────────────────────────────┘
```

## 自动化验证工具列表

### 工具 1：文档链接检查器

**脚本位置**：`scripts/validate-doc-links.ts`

**功能**：
- 检查 Markdown 文档中的内部链接是否有效
- 检查外部链接是否可访问（可选）
- 输出失效链接报告

**使用方式**：
```bash
# 检查所有文档链接
npm run validate:links

# 检查指定目录
npm run validate:links -- --dir docs/03-框架规范
```

**输出示例**：
```
[ERROR] docs/01-业务需求/01-基础设施/02-核心概念/权限系统.md:45
  → 链接失效：`[权限系统](./权限系统.md)`
  → 文件不存在：docs/01-业务需求/01-基础设施/02-核心概念/权限系统.md

[WARNING] docs/02-团队/01-团队规范/01-团队章程.md:120
  → 外部链接响应慢：https://example.com (2.3s)

[SUMMARY] 检查完成：108 篇文档，2 个错误，5 个警告
```

### 工具 2：文档模板检查器

**脚本位置**：`scripts/validate-doc-template.ts`

**功能**：
- 检查文档是否包含必需的 Front Matter
- 检查文档是否包含必需的章节
- 检查文档格式是否符合规范

**使用方式**：
```bash
# 检查所有文档
npm run validate:docs

# 检查指定文档
npm run validate:docs -- --file docs/03-框架规范/01-前端规范/01-编码基础规范.md
```

**输出示例**：
```
[ERROR] docs/03-框架规范/02-后端规范/01-编码基础规范.md
  → 缺少 Front Matter 字段：version
  → 缺少必需章节：## 验收标准

[OK] docs/03-框架规范/01-前端规范/01-编码基础规范.md
  → 所有检查通过

[SUMMARY] 检查完成：27 篇文档，1 个错误，25 通过
```

### 工具 3：代码规范检查器（ESLint + Stylelint）

**脚本位置**：已有 ESLint/Stylelint 配置

**功能**：
- 检查 TypeScript/Vue 代码规范
- 检查组件命名是否符合 `Mfw` 前缀要求
- 检查是否有 `any` 类型未标注

**配置扩展**（`eslint.config.js`）：
```javascript
export default [
  // ... 现有配置
  {
    rules: {
      // 组件命名规则
      'vue/multi-word-component-names': ['error', {
        ignores: ['Mfw[A-Z]\\w+'] // 要求 Mfw 前缀
      }],
      // 禁止 any
      '@typescript-eslint/no-explicit-any': 'error'
    }
  }
]
```

**使用方式**：
```bash
# 检查代码规范
npm run lint

# 自动修复
npm run lint:fix
```

### 工具 4：类型定义检查器

**脚本位置**：`scripts/validate-types.ts`

**功能**：
- 检查组件 Props 类型定义是否完整
- 检查 Emits 类型定义是否完整
- 检查导出的类型是否有文档注释

**使用方式**：
```bash
# 检查所有组件
npm run validate:types

# 检查指定组件
npm run validate:types -- --component MfwFormCard
```

**输出示例**：
```
[ERROR] packages/base-frontend/src/components/form/mfw-form-card/Index.vue
  → Props 类型未定义：columnList
  → Emits 类型未定义：change

[WARNING] packages/base-frontend/src/components/form/mfw-form-card/types.ts
  → 接口 FormCardConfig 缺少 JSDoc 注释

[SUMMARY] 检查完成：15 个组件，1 个错误，3 个警告
```

### 工具 5：Git 提交检查器

**脚本位置**：`scripts/validate-commit.ts`

**功能**：
- 检查提交信息是否符合规范
- 检查提交是否关联任务
- 检查是否有敏感文件提交

**使用方式**：
```bash
# Git hook 自动执行（.git/hooks/pre-commit）
# 或手动检查
npm run validate:commit
```

**输出示例**：
```
[ERROR] Commit message 格式错误
  → 当前："修复 bug"
  → 期望："<type>: <description>" (如 "fix: 修复登录页样式问题")

[WARNING] 未关联任务
  → 提交信息中未包含任务编号（如 TASK-001）

[ERROR] 敏感文件检查
  → 发现 .env 文件，已阻止提交
```

1. 任务执行完成后，首先自我反思
   ↓

   - 类型检查：npm run validate:types

   ↓

   ↓

### 集成点 2：Git 提交前检查

**.git/hooks/pre-commit**：
```bash
#!/bin/bash

# 文档变更检查
if git diff --cached --name-only | grep -q "\.md$"; then
  echo "Running document validation..."
  npm run validate:docs
  if [ $? -ne 0 ]; then
    echo "❌ Document validation failed. Please fix issues before committing."
    exit 1
  fi
fi

# 代码变更检查
if git diff --cached --name-only | grep -qE "\.(ts|tsx|vue)$"; then
  echo "Running code lint..."
  npm run lint
  if [ $? -ne 0 ]; then
    echo "❌ Lint failed. Please fix issues before committing."
    exit 1
  fi
fi

echo "✅ All pre-commit checks passed."
exit 0
```

### 集成点 3：CI/CD 检查

**.github/workflows/quality-check.yml**：
```yaml
name: Quality Check

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm ci

      - name: Document validation
        run: npm run validate:docs

      - name: Code lint
        run: npm run lint

      - name: Type validation
        run: npm run validate:types

      - name: Link validation
        run: npm run validate:links
```

## 验证命令汇总

```bash

# 代码验证
npm run lint                   # ESLint/Stylelint
npm run lint:fix               # 自动修复
npm run validate:commit        # 提交信息检查

# 全量验证
npm run validate:all           # 运行所有验证
```
