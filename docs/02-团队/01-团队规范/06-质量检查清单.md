## 前端代码检查清单

**适用任务**：Vue 组件、Composables、TSX 组件

### 命名规范
- [ ] 组件名使用 `Mfw` 前缀（如 `MfwFormCard`）
- [ ] 目录名使用 camel-case + 连字符（如 `mfw-form-card/`）
- [ ] 主组件文件 PascalCase（如 `Index.vue`）
- [ ] 子组件文件 kebab-case（如 `column-control.vue`）
- [ ] composable 使用 `use` 前缀（如 `use-draft-dox.ts`）
- [ ] 类型定义文件命名为 `types.ts`

### 类型定义
- [ ] Props 类型完整定义
- [ ] Emits 类型完整定义
- [ ] Expose 类型完整定义
- [ ] 无 `any` 类型（除非必要且有注释）
- [ ] 复杂配置项有独立接口定义

### 代码质量
- [ ] 组件使用 `defineComponent`（基础组件）或 `<script setup lang="ts">`（业务组件）
- [ ] 关键逻辑有注释
- [ ] 错误处理完善
- [ ] 无 console.log 调试代码

### 测试覆盖
- [ ] 关键功能有单元测试
- [ ] 边界条件有测试
- [ ] 异常场景有测试

## 检查清单使用说明

### 何时使用

| 时机 | 使用清单 |
|------|----------|
| 文档编写完成 | [文档质量检查清单](#文档质量检查清单) |
| 代码编写完成 | [前端/后端代码检查清单](#前端代码检查清单) |
| 设计稿完成 | [设计稿检查清单](#设计稿检查清单) |
| 测试用例完成 | [测试用例检查清单](#测试用例检查清单) |
| 任务归档前 | [任务归档前检查清单](#任务归档前检查清单) |

### 如何使用

1. **任务完成后**，打开对应清单
2. **逐项核对**，完成的打勾 ` [x] `
3. **发现问题**，立即修复后重新检查
4. **全部打勾**后，截图/记录到任务详情文档
5. **未全部打勾**，不得进入归档流程

### 记录方式

在任务详情文档中添加：

```markdown
## 质量检查记录

**检查日期**: 2026-03-30
**检查人**: @frontend (或对应角色)

**使用清单**: [前端代码检查清单](#前端代码检查清单)

**检查结果**:
- 命名规范：✅ 5/5
- 类型定义：✅ 5/5
- 代码质量：✅ 4/4
- 测试覆盖：✅ 3/3

**总计**: ✅ 17/17 通过
```
