## 修复方案

### 文档定位

| 文档 | 适用范围 | 权限类型 | 节点类型 | 同步功能 |
|------|---------|---------|---------|---------|
| `permission-management.md` | 普通权限管理 | NORMAL | MENU + TAG | ❌ |
| `pc-permission-management.md` | PC 权限管理 | PC | MENU + PAGE | ✅ |

### 设计原理

**为什么分为两个页面？**

1. **系统架构层面**: `PermissionType` 本身就是隔离的
   ```typescript
   enum PermissionType {
     PC = 'PC',         // PC 后台权限 → 同步功能
     NORMAL = 'NORMAL', // 移动端/其他 → 手动管理
   }
   ```

2. **用户体验层面**:
   - PC 权限：开发者定义路由 → 同步生成 → 配置 pcAction
   - 普通权限：产品/运营手动创建 → 灵活配置

3. **业务场景层面**:
   - PC 后台：路由稳定、结构清晰 → 适合自动化同步
   - 移动端/其他：需求多变、灵活性高 → 适合手动管理
