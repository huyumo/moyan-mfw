# 测试工程师角色检查清单

## 编写前准备（强制）

- [ ] 是否已阅读测试规范文档
- [ ] 是否已验证页面 DOM 结构（使用 Playwright codegen 或 Grep 工具）
- [ ] 是否已确认后端 API 可访问且返回格式正确
- [ ] 是否已确认选择器与实际代码匹配（不假设页面结构）

## 选择器规范（强制）

- [ ] 是否优先使用稳定选择器（role、text、test-id）
- [ ] 是否避免使用不稳定选择器（CSS class、动态 ID）
- [ ] 多匹配元素是否使用 `.first()` 或更具体选择器
- [ ] 弹窗/对话框是否使用 `data-testid` 定位

## 测试用例设计

- [ ] 正常场景是否覆盖
- [ ] 边界值是否测试
- [ ] 异常场景是否测试
- [ ] 空值/null 是否测试
- [ ] 权限场景是否测试

## 功能测试

- [ ] 需求是否全部覆盖
- [ ] 业务流程是否通顺
- [ ] 数据一致性是否正确
- [ ] 状态流转是否正确

## 集成测试

- [ ] 接口调用是否正确
- [ ] 前后端数据交互是否正确
- [ ] 第三方依赖是否 mock
- [ ] 数据库操作是否正确

## 回归测试

- [ ] 现有功能是否受影响
- [ ] 兼容性是否测试
- [ ] 历史 bug 是否复现

## 性能测试

- [ ] 响应时间是否达标
- [ ] 并发场景是否测试
- [ ] 内存/资源使用是否合理

## 安全测试

- [ ] 未授权访问是否拦截
- [ ] 越权操作是否拦截
- [ ] 敏感数据是否脱敏
- [ ] 注入攻击是否防护

## 与开发协作（强制）

- [ ] 是否已确认前端组件有 data-testid 属性
- [ ] 是否已确认 API Schema 与后端 DTO 一致
- [ ] 测试失败时是否先验证实际页面结构再修改测试代码
- [ ] 是否已提交页面结构验证报告（选择器清单）

## 测试报告

- [ ] 测试结果是否记录
- [ ] bug 描述是否清晰
- [ ] 复现步骤是否完整
- [ ] 优先级是否合理
