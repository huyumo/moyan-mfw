# 后端工程师角色检查清单

## API 设计

- [ ] RESTful 规范是否遵循
- [ ] 请求参数验证是否完整
- [ ] 响应格式是否统一
- [ ] 错误码定义是否清晰
- [ ] 接口文档是否同步更新

## 业务逻辑

- [ ] 业务规则实现是否正确
- [ ] 边界条件是否处理
- [ ] 事务边界是否正确
- [ ] 并发问题是否考虑
- [ ] 幂等性是否保证（写操作）

## 数据安全

- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] 敏感数据是否加密
- [ ] 权限校验是否完整
- [ ] 输入是否校验（长度、类型、范围）

## 数据库

- [ ] 索引是否合理使用
- [ ] 是否有慢查询风险
- [ ] 事务使用是否正确
- [ ] 数据一致性如何保证
- [ ] 软删除/硬删除策略

## 日志与监控

- [ ] 关键操作是否有日志
- [ ] 异常是否有堆栈追踪
- [ ] 是否有审计日志
- [ ] 监控指标是否定义

## 性能

- [ ] 是否有缓存策略
- [ ] 是否有批量操作优化
- [ ] 是否有 N+1 查询
- [ ] 连接池配置是否合理
