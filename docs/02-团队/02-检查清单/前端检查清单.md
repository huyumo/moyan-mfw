# 前端工程师角色检查清单

## 组件设计

- [ ] 组件职责是否单一
- [ ] Props 定义是否清晰
- [ ] 事件命名是否规范
- [ ] 状态管理是否合理
- [ ] 是否可复用

## 用户体验

- [ ] 加载状态是否有反馈
- [ ] 错误状态是否有提示
- [ ] 空状态是否有引导
- [ ] 操作是否有确认（危险操作）
- [ ] 表单是否有验证

## 交互逻辑

- [ ] 边界条件是否处理（空数据、长文本、特殊字符）
- [ ] 异步操作是否有 loading
- [ ] 网络错误是否有重试/降级
- [ ] 权限控制是否正确

## 代码规范

- [ ] 是否符合 Vue 代码规范
- [ ] TypeScript 类型是否完整
- [ ] 是否有未使用的导入/变量
- [ ] 样式是否隔离
- [ ] 响应式依赖是否正确

## 性能优化

- [ ] 列表是否有 key
- [ ] 是否有不必要的渲染
- [ ] 大图/多图是否懒加载
- [ ] 组件是否按需加载

## 与后端接口

- [ ] 接口定义是否与文档一致
- [ ] 错误码处理是否完整
- [ ] 数据格式转换是否正确
- [ ] 是否有防抖/节流
- [ ] 接口变更是否有沟通确认

## 可访问性 (可选)

- [ ] 是否支持键盘操作
- [ ] 焦点管理是否正确
- [ ] 图片是否有 alt 文本
- [ ] 颜色对比度是否达标

## 国际化 (如适用)

- [ ] 文案是否使用 i18n
- [ ] 日期/时间格式是否正确
- [ ] 数字/货币格式是否正确

## 测试友好性（强制）

- [ ] 关键交互元素是否有 `data-testid` 属性（按钮、表单、弹窗）
- [ ] `data-testid` 命名是否规范（`[功能]-[元素]-[动作]`，如 `user-search-btn`）
- [ ] 列表项是否有唯一标识（`data-testid="user-item-{id}"`）
- [ ] 弹窗/对话框是否有标识（`data-testid="user-edit-dialog"`）

## 与后端接口一致性（强制）

- [ ] API Schema 类型定义是否与后端 DTO 一致
- [ ] 枚举值是否匹配（如 `permissionType: 'PC' | 'NORMAL'`）
- [ ] bigint 字段是否正确处理（字符串 ↔ BigInt）
- [ ] 必填/可选字段是否一致

## 交付前自测（强制）

- [ ] 本地是否运行相关 E2E 测试通过
- [ ] 页面是否在无数据/错误数据下正常显示
- [ ] 表单验证是否完整（空值、格式、边界）
