# 红线规则（智能体禁止事项）

1. **重点** 所有编码任务结束后，必须运行类型检查命令确认零错误。

2. **禁止手动编写或改动 `apis` 目录下的任何文件**。`apis` 目录存放的是由 `moyan-api` 包自动生成的 API 代码，任何手动修改都会被后续生成覆盖。如需变更 API 定义，应阅读apis下的README.md文件，调用`moyan-api` 工具重新生成。

3. **项目中任何 `.ts` 文件（包括测试文件）不允许存在 TypeScript 类型错误**。所有文件必须通过 `tsc --noEmit` 类型验证，这是硬性要求。编码完成后必须运行类型检查命令确认零错误。

4. **测试任务不允许修改业务代码**。测试任务只能测试业务逻辑，不能修改业务逻辑。

5. **测试用例编写不能以通过测试为标准**。测试用例必须基于业务需求，不能基于实现细节。
